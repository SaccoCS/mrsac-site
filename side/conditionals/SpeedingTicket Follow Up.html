SpeedingTicket Follow Up

Often programs have more than one way of accomplishing the same task.  It is important that you are flexible in your programming style so that you don't get stuck using one method that might not work for every situation. 

For Example:  Look at the following three methods, which all tell if a value is positive.

A very common solution

public boolean isPositive(int x)
{
   if( x > 0 )
   {
      return true;
   }
   else
   {
      return false;
   }
}
This solution works similar to the previous, but instead of immediately returning a true or false, the value is stored in a variable that is returned later 
public boolean isPositive(int x)
{
   boolean toReturn = false;

   if( x > 0 )
   {
      toReturn = true
   }
   else
   {
      toReturn = false;
   }

   return toReturn;
}
This minimalistic solution takes a different approach.  Instead of following the "if (x>0) is true I'll return true and if (x>0) is false I'll return false" strategy, this method just returns the result of (x>0). 

This is the strategy that scares people the most.  Remember that x>0 is going to be a true or false value.  Returning that value is fine if it fits the situation.
public boolean isPositive(int x)
{
   return  x > 0;
}

 

Speeding Ticket:

Read through the solutions to the speedingTicket method below and take note of what makes each of them different.

Problem Statement:

The speedingTicket method is meant to return true if a speeding ticket is issued, and false if not. There are two parameters that are passed in. The speed variable is an int represents how fast the car is going, and the isRed variable is a boolean which will be true if the car is red and false if not.

For most cars, they will recieve a ticket if the car is going above 65 miles per hour. If the car is red, a ticket is given for driving over 60 miles per hour.

 

The area below contains many different ways of solving the speedinTicket problem.  Read through them and try to follow the logic of each one.  Essentially there are four combinations that you can test:  A red car going over 60, a red car going 60 or lower, a non-red car going over 65, and a non-red car going 65 or under.

public static boolean speedingTicket(int speed, boolean isRed)
{
    if( isRed )
    {
        if( speed > 60 )
            return true;
        else
            return false;
    }
    else
    {
        if( speed > 65)
            return true;
        else
            return false;
   }
}
 		 This is a very common way of solving this problem.  It first uses the isRed variable to split the code into two parts: the red car and the non-red car.  In each section they use the different speed limits to determine if they get a ticket or not. 
public static boolean speedingTicket(int speed, boolean isRed)
{
    int limit = 65;

    if( isRed )
       limit -= 5;

    if( speed > limit )
        return true;
    else
        return false;
}
 		 In this case the speed limit of 65 is put into a variable named limit.  If the car is red, the variable limit is lowered by 5.  After that point in code, the speed is compared to whatever the value of limit is. 
public static boolean speedingTicket(int speed, boolean isRed)
{
    int limit;

    if( isRed )
        limit = 60;
    else
        limit = 65;

    if( speed > limit )
        return true;
    else
        return false;
}

public static boolean speedingTicket(int speed, boolean isRed)
