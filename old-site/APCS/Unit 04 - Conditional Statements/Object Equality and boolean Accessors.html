Run the runner: The result is an unexpected "not equal".  This doesn’t seem to make sense. All of the instance fields are the same, which we would assume makes them the equal to each other, but == just doesn’t work that way.

 

Big Concept: == compares the REFERENCES not the INSTANCE FIELDS

Don’t forget that the variable doesn’t hold the actual object. It only holds a reference to the object.  When using == it's a comparison of references and has nothing to do with the instance fields.

Will it ever return true? Yes, but in a less useful way. The only time that it will return true is if the two reference variables are talking about the exact same object. Not an object with the same instance fields. Literally the exact same object.

For example:

public static void main()
{
   Point p1 = new Point(3,5);
   Point p2 = p1;

   String reply;

   if(p1 == p2)
     reply = " are equal";
   else
     reply = " are not equal";
            
            
   System.out.println(p1+" and "+ p2 +reply);
}

In this case, p1 == p2 is true. The key is the line:

Point p1 = p2;

Remember that this doesn’t create a new Point. It copies a reference to the same exact object. p1 == p2 returns true because they are the same reference. This has some use, but you will find that at this level it’s almost never what you want to do.

 

Equality must be programmed into the object as an accessor method:

In order to determine if two objects are equal, the programmer must provide a way via an accessor methods.   In the Point class, add a method with the following header:

public boolean equals(Point other)

This method is an accessor with a Point as a parameter. This means that if you have two Point objects you may ask one if it equals the other. The method returns a boolean, so it can be used to run an if statement.

Using our example from before, instead of saying:

if( p1 == p2 )

We should actually use:

if( p1.equals(p2) )

or, just as good:

if( p2.equals(p1) )

 

How can we write a boolean accessor method?

When writing the equals method, our job is to compare our instance fields to the instance fields of the incoming object and return true if they meet our conditions of equality, meaning that the x's and y's must be the same. Because they are private we can access our own directly, but for the parameter Point we must call the accessor methods.

public boolean equals(Point other)
{
   int otherX = other.getX();
   int otherY = other.getY();

   if( otherX == x && otherY == y)
      return true;
   else
      return false;
}

Now that the Point class has an equals method, the runner can use it instead of ==

public static void main()
{
   Point p1 = new Point(3,5);
   Point p2 = new Point(3,5);

   String reply;

   if(p1.equals(p2))
      reply = " are equal";
   else
      reply = " are not equal";
            
            
   System.out.println(p1+" and "+ p2 +reply);
}

Now add another method to the Rectangle class with the following header:

public boolean containsPoint(Point p)

The purpose of this method is to return true if the parameter Point is within the boundaries of the Rectangle.  Don't forget that four things must be true for this to happen.  It may be helpful to first figure out the x values of the left/right side of the Rectangle as well as the y values of the top/bottom of the Rectangle.

 

Use the following Runner to test your containsPoint method.  The only method that you should really look at is the onMousePress method, which uses the MouseEvent to create a Point and then chooses a String based on the result of the containsPoint method.

import sacco.*;
import sacco.gui.*;

public class RectangleViewer extends SaccoWindow
{
    private Rectangle rect;
    private String inOrOut;
    
    public RectangleViewer(Rectangle rect)
    {
        this.rect = rect;
        inOrOut = "Click!";
    }

    public void paintWindow(Canvas c)
    {
        c.setColor("RED");
        c.fillRectangle(rect.getX(), rect.getY(),rect.getWidth(),rect.getHeight());
        c.setColor("BLACK");
        c.setThickness(5);
        c.drawRectangle(rect.getX(), rect.getY(),rect.getWidth(),rect.getHeight());
        
        if(inOrOut != null)
        {
            Font f = new Font(30);
            f.setBold(true);
            c.setFont(f);
            
            
            int width = c.getStringWidth(inOrOut);
            int x = rect.getX()+rect.getWidth()/2-width/2;
            int y = rect.getY()+rect.getHeight()/2+c.getStringAscent()/2-c.getStringDescent();
            c.drawString(inOrOut,x,y);
        }
    }

    public void onMousePress(MouseEvent m)
    {
        Point p = new Point(m.getX(),m.getY());       

        if(rect.containsPoint(p))
            inOrOut = "Inside";
        else
            inOrOut = "Outside";
    }

    public static void main()
    {
        Rectangle r = new Rectangle(50,150,150,250);
        RectangleViewer rView = new RectangleViewer(r);
        rView.showWindow();
    }
}
