SaccoWindows and Self Drawing Objects

Previously we've extended SaccoWindows that drew some pictures.  In more complicated programs objects get involved to make things easier. 

Copy the following class into your project.  Notice that it does the three things that every SaccoWindow needs:

1) import sacco.*;
2) extends SaccoWindow
3) overrides the paintWindow method

import sacco.*;

public class QuestionBlockWindow extends SaccoWindow
{

   public QuestionBlockWindow()
   {

   }

   public void paintWindow(Canvas c)
   {
      //draw sky
      c.setColor("blue");
      c.fillRectangle(0,0,500,500);

      //draw grass
      c.setColor("green");
      c.fillRectangle(0,336,500,200);

      //draw border line
      c.setColor("black");
      c.setThickness(3);
      c.drawLine(0,336,500,336);

      //draw cloud
      c.setColor("white");
      c.fillOval(288,48,144,48);

      c.setColor("black");
      c.drawOval(288,48,144,48);
   }

}

Now set up the following runner, which constructs a new QuestionBlockWindow and calls its showWindow method:

public class QuestionBlockWindowRunner
{
    public static void main()
    {
        QuestionBlockWindow myWindow = new QuestionBlockWindow();
        myWindow.showWindow();
    }
}

Using a Self Painting Object:

The QuestionBlockWindow class is set up to draw a basic background which we're going to improve with some self drawing objects.  In sacco.jar there is a QuestionBlock class, which has instance fields that keep track of an x, y, and an image to display.  We want to display this QuestionBlock, so we'll need to incorporate it into our project.

1)  Make it an instance field:
This object is one that we want to keep track of for the life of the Window, so we'll declare it as an instance field.

private QuestionBlock myBlock;

2)  Set it up in the constructor:
Go to the QuestionBlockWindow's constructor and use it to set up the instance field that you just created.  You can't set the x,y or scale using just the constructor, so this is a multi-step process.

myBlock = new QuestionBlock();
myBlock.setX(240);
myBlock.setY(192);
myBlock.setScale(3);

Now we've got an instance field that has an x,y, and scale that we want, but when we run our program it still doesn't show up.  That's because our paintWindow method hasn't been changed to draw the QuestionBlock object.

Read through the current paintWindow method; it uses Canvas' methods like fillRectangle and drawLine to get the background, but there is no drawQuestionBlock method.  The answer lies in the QuestionBlock class.  There is a paintSelf method that will accept a Canvas and draw the QuestionBlock using it.

The paintWindow method's strategy is now going to be the following:
- paint the background using the Canvas object
- hand the Canvas variable to the QuestionBlock so that it can draw itself on it.  This is how the images get combined.

Change your paintWindow method to be the following:

        //draw sky
        c.setColor("blue");
        c.fillRectangle(0,0,500,500);

        //draw grass
        c.setColor("green");
        c.fillRectangle(0,336,500,200);

        //draw border line
        c.setColor("black");
        c.setThickness(3);
        c.drawLine(0,336,500,336);

        //draw cloud
        c.setColor("white");
        c.fillOval(288,48,144,48);
        
        c.setColor("black");
        c.drawOval(288,48,144,48);

        //have the block paint itself using the Canvas
        myBlock.paintSelf(c);

The only difference is the the very last line, which hands off the Canvas to the QuestionBlock so that it can paint itself on top.

Assignment 2:  Creating a self-painting object

Create a class named CIrcle, which will be in charge of keeping track of a circle's location, radius, and color.

Add three ints as instance fields to represent the Circle's location and radius.  Also add a String to represent the Color.

The Constructor for Circle should take in three ints as parameters, an x,y, and radius.  The color should default to "red".

Add a method with the following header:

public void paintSelf(Canvas c)

This method should use the Canvas object to paint a Circle like the one below, using the x,y,radius, and color instance fields

CANVAS API

Copy the following class into your project.  Notice that it's similar to the QuestionBlockWindow above

import sacco.*;
public class CircleWindow extends SaccoWindow
{
    private Circle circ;
    
    public CircleWindow()
    {
        circ = new Circle(0,0,64);
    }
    
    @Override
    public void paintWindow(Canvas c)
    {
        circ.paintSelf(c);
    }
    
    public static void main()
    {
        CircleWindow cWind = new CircleWindow();
        cWind.showWindow();
    }
}

 

 

Keyboard interaction with SaccoWindows:

Similar to using MouseEvents, SaccoWindows can handle keyboard events as well.

In the CircleWindow class, add the following method:

@Override
public void onKeyPress(int key)
{
   System.out.println(key);
}

Every key on the keyboard has an int assosciated with it.  Run the program and hit a few keys.  You'll notice that a different int will be printed out each time.

 

Adding the Ability to Move the Circle:

Now that we've got the ability to listen for key presses we can make the circle move in all four directions.  The issue is that we need a good way to communicate what direction the Circle should move.

Add four public constants named UP, DOWN, LEFT, and RIGHT to the Circle class.  Put any random int that you want into them.  As long as they are different, we don't care about the values because we are only going to use the four named variables.

Add a mutator method named move, which takes in an int as a parameter and is in charge of moving the circle in one of the four directions by changing the x or y variable.  The int passed in will be equals one of the four direction constants.  You should use four if statements, each one testing if the parameter matches a direction constant and you should increase or decrease the x or y variable by one radius.

 

Helping the Window Control the Circle:

Now that the Circle has the ability to change its location, we just need to have the window's onKeyPress method control the circle.  It's time for another set of if statements.  This time the if statements are looking at the key int and looking for the four arrow keys ( left is 37, up it 38, right is 39, down is 40).

Depending which keycode you find, you should call the circle's move method and pass in the appropriate public constant:

For Example:

if( key == 37 )
   circ.move(Circle.LEFT);

 

This is one use of the public constants.  We don't care about the exact values stored in them, but they allow us to tell the move method which direction to go while still remaining readable.

Run the program one more time.  Try out the arrow keys and check to see if you can move the circle around.
