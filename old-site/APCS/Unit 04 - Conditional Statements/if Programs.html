if Programs

 

Each of the following programs requires the use of if statements.  Think carefully of whther or not you need to have an else attached to the if.

 

Color

Copy the Color class from the previous unit into the current project.

One important rule of colors is that the RGB values cannot be outside of the range 0-255.  Going outside of this range can cause programs to crash.

Color's constructor currently takes in three parameters and blindly assigns them to the instance fields.  Modify it using if statements that will protect the Color from going out of range when given bad parameters.  For each of the three instance fields, if the value is too large or too small, assign it to 255 or 0.

Create a runner that uses a Scanner to ask the user for 3 values and construct a Color object with the values.  After that, print out the object.

Run the program several times and make sure that any value that is out of range is automatically set to 0 or 255.  For example: if you give the values 270, 50, and -9, the color printed out should have the adjusted values 255, 50, and 0

 

 

BankAccount:

Copy the BankAccount class from the previous unit into the current project.

Sometimes Banks expect you to keep a certain amount of money in your account, and if you drop below it you are penalized.

Add a double constant named MINIMUM_BALANCE and assign a value of 1000 to it.

Modify the withdraw method so that it first performs the subtraction, and then calls the penalty method if the balance has dropped below the MIMIMUM_BALANCE.  Yes, use the instance field.

 

Create a runner that constructs a BankAccount object with your name and a 1500 balance.  First print out the BankAccount and then withdraw 600 and print out the BankAccount again.

If it works properly, there should be 880 left.

 

Car

Copy the Car class from the previous unit into the current project.

At the moment, the drive method can blindly "drive" a distance that will cause the gas tank to be negative.  Modify it so that if the parameter is too large, the car will only go as many miles as the gas tank will allow. 

 

Create a Runner that will construct a Car of your choice with 25 mpg.  Then add 5 gallons to the tank, drive 100 miles, and print out the Car.  After that, drive 100 more miles.

If done properly it should first print out a car with 100 miles and 1 gallon remaining, and then 125 mile and 0 gallons remaining.
