Timer Driven Objects

In the sacco jar there is a class named SaccoTimer, which can be used to repeatedly call a method. Copy the following class into your project:

import sacco.*;
public class Timer extends SaccoTimer
{
    public Timer()
    {
        this.setDelay(1000);
    }

    @Override
    public void onTimerTick()
    {
        System.out.println("Hello World!!!");
    }

    
}

Don’t stress too much about how that works yet (next unit). The important thing to know is that the code in the onTimerTick method is called repeatedly with a one second delay each time. Think of it as an infinite while loop with a delay. When the main method of the Timer class is run, "Hello World" will repeatedly be called

Use the following main method to verify that it works.

public static void main()
    { 
        Timer t = new Timer();
        t.launch();
    }


The Cat Class

Consider the simpleCat class below, which has a speak method that, when repetitively called, will say “mew” five times and then start saying “MEOW”.

public class Cat
{
   private int count = 0;
   private String sound = "mew";

   public void adjustSound()
   {
	count++;
      if(count>=5)
         sound = "MEOW!";
      
   }

   public void speak()
   {
      System.out.println(sound);
      this.adjustSound();
   }
}

In order to simulate a lonely cat, we’re going to create a timer class that will have a Cat as its instance field and will call the speak method every time that the timer ticks.

import sacco.*;
public class CatTimer extends SaccoTimer
{
    private Cat myCat;

    public CatTimer(Cat tmpCat)
    {
        this.setDelay(1000);
        myCat = tmpCat;
    }

    @Override
    public void onTimerTick()
    {
        myCat.speak();
    }
}

Now that we have two objects that are made to work together, we’ll need a main method to get things started.

public static void main()
{
    Cat kat = new Cat();
    CatTimer timer = new CatTimer(kat);
    timer.launch();
}

Run this method and make sure that the cat is meowing on the terminal.
What's happening is that the CatTimer depends on a Cat object. Every time the timer 'ticks', the cat is told to speak, which is output to the termi.nal



The Dog Class

Now consider the Dog class below: 

public class Dog
{
    private String sound = "Howl";
    
    public void lengthenHowl()
    {
         sound = sound.substring(0,sound.length()-1)+"wl";
    }
    
    public void speak()
    {
       System.out.println(sound);
       this.lengthenHowl();
    }
}

Create a class named DogTimer that is exactly the same as the CatTimer class except modified to work with a Dog. Use the following method to test it out:

public static void main()
{
   Dog d = new Dog();
   DogTimer t = new DogTimer(d);
   t.launch();
}

You should get increasingly long howls

public class Parrot
{

    public void flipSound()

        bool = !bool;
