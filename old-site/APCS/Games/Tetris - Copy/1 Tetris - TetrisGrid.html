TetrisGrid

So far we've created the Location class so that we can pass row/column data around easily and now we can start looking at how to represent the game. We have to think differently about the two different ways that the colored blocks are represented in the game.

1) The TetrisPiece:  Four Blocks that are movable and rotatable.
2) The TetrisGrid:  All of the stationary Blocks left over in the grid. Here we will need to be able to

Create a class named TetrisGrid.  The TetrisGrid class will represent the grid as a whole. At its core will be a 2D Color array with each Location corresponding to where it appears on the screen.

The Constructor for TetrisGrid should take in two ints. The number of row and columns that the TetrisGrid should have. It should then use those values to construct a Color[][] of the matching size into the instance field.

Add methods numRows and numCols to return the dimensions of the grid.

More Methods

The TetrisGrid will have a 2D array as an instance field, but since it's a private instance field it cannot be directly accessed externally.  In order to allow this object to be controlled, some methods need to be added to allow the TetrisGrid object to be modified.

public Color get(Location loc): returns the Color. value at the given Location.  You will have to extract the row and column out of the parameter so you can access that part of the array.

public void set(Location loc, Color c): This method puts the given Color in the array at the given Location.

public void randomBlock(Location loc): This method will construct a random Color and add it to the grid at the parameter location.

public void clear(Location loc): This method will 'clear' the given location by assigning a null value to that spot in the array.

public void clearAll(): This method will 'clear' the entire array. While this may be done using a fancy loop, constructing a brand new Color[][] with the same dimensions as the original.

public boolean isValid(Location loc): This method is to help make sure that we don't crash the program by going out of bounds.  Extract the row and columns from the parameter and return false if the row or column is either too large or to small.  Returning a true value should only happen if accessing both the row and the column will not cause an indexOutOfBounds exception.

public ArrayList<Location> getOccupiedBlocks(): This method creates and returns an ArrayList of all the occupied locations that are in the 2D array.  You will have to loop through every location in the grid and add a new Location object to your ArrayList if that location is occupied. You can tell if a location is occupied or not by checking to see if the value there is null or not.

Override the toString method to return a String that contains "TetrisGrid:" and is followed by the toString of every occupied Location in the grid. The easiest way to do this is to rely on the getOccupiedLocations method that you just wrote.

The quickLoad method:

Getting a block into the grid takes a decent amount of code.  You have to construct the Blocks, construct the Location, and call the addBlock method.  This isn't too bad for one Blocks, but for the future it would be nice if we could add many Blocks at a time for testing purposes.

Add a method with the following header:

public void quickLoad(int[][] locs)

The purpose of this method is to accept a 2D array of ints with exactly two columns.  Each row of this array represents a different location with it's two ints.  This method should loop through each row, extract the values at index 0 and 1 and add a random block at that location.  This way we can call quickLoad(new int[][]{{1,2},{3,4},{5,6}}; and it will quickly add three blocks to the grid for us.

 

Test it with the following runner:
