Snake Movement

In order to program the Snake to move, we are first going to set up a system so that we can refer to each of the four directions. In the Snake class, add four public static final ints named LEFT, UP, RIGHT, DOWN. You may choose any value for them, as long as they are different. A common use is to assign the values of 37,38,39, and 40, which are the corresponding key codes for the arrow keys. Since these variables are public and static, we can refer to them from anywhere in the program by saying Snake.UP or Snake.DOWN.

Go to the Square class and add the following method:

public Square nextSquareInDir(int dir)

This method's purpose is to look at the 'dir' parameter and check if it matches one of the four direction variables. If it does, the method's job is to construct and return the next Square in that direction (change the row or column by one). Don't forget that a larger row value is DOWN the screen. If the parameter doesn't match, simply construct and return a Square with the current row and column.

Back in the Snake class, add a method to the snake class with the following header:

public void move(int dir)

The purpose of this method is to 'move' the Snake, but the way that a Snake moves is by putting another piece on its front. Since we are following the rule the the front piece goes at index 0 of the ArrayList, access the first square in the list and use the nextSquareInDir method to get a new square. Put that square at index 0 of the ArrayList.

Lastly, go to the SnakeWindow's onTimerTick method. At this point, all you need to do is call your snake's move method. For now, you may temporarily pass in whichever direction int you want. Run your program to test it out. The Snake should move in the direction you gave and off the screen. Try a few more directions to make sure they work.

Controlling the Snake

The snake is to be controlled by using the four arrow keys, but we don't want to set the direction as soon as the key is pressed. Instead, the SnakeWindow class is going to keep track of the last direction pressed until it's time to move the snake. To accomplish this, add an int to the SnakeWindow to represent the last direction pressed. Assign 'up' to it in order to start the snake in the upward direction.

Back in the onTimerTick method, change your call to the move method so that it takes its direction from your new instance field

Finally, in the onKeyPress method your job is to look at the key pressed and assign it to the instance field only if it is one of the four directions.

Run your program one more time. This time you should be able to control where the snake is heading. Test out all four arrows to make sure that they work the way you'd expect.

Limiting the Snake

At this moment the snake grows forever, but it should not. In reality, we want the snake to be limited to the size variable that we made when we first made the snake. Go to the Snake's move method. After you've added the new head piece, compare the size of the list with the size variable. If it's too large, remove the extra squares from the back end, not the front.

Now test out your program again. This time you should have control over the snake and once it hits the proper length it will stop growing

Preventing Overlap

Right now the Snake will go in any of the four directions. This can cause a problem if its told to move in the opposite direction as its heading. We need to prevent the Snake from moving in the direction opposite to its current direction.

Add an int as an instance field in the Snake class that will represent the 'current' direction and assign UP to it. Now go down to the move method, which is currently given a direction that may be the opposite of our current direction. If it is, use move in the 'current direction' instead of using the parameter's value. You must also make sure that by the end of the move method, whichever direction you went in is assigned to the 'current direction' instance field so that it may be looked at next time.
