ChatBot

The ChatBot is an object computer that will respond to certain statements differently. Before we jump into how it works, let’s look at the runner. It constructs a ChatBot object, gets its greeting, and then starts a loop that will repeatedly print out the ChatBot’s response until the user enters “bye”. Read the code below and make sure you understand what each line of code is for.

public static void main()
{
   ChatBot chatter= new ChatBot();
   
   System.out.println (chatter.getGreeting());
   java.util.Scanner in = new java.util.Scanner (System.in);
   String statement = in.nextLine();
   
   while (!statement.equals("bye"))
   {
      System.out.println (chatter.getResponse(statement));
      statement = in.nextLine();
   }
}

Copy the ChatBot class below into your project

public class ChatBot
{
   /**
   * Get a default greeting 	
   * @return a greeting
   */
   public String getGreeting()
   {
      return "Hello, let's talk.";
   }
   
   /**
   * Gives a response to a user statement
   * 
   * @param statement
   *    the user statement
   * @return a response based on the rules given
   */
   public String getResponse(String statement)
   {
      String response = "";
      if (statement.indexOf("no") >= 0)
      {
         response = "Why so negative?";
      }
      else if (statement.indexOf("mother") >= 0
            || statement.indexOf("father") >= 0
            || statement.indexOf("sister") >= 0
            || statement.indexOf("brother") >= 0)
      {
			response = "Tell me more about your family.";
      }
      else
      {
         response = getRandomResponse();
      }
      return response;
   }
   
   /**
   * Pick a default response to use if nothing else fits.
   * @return a non-committal string
   */
   private String getRandomResponse()
   {
      final int NUMBER_OF_RESPONSES = 4;
      double r = Math.random();
      int whichResponse = (int)(r * NUMBER_OF_RESPONSES);
      String response = "";
		
      if (whichResponse == 0)
      {
         response = "Interesting, tell me more.";
      }
      else if (whichResponse == 1)
      {
      response = "Hmmm.";
      }
      else if (whichResponse == 2)
      {
         response = "Do you really think so?";
      }
      else if (whichResponse == 3)
      {
         response = "You don't say.";
      }

      return response;
   }
}

String has more than one indexOf method

So far, the indexOf that we’ve used only takes in a single String. It either returns the location of the first letter of the String, but it returns a -1 if the String is not found. There is a second index of that is a little bit different. It has the following header:

public int indexOf(String str, int startIndex)

This version of indexOf will only look at locations at startIndex or above. If it’s not within those boundaries, a -1 is returned.

 

Better Keyword Detection

At the moment we have an issue. If your statement is “I know that this statement is positive” will find the word “no” in it. This means that indexOf is not the only thing that we need in order to detect keywords. We are going to have to write a better version of it that also makes sure that there are no letters before or after the keyword.

What makes a the keyword 'proper'?

  • It is found within the String
  • There is either no character before it, or the character before it is not a letter (use a compareTo range to determine this)
  • There is either no character after it, or the character after it is not a letter

This means that the characters before and after the keyword are not letters. Be careful: blindly grabbing the character before and after the word could cause out-of-bounds problems. Make sure that you are careful that you don't actually perform the substring if that's the case.

Overloading findKeyword

The findKeyword method currently takes in a statement, a goal, and a starting index. Add the following method header:

public int findKeyword(String statement, String goal)

This version of the method only takes in two Strings, the statement and the goal, but no index. The idea is that this method will return the first index of the proper keyword from the beginning of the String, meaning no index is required. Use exactly one line to implement this version of findKeyword.

Once you finish this, modify the getResponse method so that it no longer calls indexOf. It should call findKeyword instead. Note: This new method is not in the String class, so simply pasting findKeyword over idexOf will not compile.

You and I

Add one final method to the ChatBot class

public String youMeStatement(String statement)

The purpose of this method is for the situation where the word 'you' is followed by the word 'me', such as "you amaze me" or "I have to say, you disgust me". This method should find what's between the two methods and rearrange it to say "Why do I _______ you?".
Implement the youMeStatement method. At this moment you can assume that the words "you" and "me" are in the String and in that order.
