TextFormatter

The TextFormatter class was released as a possible AP test question after doing the ChatBot assignment. Though a for loop could be appropriate for traversing, try to use a while loop in a similar way to how the ChatBot class did it.

When Web developers want to convert plain text documents to HTML, they need to make changes to the text to insert tags and replace certain characters. For this question, you will write part of the class to convert text to HTML. You will write methods to replace underscores ("_") with tags indicating the text should be in italics.

Copy the following class to your project:

  1. Write the method findString. This method will take a string of length 1 to find in the line, at a given starting point. It will return the location of the goal string. This differs from the indexOf method of the String class because it requires the string be just a single instance of the string, so if the string appears two or more times consecutively, it will not return any of those values. If there is no single instance, the method should return -1.

    Consider the following examples, where line has the value "aabaccb".

    • findString("a",0) should return a 3, since the first two a's are duplicates
    • findString("b",4) should return a 6 because the first time that "b" occurs at or after position 4 is in postion 6.
    • findString("c",0) should return a -1 because there is no single instance of "c" in the line

  2. Write the method countStrings. This method will take a string of length 1 to find in the line. It will return the number of times single instances of the goal string appear in the string.

  3. Write the method convertItalics. If the line has an even number of single underscores, a line with those underscores converted to alternating <I> and </I> tags should be returned. The first underscore will be converted to <I>, the second to </I>, the third to <I>, and so on. If the line does not have an even number of <I> tags, the original line should be returned. Consider the following examples.
    • "This is _very_ good." should return "This is <I>very<I> good."
    • "_This_ is _very_ _good_." should return "<I>This</I> is <I>very</I> <I>good</I>."
    • "This is _very good." should return "This is _very good."
    • "This is __very good." should return "This is __very good."
