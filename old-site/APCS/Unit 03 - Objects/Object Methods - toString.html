The toString Method

So far we have been working on the Name class.  We gave it instance fields, added a constructor, and added some accessor methods.  We have been writing some simple Runner classes that have been constructing the objects and then printing out their crucial details.

 

What happens when we System.out.println an object variable directly?

Right now NameRunner looks like the following:

public class NameRunner
{
    public static void main()
    {
        Name n = new Name("Marty","McFly");
        
        System.out.println("First Name: "+n.getFirst());
        System.out.println("Last Name: "+n.getLast());
        System.out.println("Initials: "+n.getInitials());
      
  
        Name n2 = new Name("Jon","Snow");
                
        System.out.println("First Name: "+n2.getFirst());
        System.out.println("Last Name: "+n2.getLast());
        System.out.println("Initials: "+n2.getInitials());

    }
}

We are accessing every individual instance field and printing them out individually.  Change the runner to look like the following:

public class NameRunner
{
    public static void main()
    {
        Name n = new Name("Marty","McFly");
        
        System.out.println("Name: "+ n);
        System.out.println("Initials: "+n.getInitials());
      
  
        Name n2 = new Name("Jon","Snow");
                
        System.out.println("Name: "+n2);
        System.out.println("Initials: "+n2.getInitials());

    }
}

Run this method.  It should give you an output similar to the following:

Name: Name@145f572
Initials: M.M.
Name: Name@1f530d8
Initials: J.S.

The part highlighted in red is what happens when you put an object into a println.  When given to a println, an object is supposed to be represented in text, but complex objects could have 30 different variables and Java doesn't know what to do with them.  So basically it will print out the name of the class, an '@' character, and a seemingly random number (think of it like an object's serial number).

 

How to determine how an object represents itself in text:

Name@145f572 is not how I want my name object to be represented when I try to print it.  As the designer of the object, I get to choose the text that represents the object by writing a very specific method.

Add the following method header:

public String toString()

This is the exact method header that you need to add to any object if you want to change how it represents itself.  When it's time to concatenate together an object with another String, this method will secretly be called.

This means that the line:

System.out.println("Name: "+n);

Secretly reads

System.out.println("Name: "+n.toString());

This is automatic, you will rarely have to call the toString method directly.  It is called automatically at concatenation time.

 

 

Implementing the toString method:

For the Name object it's pretty straight forward.  The way that a Name object should represent itself is with a simple String that contains the first and last name.  Make the toString method look like the following:

public String toString()
{
  return first+" "+last;
}

 

Now run the main method again.  Instead of Name@145f572, it prints out "Marty McFly" or "Jon Snow".

Name: Marty McFly
Initials: M.M.
Name: Jon Snow
Initials: J.S.

You must be exact!

Java is looking for a specific method with a specific method header.  This means you must have the same return type, name (capitalization counts), and parameters (none).  If you change any of those slightly, it will revert to the old Name@145f572 style of String.
