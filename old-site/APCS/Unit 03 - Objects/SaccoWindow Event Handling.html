Event-Based Programming with SaccoWindow

In the last assignmnt you created the HelloWindow and the corresponding WindowRunner.  It didn't do much, but it displayed Hello World in a fancy way.

 

Event Handling:

So far our programs have been pretty straight forward in the order in which they run.  The user doesn't really have any say in it.  Our programs sometimes have user interaction, but it's very prescribed.  It has to come at pre-determined moments in the code.

There is another way to program user interaction and it's much more reactionary.  Leave the rest of the HelloWindow as it is and add the following method:

@Override
public void onMousePress(MouseEvent m)
{
   System.out.println("CLICK!");
}

 

What is this method?

Similar to the paintWindow method, the onMousePresss method is a method that is already built into the SaccoWindow.  It currently does nothing.  If we want to change how it functions in the HelloWindow, we must override it and add the code that we want to replace it with.  In this case, it's a simple println that says "CLICK!".

 

When is this method called?

 You will never call this method directly.  This method is what is called an Event Handler.  When a user does something like click a mouse or press a key on the keyboard it's known as an event.  Events don't happen at prescribed times, so the way that we prepare for them is with an event handler like the onMousePress method.

Every single time you press a mouse button on the window the onMousePress button is automatically called.  Test this out by running the program and clicking several times on the window.  Each time that you click the window "CLICK!" should be printed as a reaction to the Mouse Event.

 

 

What's this MouseEvent parameter?

The MouseEvent parameter is an object that holds three pieces of data as instance fields: the x and y of where the mouse was on the window when the click occurred and the number that represents which button was pressed.

During this method we can access that information using the getX, getY, and getButton methods that you can see in the MouseEvent API.

Change the HelloWindow onMousePress method to be the following:

@Override
public void onMousePress(MouseEvent m)
{
   int x = m.getX();
   int y = m.getY();
   int button = m.getButton();
   System.out.println("CLICK! ("+x+","+y+") with button #"+button);
}

Run the program.  This time you should see a different output depending on the location clicked.  Try different buttons on the mouse and see what each number stands for.

 

Changing Colors:

Let's make one last modification to the HelloWindow.  When we click the window, let's have the background change from blue to red.

What needs to change?

For this program we are needing to switch from a blue color to a red color.  This means that we will have to use a variable to represent the background color.  Since a Canvas can use a String to represent a color, we'll stick with that for simplicity.

- Add a String as an instance field named colorStr.  This String will represent the color of the background.

 - Now that we have an instance field, add a constructor that will take in no parameters and have it simply put "BLUE" into the instance field.

- Jump down to the paintWindow method and find the line where you set the color to "BLUE" and replace it with the colorStr instance field.

 

All that we've done is replace the String "BLUE" with a String variable that contains the String "BLUE".  Since that variable never changes, this should not change your program in any way.  Run it one more time to make sure that nothing has changed before moving on.

 

Now replace the onMousePress method with the following:

@Override
public void onMousePress(MouseEvent m)
{
   colorStr = "RED";
}

This means that everytime the mouse is pressed, the colorStr is set to "RED".  The effect of this is that the paintWindow method relies on this variable to tell it what color to be.  Now that it says "RED" the color will change to red.

 

Run the program and click the mouse on the window.  It should change from blue to red.  It will not change back because we've only programmed the ability to set the variable to "RED", not back to "BLUE".

 

onMouseRelease

There is more than one MouseEvent method that you can override.  This one is called onMouseRelease, and the computer will wait until you let go of the mouse before calling this method (unlike the onMousePress method that is called when the mouse button is pressed down..

Add the following method to the HelloWindow class.  This is a new method, not a replacement for the other mouse method.

@Override
public void onMouseRelease(MouseEvent m)
{
   colorStr = "BLUE";
}

This one sets the color back to blue.

 

Recap:  The colorStr variable starts out with a "BLUE" value in it.  When the mouse is pressed, the colorStr is set to "RED".  This only lasts until the mouse button is released, because at that moment the onMouseRelease method will set it back to blue.

 

Run the program and click the window a few times.  It should remain red for as long as the mouse is held down.
