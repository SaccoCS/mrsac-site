Objects: Instance Fields and Constructors

 Objects are a very important concept in programming.  It is important that we both understand how to use objects and also how to design them. 

 What are Objects?:  Objects can be used in many ways, but it's best to think of objects as a data type that bundles together multiple pieces of data.

We have already been using a type of object in our programs.  A String is much more complicated than an int or a double.  It is a piece of data that bundles together series of characters.

 

 

Designing An Object Example:  Name

The first object that we are going to design is a simple object that is designed to bundle together a person's first and last name.  First we must think of the instance fields, or data that the object will contain.  In this case it's two Strings.

In your project, create a class named Name, and put the following code in it:

public class Name
{
    public String first;
    public String last;
}

 

What's going on here?

You must not think of this as your standard "main method" class.  This class is meant to be a blueprint for an object called Name.  So far, this object has two pieces of information that it stores ( known as instance fields);  A String for the first named and a String for the last name.

 

How do we use objects?

An object class doesn't work on its own.  It is simply a design for a piece of data.  In order to use it, we must have a program with a main method that will run it.  This program is known as a Client class, a Tester class, or a Runner class and its purpose is to use the object as data.

An arrow will be drawn, because the NameRunner class makes use of the Name data type.

 

In the same project as the Name class, put the following class which constructs a Name object and then prints out the values stored in its first and last variables.

public class NameRunner
{
    public static void main()
    {
        Name n = new Name();
        
        System.out.println("First Name: "+n.first);
        System.out.println("Last Name: "+n.last);
    }
}

 

Constructing:

When an object is first created,  it must first be constructed.  In the main method, the following line constructs the Name object and stores it in a variable named n:

Name n = new Name();

The left side of the '=' is Declaring a variable n be a Name variable.  This is exactly the name of the object class that we first wrote.

The right side of the '=' is how you construct an object.  The word new is a special word in java that indicates an object is being constructed.  After new is the constructor, which has the same name as the object.

If the object were in a class named Bird, the construction statement might look like:

Bird myBird = new Bird();

Notice that the word Bird appears twice, in the same places that Name did in the previous example.  This will often be the case.

 

Accessing the two variables:

Name was designed so that every name variable had a first and last name.  In the second part of the main method, the contents of n's first and last are printed out.  

       System.out.println("First Name: "+n.first);
       System.out.println("Last Name: "+n.last);

Since the design of the Name object never puts anything in either the first or last, the value is null.

First Name: null
Last Name: null

 

public class NameRunner
{
    public static void main()
    {
        Name n = new Name();
        
        System.out.println("First Name: "+n.first);
        System.out.println("Last Name: "+n.last);
        
        Name n2 = new Name();
        
        n2.first = "Jon";
        n2.last = "Snow";
                
        System.out.println("First Name: "+n2.first);

    }

This time a second Name variable is made, but the first and last variables are set to "Jon" and "Snow".   Running this again will print out the two null values stored in the n variable as well as the "Jon" and "Snow" variables stored in the n2 variable.

This time the output should be:
First Name: null
Last Name: null

First Name: Jon
Last Name: Snow

Constructors

With our Name object, it's not really complete until you have assigned values to its first and last variables. 

Name n = new Name();        
n.first = "Martin";
n.last = "McFly";

If you think about it, there isn't really a situation where you would create a name an not immediately assign the first and last variables.  In order to get this done, we are going to add a constructor to the Name class.  A Constructor is a special method that is meant to make assigning the initial values to the instance fields easier.

 

Our Goal:  To add a constructor to the Name class so that instead of the three lines above, the object could be constructed with one line, like the following:

Name n = new Name("Martin", "McFly");

Notice that this time, the values of first and last are passed in as parameters during construction.  This ability needs to be added to the Name class.

 

Designing a Constructor:

A Constructor's job is purely to help set the instance fields.  In the case of the Name class, the constructor needs to take in two parameters; one for the first name and one for the last.  Here is the new version of the Name class with a constructor added. 

public class Name
{
  public String first;
  public String last;
  
  public Name(String f, String l)
  {
    first = f;
    last = l;
  }
}

1)  Naming the Constructor:  When naming the constructor, it must be named exactly the same as the class.  We will always make our constructors public.

2) Creating Parameter Variables: This is where the programmer decides what information is required by the object upon creation.  In our case the first and last name are supposed to be given.  By putting String f, and String l, we are providing locations for the first and last name to be passed in when the constructor is called.  Over in the Runner program, we will construct using the line Name n = new Name("Martin", "McFly");.  "Martin" will be passed into the f parameter, while "McFly" will be passed into the l parameter.

3) Assigning values to the instance fields:  The constructor's job is to assign values to the instance fields, in this case the first and last variables.  Since the two values that we care about are already given to us as parameters, the only thing that this constructor needs to do is assign the parameters to the instance fields.  When the line Name n = new Name("Martin", "McFly"); is called, "Martin" is passed into the f parameter.  Since the end goal is to have "Martin" in the first instance field, first = f; is a required line.  The last name is handled siilarly.

 

Using the Constructor:

Now that the Name object has this cool constructor it is unnecessary to set all the instance fields one at a time.  Instead we can use the new constructor that we just created.

public class NameRunner
{
    public static void main()
    {
        Name n = new Name("Marty","McFly");
        
        System.out.println("First Name: "+n.first);
        System.out.println("Last Name: "+n.last);
        
        Name n2 = new Name("Jon","Snow");
        
        
        System.out.println("First Name: "+n2.first);
        System.out.println("Last Name: "+n2.last);
    }
}
