Objects with Methods: Non-Basic Accessors

The big three ideas that we have to think about when dealing with objects are instance fields, constructors, and methods.  Look at what the Name class was the last time that we left it. 

public class Name
{
    private String first;
    private String last;
    
    public Name(String f, String l)
    {
        first = f;
        last = l;
    }
    
    public String getFirst()
    {
       return first;
    }
    
    public String getLast()
    {
    	return last;
    }
    
}

 

Remember the purpose of the getFirst and getLast method.

As a result of making all of our instance fields private, we had made variables that would bundle data, but had no way of getting the information back out.

Previously we could construct a Name, and we could access the variable directly.

Name n = new Name("Steve","Urkle");
System.out.println(n.first);

But now that first is private, accessing it directly is no longer possible.  The getFirst and getLast methods are there so that the client code can get the information back out:

Name n = new Name("Steve","Urkle");
System.out.println( n.getFirst() );

 

More Methods Make More Useful Objects

So far the methods that we've seen simply let us access the instance fields directly.  This means that n.getFirst() will return the String that's in n's first variable.  But methods don't have to be so direct.  For example, the methods that we've used for String, like substring or indexOf, return useful to anybody using a String object.  When an object has useful tools like these it is easier to use.

 

For Example:  Name so far has the ability to tell us its first and last name.  We will add the ability for a Name object to tell us the initials of the Name stored in the instance fields.

Before we start writing the method, let's think of how it might be used:

Name n = new Name("John","Coffey");
String inits = n.getInitials();
System.out.println("Initials: " + inits);

If the getInitials method works properly, this should print out:
Initials: J.C.

 

Designing the method:

Method Header:
When trying to design the getInitials method, the first thing that you think of is the return types, and the parameters.  

String inits = n.getInitials();

Parameters:  The parameters are information that the method might need outside of the instance fields.  For the getInitials method to function it only needs the first and last name, which are already stored in the instance fields.  For this reason, this method will have no parameters.

Return Type:  When trying to determine the return type, think hard about what type you would expect from the method (int, double, String).  In this case I would expect back the initials as a String ("J.C.", "A.S.", "D.T.") and never an int/double. 

Now that we know that the return type is a String and that there should be no parameters we are ready for the method header:

public String getInitials()
{

 

Implementing the Method:

Now that we know the header, it's time to implement the code that makes the program function.  When writing an object-method, you must remember that you have access to the instance fields.  The purpose of this method is to generate and return a String containing the initials of the name.  This means that we will have to access our instance fields.

public String getInitials()
{
   String initOne = first.substring(0,1);
   String initTwo = last.substring(0,1);
   return initOne + "." + initTwo + ".";
}

 

Updating the Runner:

Now our Name objects can tell us their first name, last name, and initials.  We simply need to update our runner to test out our most recent method

public class NameRunner
{
    public static void main()
    {
        Name n = new Name("Marty","McFly");
        
        System.out.println("First Name: "+n.getFirst());
        System.out.println("Last Name: "+n.getLast());
        System.out.println("Initials: "+n.getInitials());
      
  
        Name n2 = new Name("Jon","Snow");
                
        System.out.println("First Name: "+n2.getFirst());
        System.out.println("Last Name: "+n2.getLast());
        System.out.println("Initials: "+n2.getInitials());

    }
}

 

As you can see, our Name object has become more than just a location for two Strings.  By adding methods that are useful tools, we can make them better and more object-y.
