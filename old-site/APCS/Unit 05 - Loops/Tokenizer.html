Tokenizer

Create a class named Tokenizer.   A Tokenizer is an object that will hold a String as an instance field and be able to return that String one word at a time until there are no more words left in the String.  At that point it will simply return the empty String "".

The constructor of Tokenizer should take in a String and apply it to the instance field.

 

The way that we are going to keep track of what the next token should be is to have an int that will represent the index that the next token starts from. Each time a token is taken, the int will be changed to the location of the next

A Tokenizer should have the following methods:

public String getToken()

This method is both an accessor and a mutator.  It's purpose is to return the first word in the String, but also to remove the first word from the instance field so that the next time the getToken method is called it returns a different result.  

Warning:  Spaces are useful, but make sure that your code is ready for the situation when there is no space.  It can cause problems if you're not prepared for it.  If the last word is returned, the instance field should be set to the empty String "".

For example:  Consider a Tokenizer that is constructed with "Hey Mister Sacco" and then the getToken method is called four times.

Starting instance field Returned from getToken Instance field after getToken is called
"Hello Mister Sacco" "Hello" "Mister Sacco"
"Mister Sacco" "Mister" "Sacco"
"Sacco" "Sacco" ""
"" "" ""

Don't forget that you cannot do anything after you say "return ____".  For this accessor/mutator method I suggest putting the value that you would like to return into a variable instead of returning it right away.  This will allow you to make modifications to the instance field before you return.

 

 

public boolean hasMoreTokens()

This method is an accessor that will return true if there are one or more words remaining.  Some might think that it it has to do with the index of the space, but it does not.

 

 

Testing:

Use BlueJ to test out your Tokenizer.  Construct a Tokenizer with "Computer Science is awesome" and then inspect that object.  After that, alternate calling the hasMoreTokens() and getToken() method, making sure that they work properly without crashing.  In the end, you should be able to keep calling the getToken and hasMoreTokens methods and getting false and "" as a result.
