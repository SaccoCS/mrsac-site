do - while

Note:  do-while loops are untested.  If you would like to forget them after this assignment, you may.

 

The while loop is useful in many situations, but sometimes causes some awkward coding.  Consider the following code, which is meant to loop until the user enters "exit".

public static void main()
{
    Scanner scan = new Scanner(System.in);
    System.out.println("Type EXIT to stop looping");
    String input = scan.nextLine();

    while(!input.equals("EXIT"))
    {
        System.out.println("Type EXIT to stop looping");
        input = scan.nextLine();
    }
    
    System.out.println("Terminating...");
}

The entire loop is controlled by the input variable.

 

Repeated Code:

If you look at the code above, you'll notice that there is some repeated code.  The println and the scan.nextLine are essentially doing the same thing.  The reason that we're repeating code is that there should be a value stored into input before the while loop starts.  This is because the first thing that the while loop does is to check if the input is "EXIT".

 

Solution:  The do-while loop

The while loop has a cousin named do-while.  The structure is as follows

do
{
    <some code>

}while( <boolean condition> );

 

This is very similar to the standard while loop, except for one thing:  The body of the loop runs one time before the boolean condition is checked.  This can solve the problem that we ran into above.

public static void main()
{
    Scanner scan = new Scanner(System.in);
    String input;

    do
    {
        System.out.println("Type EXIT to stop looping");
        input = scan.nextLine();

    }while(!input.equals("EXIT"));
    
    System.out.println("Terminating...");
}

Now that we're using a do-while we don't have to worry about setting up the input variable.  The loop will run one time before checking the input variable, meaning that the first value will be assigned during the loop.

 

Some Warnings about do-While:

1)  There is a semi-colon after the boolean condition.  This is easily forgotten, as it is different than a standard while loop.

2) The control variable must be declared outside of the loop.  Notice the line String input;  This is a necessary line because it declares the input variable to exist outside the body of the loop.  If it was declared inside the body it would not exist when the while loop tries to check !input.equals("EXIT");

 

 

Assignment:

Take a while-loop assignment that you've done so far and convert it to use a do-while loop instead.  Not all loops are prime candidates for this.  The do-while is good in situations where you had to duplicate some of your code so that you could start the while loop with good values.
