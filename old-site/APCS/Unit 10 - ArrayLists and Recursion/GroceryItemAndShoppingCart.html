Copy the ShoppingCart class below. Its core instance field is a List of GroceryItems that might be ready for checkout. The constructor calls a static method built in to the GroceryItem class so that it generates a random list of various GroceryItems for the purpose of testing. 

import java.util.*;
import sacco.objects.GroceryItem;
public class ShoppingCart
{
   private ArrayList<GroceryItem> items;

   public ShoppingCart()
   {  items = GroceryItem.listFactory(15); }

   public ArrayList<GroceryItem> getItems()
    {
        return items;
    }

   public double getTotal()
   {   /* To be implemented by you  */ }

   public ArrayList<String> getCategories()
   {   /* To be implemented by you  */ }

   public void groupByCategory()  
   {   /* To be implemented by you  */ }

   public String toString()
   {
       String str= "";
       for(GroceryItem g: items)
       {
            str += g+"\n";
       }
       return str.substring(0,str.length()-1);
    }

    public static void main()
    {
        ShoppingCart cart = new ShoppingCart();
        System.out.println("Categories: "+cart.getCategories());
        cart.groupByCategory();
        System.out.println("\nGrouped List:\n"+cart);
    }
}

 

Methods:

Implement each of the following methods. After each one runs, the instance fields should remain unchanged. The main method will create a ShoppingCart object, print out it's categories, call the groupByCategory method and print out the toString. Use this to test and make sure that your code is functioning.

 

public double getTotal()

Calculate and return the sum of cost of all the items in the cart

 

public ArrayList<String> getCategories()

There are several different possibilities for categories of the GroceryItems, but not all of them might be in this ShoppingCart’s list. Implement the getCategories method, which should return an ArrayList<String> with each of the Categories found in the List.

The returned ArrayList should contain no duplicate Strings. There are several ways of doing it, but most will end up using a loop with a boolean flag to determine if the item is already in the list before they add.

For Example: If the list contains the following items:

Pork | $13.12 | Meat
Toothpaste | $3.11 | Household
Chicken | $7.15 | Meat
Asparagus | $6.24 | Produce
Carrots | $4.91 | Produce

The returned ArrayList would contain the values:
[“Meat”, “Household”, “Produce”]
The order of the categories is not important

 

public void groupByCategory()

When the ArrayList<GroceryItem> is generated, the elements in it are shuffled. After this method is called, the items variable should contain an ArrayList<GroceryItem> with all of the same items, but grouped by their category (all the Dairy together, all the Meat together, etc).

For Example: If the list contains the following items before the groupByCategory method is called:

Pork | $13.12 | Meat
Toothpaste | $3.11 | Household
Chicken | $7.15 | Meat
Asparagus | $6.24 | Produce
Carrots | $4.91 | Produce

After it's called, the elements would be in the following order:

Pork | $13.12 | Meat
Chicken | $7.15 | Meat
Toothpaste | $3.11 | Household
Asparagus | $6.24 | Produce
Carrots | $4.91 | Produce

Note: The order of the categories isn’t important. It’s only important that they’re grouped
