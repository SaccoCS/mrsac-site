Selection Sort with String Arrays

 

When dealing with large amounts of data, the ability to sort things becomes important.  There are many common strategies that are used, but today we’re going to use the Selection Sort strategy.  The general strategy is as follows:

1)  Start with the first index as your ‘focus index’
2)  Find the index of the lowest value in the array, but only the values at the focus index and after.
3)  Swap the lowest value with the value at the focus index.
4)  Increment the focus index.
5)  Repeat steps 2-4 until the focus index has been every index in the array (technically you don’t need to do the last index).

Important to know:  "Why don't we return a sorted array?"

The main method does four things:

1) Create array arr and populates it with 30 random letters
2) print out the values of arr so you can see how unsorted they are
3) selectionSort(arr)
4) print out the values of arr so you can see how sorted they are

How can arr be any different if we don't say arr = selectionSort(arr)?

Arrays are technically objects.  We've previously discussed that when an object is passed as a parameter the method doesn't just get a copy of the object, it actually gets a reference to the original.  When you make changes to the parameter array in the selectionSort method, it will actually affect the original array.  You can test this out by commenting the line that says selectionSort(arr) and running the main.  You'll notice then that the two arrays printed out are unchanged when the method is not called.

Remember:  Objects are passed by reference, meaning that modifying a parameter will change the original.  Primitives (int, double, boolean) are passed by value, meaning the parameter is simply a copy of the original's value.
