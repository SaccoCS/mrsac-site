Casting doubles to ints

Casting is an operation that will temporarly change a value from one type to another (providing they're compatible).

 

For example, assume we're writing a program that takes in from the user their batting average and how many at bats the batter had, and then outputs the number of hits that the batter had.

public static void main()
{
   double battingAve = ...from the user
   int totalAtBats = ...from the user

   double hits = battingAve*totalAtBats;
   System.out.println("You had " + hits + " hits");
}

The problem here is formatting.  If the user enters a batting average of .250 and batted 76 times, the program would output:

You had 19.0 hits

While this is true, the 19.0 is really bothersome.  The resulting number of hits is never going to need the decimal.  It really should be an int, but the following change will not help:

int hits = battingAve*totalAtBats;

The compiler won't allow this based on type matching.  The problem now is that the right side is a double, which cannot be stored as an int (it doesn't know what to do with everything after the decimal point).

The solution is to cast the value as an int after the calculation has been made:

int hits = (int)(battingAve*totalAtBats);

This time the battingAve*totalAtBats calculation happens first, giving a value like 19.0.  The resulting value is then cast into int so that is compatible with the hits variable.  The hits value would contain the int 19, which we wanted.

 

What does casting do with the decimal?

A double carries numbers with decimal points.  It is impossible to represent the exact same number as an int.  When casting a double to an int, the resulting int will be the truncated version of the double, meaning it simply takes the integer portion and discards everyting after the decimal.
