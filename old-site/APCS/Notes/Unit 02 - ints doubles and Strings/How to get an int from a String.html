Getting an int from a String

Often times when you recieve information from the user it's in the form of a String.  This isn't always the best option for you to work with.  Strings have no mathematical value and much of the time programming requires that you perform some sort of math calculation to get the job done.

Be careful. The + symbol has two different purposes. For example:

String num = "17";
num = num + 5;
System.out.println(num);

At first glance it may look like the output of this code would be 23, but that is incorrect. When the + symbol is between a String and anything, it concatenes the data into a new String. This means that the 5 is just added to the end, and the output is 175.

Converting to an int

Luckily, there is a class named Integer, which has one particular method named parseInt that is useful for getting ints from a String.  The purpose of the parseInt method is to accept a String as a parameter and return the int version of the contents of the String. Since it is a static method, we call it using Integer as the class name, parseInt as the method name, and give it a String as a parameter.

For example, consider the following snippet of code:

String numStr = "439";
int num = Integer.parseInt(numStr);

Even though "439" looks like an int, the fact that it's a String means that it has no mathematical value.  The second line of code passes numStr into the parseInt method, which will examine the "439" String and return the int version of it.

What's the difference?

Consider the code below, which has added two printlns to the previous snippet.  

String numStr = "439";
int num = Integer.parseInt(numStr);
System.out.println( numStr + 22 );
System.out.println( num + 22 );

The output of this code will be:

43922
461

The reason that the two outputs are different is because the first println has a String being added to an int.  In this case, the plus symbol means concatenate and it simply tacks the 22 onto the end of the String 439.  The second println contains two ints, the value stored in num and the 22 that's being added.  Adding two ints together works mathematically, and the printed value is the actual addition of the two values.

Getting from the User

We already know how to get a String from the user using JOptionPane

String value = JOptionPane.showInputDialog("Enter a number");

But a String is the ONLY thing that you can get, so in reality getting an int is a two-step process. One line to get the String version of the input, and another to convert that input to an int.

String value = JOptionPane.showInputDialog("Enter a number");
int num = Integer.parseInt(value);

Now that we have an int, we can perform the necessary math to accomplish our task.
