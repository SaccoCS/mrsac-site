Using Variables

During the HelloWorld assignment you had the following line:

System.out.println("Hello World!!!");

This line calls a method named println. The definition of println states that it will print out the String that is between the parentheses. Quotations surround a “String Literal”, which is the most straightforward way to represent a String. Anything that is between the quotes is considered text only, with the exception of quotes and slashes.

Creating and Using a Variable

Variables are used to store data, such as a String, for later use in a program. The following snippet of code prints out Hello World!!! just as your first program did:

String phrase = “Hello World!!!”;
System.out.println(phrase);

The difference is that this code first stores "Hello World!!!" to a variable named phrase and then it prints out

We need to take a close look at these lines carefully.


String phrase = “Hello World!!!”;

There are two specific things happening here:

  1. String phrase

    This portion is a declaration statement. It DECLARES that the word phrase will be a variable that holds a String for the rest of the program.

  2. = “Hello World!!!”;

    This uses the equals sign to ASSIGN the String “Hello World!!!” to the variable that was created. The equals sign is actually called the assignment operator, because its purpose is to assign values to variables.



System.out.println(phrase);

Previously we passed String Literals into the println method. We can also put a variable that holds a String. Since phrase is now a variable that holds the String “Hello World!!!”, passing it into the println method will cause “Hello World!!! will be printed.

It is important to note that there are no quotation marks in this println command. If you were to write System.out.println("phrase");, Java would consider it a String that says "phrase" and that's what it would print out to the terminal window.

Reusing Variables

Variables are created so that they can be replaced if they need to be. Consider the following code:

public class StringChanger
{
   public static void main()
   {
      String phrase = "Hello World!!!";
      System.out.println(phrase);

      phrase = "Sacco Rules!!!";
      System.out.println(phrase);
   }
}

If you ran the main method, the output would be:
Hello World!!!
Sacco Rules!!!

Even though the two printlns are identical, the contents of the phrase variable are changed to "Sacco Rules!!!" right before the second println

Concatenating Strings

You can use the ‘+’ sign to merge two different Strings into one String. This is called concatenation.

For example:
“Hello ” + “World!!!” makes the new String “Hello World!!!”


You may also use variables in concatenation:
String name = “Sacco”;
“My name is ”+name   makes the new String  “My name is Sacco”

Choosing a Variable Name

When using variables you get to decide what variable name to use. Often times students think that phrase is the only possible variable name, but you can choose (almost) any word that you want to be the variable name. You should aim to make them as descriptive as possible. I chose phrase when we were putting the generic "Hello World" into it, but name was a much better choice for storing "Sacco"

Get in this habit now. Don't be lazy with your variable names. You'll regret it once our programs start getting complicated.

Example Program

The following is a sample of a simple program that requires the use of a variable.

Write a program named Greeter. The main method of Greeter should create a String variable named name and assign your name to it. Then use it to print out a string in the form “Hello ! How are you?”

Solution:

public class Greeter
{
   public static void main()
   {
      String name = “Sacco”;
      System.out.println(“Hello, ”+ name +“!  How are you?”); 
   }
}

Note that the println can get confusing with all of the quotation marks. Pay attention to the color of the text. The text that is green will appear how you type it, but the variable's color is black, meaning that it will print the contents of the variable.

Important Side Note: Quotation Pains

The Greeter class above is proper code, but if you copy and paste it from your browser into a BlueJ window it will not compile. It will give you the error: illegal character \8220

This is a very frustrating error to come across because it has nothing to do with how correct your code is. The actual problem is that the quotation marks are too fancy. It turns out that there are multiple characters that represent quotation marks, but only one of them is recognized properly by Java. This error occurs occationally when code is being copied from a web browser. It can ruin your day.

The solution is to simply delete the quotation marks that are there and type new ones in their places.
