Using Methods from Other Classes
User Input

So far the programs that we've written are not useful in any way.  They simply create a variable, store a value in it, and then print it out.

public class Pointless
{
   public static void main()
   {
      String description = "unnecessary";
      System.out.println("Programming like this is " + description);
    }
}

While this is good for practicing declaring variables and assigning a value to it, the truth is that the variable description isn't needed.  This is because we already know the value that will be stored in it.  The program would gain a little pizazz (though not a ton) if we were to incorporate the user in some way.

 

Using other classes:

Java comes with a library of classes that can do all sorts of things that are useful.  With a little knowledge we can take advantage of them.

In order to call a static method from another class, you must know three things: the name of the class that the method is in, the name of the method, and are there any parameters. Once you know those, you may call the method using the following format:

ClassName.methodName(parameters);

In order to get input from the user, we're going to use another class named JOptionPane.  JOptionPane has many methods that deal with putting simple windows up on the screen.  One of the functions that JOptionPane has is a method named showInputDialog, which will pop up on the screen and wait for the user to hit the OK button.

Similar to the println method that we've been using , the showInputDialog method requires a String to function. The String that you give it will appear on the window that pops up.  In the image above, the String would have been "Hello World!!!". 

Now that we know the class name, the method name, and the parameters, we know what the method call is going to look like:

JOptionPane.showInputDialog("Hello World");

Since the showInputDialog method is in a class that's a part of the Java Library, we first have to import it.  This means we need an import statement before our class header.  For JOptionPane, the import is javax.swing.JOptionPane (you will never have to memorize any import statement for a test).

import javax.swing.JOptionPane;
public class Pointless
{
   public static void main()
   {
      String description = "unnecessary";
      System.out.println("Programming like this is " + description);
    }
}

Now that our program is aware of the JOptionPane class we can call the showInputDialog method, so the full method call would be JOptionPane.showInputDialog

 

import javax.swing.JOptionPane;
public class Pointless
{
   public static void main()
   {
      String description = JOptionPane.showInputDialog("Please Enter a Description");
      System.out.println("Programming like this is " + description);
    }
}

Notice that the method call is in the location where the String "unnecessary" is.  This is because the showInputDialog method is a method that returns a String.  In this case, the resulting String is assigned to the description variable so that it can be used later in the program.

 

Warning: Common Mistake:

If you want to keep the data that you get from a method you MUST assign it to a variable.  It is very common for students to forget this

 

import javax.swing.JOptionPane;
public class Pointless
{
   public static void main()
   {
      JOptionPane.showInputDialog("Please Enter a Description");
      ...

This is technically allowed and it will compile,  but since the returned String isn't assigned to a variable it's lost into nothingness.

Recap: The JOptionPane class has a method named showInputDialog. When called, this method will get a String from the user using a small window. This method "returns" that String, which can then be assigned to a variable and used in the program.
