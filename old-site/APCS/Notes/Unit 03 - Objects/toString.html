The toString Method

The toString method is a very specific method that has a very specific purpose.

 

What's toString's purpose?

When Java has simple data, such as an int, double, or String, it's easy to concatenate with a String.  Consider the code below, which uses a fake object named RandomObject

RandomObject r = new RandomObject(17,22,98,37);
System.out.println( r );

Java knows all of the object's instance fields, but doesn't know what characters to print out.  Should it list them with commas?  Should it only print the first one?  Should it print with any extra characters?

The toString method's purpose is to create the String that represents the instance fields of this object.

 

 

How to write the toString method:

It must have EXACTLY this header:
public String toString()

This is the method that Java looks for by default, if you change it in any way it counts as a different method.

As you can see, the return type of the toString method is a String.  The purpose of the toString method is to create and return a String that represents the object.  There is no specific formula for the String, but usually it should include the instance fields in some way.

 

Example:  Consider the Student class below, which contains a Student's name and gpa

public class Student
{
   private String name;
   private double gpa;
   
   public Student(String n, double g)
   {
      name = n;
      gpa = g;
   }
   
   public String getName()
   {
      return name;
   }
   
   public double getGPA()
   {
      return gpa;
   }
   
   public String toString()
   {
      String ret = "Name: "+ name + "GPA: "+gpa;
      return ret;
   }
}

If you look at the toString method, it puts the name and gpa variables into a String that it subsequently returns.

This will change the result of the following code:

Student s = new Student("Sacco", 3.8);
System.out.println( s );

The ouput of this would be "Name: Sacco GPA: 3.8", since that's exactly the String that the toString would return at that moment.

 

 

What happens if there is no toString?

If there is no toString, Java will default to a weird String that looks similar to the line below:

Student@3c23ba

This String is not very helpful for us.  It starts with the name of the class, followed by an @, and followed by a hex number that we don't want.  If you see this and it's not expected you should go to the class and check for the exact toString header:

public String toString()

It is NOT:

public String getString()
public void toString()
public String toString(String str)
