null

Most of the time you create an object variable you constuct an object directly into it:

SomeObject s = new SomeObject();

null is a special value that an object variable may hold which represents the absence of an object.

SomeObject s = null;

 

null is dangerous:

You must be careful when dealing with null values.  It is perfectly legal to have a variable that contains a null value, but the second you try to call a method using that variable you will crash your program with a NullPointerException.

SomeObject s = null;
s.someMethod(22);	<-- CRASH!!!!

 

 

A common reason for NullPointerExceptions:

80% of the time, a NullPointerException is caused by not being careful with your constructors.  Since the constructor's purpose in life is to put good (non-null) values into the instance fields of an object, leaving a variable as null would indicate that it wasn't completing the task. 

When you get a NullPointerException, you should make sure that your Constructor:

1) Is named properly with no return type.  If your constructor has a return type, it is a regular method and not automatically called on construction.

2) Is assigning a value to the instance field that was null.  Commonly students write their assignment statements backwards, forgetting that their instance fields need to be on the left of the = .
