Constants

Sometimes a class needs some information that isn't supposed to change.  It should remain constant.

The most common example of a constant is the PI variable in the Math class.  It holds 3.1415926..., and it should never change (pi is always the same).

 

Constants are final and static

A constant is like an instance field, except that it should not change.  Adding the keyword final makes it unchangable once it's set; the compiler will make sure of that.  They should also be static, making them belong to the whole class and not just a single object.

For Example:  In the Math class there is an instance field defined in the following way:

public static final double PI = 3.141592653589793;

 

Naming Conventions:

When naming a constant, it is good style to label the variable with ALL_CAPS, using an underscore between any separate words.

 

 

public or private:

This depends on the purpose of the constant.  A variable like PI is meant for the whole world to use, but it depends on its purpose:

- If the purpose is for the entire world to have access so that they don't have to remember it, like PI, then it should be public.

- If the purpose is to simply hold on to a value that is used within an object, then it should be private.

 

For Example, consider the BankAccount class: 

public class BankAccount
{
    private double balance;
    private String accountName;
    private static final double PENTALTY = 20;
    
    public BankAccount(String name, double bal)
    {
        accountName = name;
        balance = bal;
    }

    public double getBalance()
    {
        return balance;
    }

    public String getAccountName()
    {
        return accountName;
    }

    public void changeAccountName(String name)
    {
        accountName = name;
    }

    public void deposit(double cash)
    {
        balance += cash;
    }

    public void withdraw(double cash)
    {
        balance -= cash;
    }
    
    public void penalty()
    {
        balance -= PENALTY;
    }

    public String toString()
    {
        return "Account Name: "+accountName+"\nBalance: $"+balance+"\n";
    }
}

The penalty method of the BankAccount class is to deduct a flat $20.00 from the balance.  Since that number is not going to change, a constant named PENALTY was added to hold it.

Notice that its private, a programmer's decision since that variable is really there for use inside of the object.
