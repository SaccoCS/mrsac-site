Constructors

What is the purpose of the Constructor?

When writing an object, you don't know exactly what values should be in your instance fields.  The constructor is a special method whose purpose is to put the initial values into the instance fields.

Consider the Student class below:

public class Student
{
   private String name;
   private double gpa;
   
   public Student(String n, double g)
   {
      name = n;
      gpa = g;
   }
}

The constructor is this method:

 public Student(String n, double g)
{
   name = n;
   gpa = g;
}

This constructor has two parameters, a String and a double.  This is what allows us to construct our objects and pass in the information at the same time, like so:

Student stu = new Student("Gerald I. Joe", 3.85);

"Gerald I. Joe" would be passed into the parameter n in the constructor above.  As you can see, that value is assigned to the instance field (the more important variable).  The gpa is handled similarly.

 

Naming the Constructor:

Objects can have many methods, but in order to count as a constructor you must follow two rules:

1) It must be named exactly the same as the class. 

 In our example of the Student class, the constructor's header was:

public Student(String n, double gpa)

You can also see that this is where any parameters that the constructor needs would go.


2) There is no return type.

A common mistake is to accidently provide a return type for the constructor.  For example:

public String Student(String n, double gpa)

This will compile, but it doesn't count as a constructor.  It is just a normal method with a poorly chosen name.  Don't ever do this.
