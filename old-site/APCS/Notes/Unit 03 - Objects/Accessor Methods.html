Accessor Methods

Objects are all about managing the instance fields.  The problem is that the variables are private.  The purpose of an Accessor method is to access some information that the object contains.  Generally an accessor method doesn't change the object's instance fields, it just returns information. 

A simple example of an accessor method is the substring method of String.  Once a String is constructed, we can call the substring method to access some of the letters in the String (depending on the indices input).

String str = "Hello World";
System.out.println( str.substring(1,4) );

The code above will print out "ell", which is some instance-field related information that the substring method returned.

 

Methods are not variables:

A common mistake is to treat a method like a variable.

String str = "Hello World";
str.substring(6,11) = "Sacco";
System.out.println(str);

This is an attempt at assigning the value "Sacco" to the locations of "World".  This does not work!  You can never assign a value to a method like this.  If you want to change the instance fields of the object, you must use a mutator method.

 

Basic Accessor Methods

The simplest accessor methods come from the fact that instance fields are private.  Their purpose is to simply return an unmodified instance field.  This is so that the client can know what the instance fields are, even though they're private.

Often these methods are named using the word get, such as getName, getWidth, or getX, but the name could technically be whatever the programmer wants.

 

Non-Basic Accessor Methods

These are methods that are meant to pull some information from the object, but not just an unmodified instance field.  When writing these, you will need to use the instance fields to generate a new piece of information that you will then return.  The instance field should remain unchanged.


Example:

Consider the Name class below:

public class Name
{
   private String first;
   private String last;
   
   public Name(String f, String l)
   {
     first = f;
     last = l;
   }
   
   public String getFirst()
   {
      return first;
   }
   
   public String getLast()
   {
      return last;
   }
   
   public String getInitials()
   {
      String firstInit = first.substring(0,1);
      String lastInit = last.substring(0,1);
      String ret = firstInit+"."+lastInit+".";
      return ret;
   }
   
   public void setFirst(String f)
   {
      first = f;
   }
   
   public void setLast(String l)
   {
      last = l;
   }
}

 

Name's Basic Accessors:
In the Name class, the two basic accessors are the getFirst and getLast methods.  They simply return the appropriate variable.  

public String getFirst()
{
   return first;
}

This allows the client to know what the object's instance fields contain.

Name n = new Name("Flash","Gordon");
System.out.println(n.getFirst());

The snippet of code above will print out Flash, because that's what is stored in n's first instance field.  Otherwise there is no way of knowing what the fields of a Name object actually are.

 

 

Name's Non-Basic Accessors:
In the Name class there is one non-basic accessor method named getInitials.  It is an accessor because it returns some 'instance field related' information without changing the instance fields, but it is non-basic because it does more than just return the value.

   public String getInitials()
   {
      String firstInit = first.substring(0,1);
      String lastInit = last.substring(0,1);
      String ret = firstInit+"."+lastInit+".";
      return ret;
   }

This allows the client to ask any Name object for its initials, even though there is no specific instance field for it.
