boolean flags

When dealing with traversing a set of data, the concept of a boolean flag becomes important.  A boolean flag is a variable that starts out set to a specific value.  As the program goes on, if a specific situation occurs, then the flag is set to the opposite.  A key to working with boolean flags is that you never set them back to their original state.

 

For example:  Consider a method that looks at a String and returns true if it has an "a" in it, false if not. 

The strategy is to traverse the String and check each letter to see if it's an "a".  That is the moment that we know the result of the method is true.

This is a situation where we want to traverse through the String and look at a single letter at a time.

public static boolean hasAnA(String str)
{
   for(int i = 0; i<str.length(); i++)
   {
      String letter = str.substring(i, i+1);
   
   }
}

This is the standard traversal loop.  The int i is set up to represent every index in the String.  The substring is set up to get the a single character String that is at i.

 

Now that we can look at each letter, we need a way to return the appropriate boolean value.  Since the purpose of the method is to return true if an "a" is found, we can say that the first "a" we come across is the moment that we know the method will return a true.  If we never come across an "a", then we know that the method will return true.  This can be done using a boolean flag. 

 

public static boolean hasAnA(String str)
{
   boolean hasA = false;

   for(int i = 0; i<str.length(); i++)
   {
      String letter = str.substring(i, i+1);

      if( letter.equals("a") )
         hasA = true;   
   }

   return hasA;
}

The boolean hasA starts at false, and as the loop progresses it will check each letter and see if it's an "a".  The first time an "a" is come across, the flag is set to true.  If the loop never comes across an "a", it will remain false until it is eventually returned.

For Example:  if the String is "flamingo",  the boolean's value will be false for two iterations.  Once it comes across the "a", it is set to true.  It remains true for the remainder of the loop, even though none of the other letters are an "a".

 

 

boolean Flags don't Change Back:

Many students are addicted to else's.  They often make the mistake of adding one that ruins the loop.

    if( letter.equals("a") )
      hasA = true;
   else
      hasA = false;

In this case, if the String was "flamingo" the boolean's value would be set to true when it came across the "a".  The problem happens the next time the loop comes around.  Since the next letter isn't an "a", the boolean gets set back to true.  This is bad.   It changes the variable back to false, making it as if the loop had never come across an "a" before.

 

 

Making the Right Assumption:

When using a boolean flag, it is important to choose which value to assign the boolean first.  The hasAnA method would not function properly if it had started as a true value and then set to false the first time you didn't find an "a".  Think about the two statements below.

 - The first "a" that we find guarantees that the method will return a true

- The first non-"a" that we come across guarantees that the method will return a false

 Hopefully the first phrase sounds better than the second one.  If the second one were true, the "f" in flamingo would mean that there wasn't an "a" in the String.  Always take a moment to make sure that the assumption that you're making is the right one.
