Conway's Game of Life

Conway’s Game of Life is an organism simulation “game” from as far back as the 1970’s that was meant to show how simple rules can cause complex effects when done on a large scale. To start, various locations ( or “cells” ) in an empty grid are “populated”. When the grid is told to Step, each cell either lives or dies according to how many nearby cells are populated.

Watch a video explanation...  at home I guess.

The Rules:

Populated Squares (locations that hold a value of true):
Each cell with one or no neighbors dies of loneliness.
Each cell with four or more neighbors dies of overpopulation
Each cell with two or three neighbors survives to see the next step.

Unpopulated Squares (locations that hold a value of false):
Each cell with three neighbors becomes populated.

While it is not a normal game, many people look for patterns that can stay alive for as many generations as possible.

Tips for programming:

numOfNeighbors:
It is common for people to try and use 8 different if statements to check each box around r, c. This can get pretty ugly, so instead use a for loop to loop through all the combinations of rows and columns starting r-1,c-1 and going to r+1,c+1.


Inside the loop you’re going to want a big if statement to see if the box that you’re looking for is:

a. Occupied
b. Inside the boundaries of the 2D array (this needs four statements, one for each boundary)
c. Not the center cell (r,c), because it doesn’t count as a neighbor to itself. Only adjacent cells do.

If you use one big if statement to check all of these, make sure to use short circuit evaluation to make sure that the cell is within the boundary before you try to access that cell. The other way will give you an array index out of bounds exception.

Test your numOfNeighbors method by right-clicking on a square in the runner.  A window will pop up with the value it returns.  If it freezes, you probably have an infinite loop.  Make sure to click in the four corners to test for out of bounds.
