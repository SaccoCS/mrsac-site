While Loop Programs 3

Each of these programs will use at least one while loop in its solution.  Remember that inside a While loop you have to change some variables for the following time the loop runs.

SuperSafeDivider – This program will be very similar to SafeDivider. It will ask the user for a numerator and a denominator, but this time if the denominator is zero, a loop should activate that asks the user to input a non-zero number until they actually do it.

Example:
Please enter a numerator:
9
Please enter a denominator:
0
Please enter a non-zero denominator:
0
Please enter a non-zero denominator:
4
9 / 4 = 2 remainder 1

Hint:  You will ask the user for a numerator and a denominator once before the loop starts, and then use the denominator's value to keep asking until it's not zero

 

ImprovedGuessingGame: The original GuessingGame only allowed you to guess one time and told you if you were right or wrong. Create a new program named ImprovedGuessingGame, which will determine a random number between 1 and 20. This game will then repeatedly ask the user to guess higher or lower until they have the right answer.

For Example:
I’m thinking of a number from 1-20:
7
Incorrect, guess higher
18
Incorrect, guess lower
14
Incorrect, guess lower
10
Incorrect, guess higher
12
Correct! The number was 12.

Hint:  Select your random number and ask the user for one guess before the loop starts.  Inside the loop you will use an if statement to either print "guess higher" or "guess lower" and retrieve a new guess from the user.

 

DiceGame: Write a program named DiceGame, which will simulate a non-gambling Dice Game with the following rules:
1) The first Roll of two dice is made, that value is considered the “point”. The goal for the rest of the game is to roll the point again before a 7 is rolled.
2) The user “rolls” repeatedly until either they roll the point again or a seven is rolled. At either point the game is over.
3) If the point was rolled, the user wins. If not, the user loses.

For Example:

Initial Roll Coming Out!
You rolled a 2 and a 4 to get a 6.
The point is 6. Let the Game Begin!

You rolled a 3 and a 5 to get a 8
You rolled a 1 and a 4 to get a 5
You rolled a 5 and a 6 to get a 11
You rolled a 3 and a 4 to get a 7

Sorry, you lose everything

Strategy: before you start looping, you must roll once for the "point" and once more for the first roll.  Then a while loop should keep re-rolling until the roll is a 7 or the point.  After the loop has run, use an if statement to determine if the user has won or lost.

In the very beginning of the program, create the following variables:
int dice1=0;
int dice2=0;
int totalRoll = 0;
int point =0;

As you run the game, you should be replacing these values when appropriate.

Optional:  BigPigLatin – This is the last Pig Latin program, I promise.  Previously, you set up a class that had two important methods named getPigLatin and indexOfFirstVowel, which you should copy into a new class named BigPigLatin.

The previous program was limited because it only could handle a small amount of words, but you’ve just written a program named SentenceSplitter that was able to chop up sentences into one word at a time. 

Using the SentenceSplitter program as a reference, create a program named BigPigLatin which will prompt the user to input a sentence, and translate the entire sentence to Pig Latin.  Remember to use the index of the space to help you retrieve one word at a time and convert it to Pig Latin.
