Painting with Images

While you can draw basic shapes on your SCanvas, using real images is often necessary to make your programs classy.

Previously we've used SLabels to hold our images, but we want them to be a part of our canvas, so NO MORE SLABELS.  Only use them when you want them to be a part of a GUI.

The Picture Class:

We've used the Picture class before to represent images on the screen.  Remember that if you want to use the images, you should first save them into the folder that all your .java files are.

Your instance fields should create a variable of type Picture

private Picture myPic;

In the constructor, you load the images from the folder using the loadFromFile command:

myPic = Picture.loadFromFile("Some Image.jpg");

Download the following image to your project folder (and only your project folder)

Now copy the following program to your project:

import sacco.*;
import sacco.gui.*;
public class SRHSLogo extends SCanvas
{
    private Picture logo;
    
    public SRHSLogo()
    {
        logo = Picture.loadFromFile("srhs-logo.jpg");        
    }
    
    @Override
    public void paintCanvas(PaintBrush brush)
    {
        brush.drawPicture(logo,0,0);
    }
    
    public static void main()
    {
        SFrame frame = new SFrame();
        frame.add(new SRHSLogo());
        frame.setSize(500,500);
        frame.setVisible(true);
    }
}

The key to this code is the following:

brush.drawPicture(logo,0,0);

This draws the Picture with its upper-left corner at (0,0).  When you draw the image this way, it draws it using the original image's natural width and height.  If you run the program you should get the following:

 

The image is too big to fit in the frame.  This is because the natural width and height of the image are too large.  There is another way of drawing Pictures that involves providing a width and a height.  Go to the paintCanvas method and replace the drawPicture line to be the following:

brush.drawPicture(logo,0,0,200,300);

This will resize the image to be 200x300.  Since the original image was nearly a square, it will have to stretch the image and cause some issues.

Picture has two methods that will tell us what the natrual size of the image is, named getWidth and getHeight.  Try the following way of painting, which uses these methods to paint the image at exactly half the width and height:

brush.drawPicture(logo,0,0, logo.getWidth()/2, logo.getHeight()/2);
