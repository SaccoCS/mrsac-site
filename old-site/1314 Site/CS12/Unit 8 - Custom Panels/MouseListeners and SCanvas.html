MouseListeners and SCanvas

Previously we've dealt with ButtonListeners and KeyListeners, but when dealing with a canvas, MouseListening becomes important.

Similar to how the SCanvas has a paintCanvas method and we're just rewriting a 'better' version of it to make our programs pretty,  the SCanvas is already automatically listening for events and we just have to override the special methods.

public void onMousePress(MouseEvent m)

Similar to a KeyListener's onKeyPress method, this method is called any time that you press the mouse button on the canvas. 

public void onMouseRelease(MouseEvent m)

This method is called when a mouse button is released.

Copy the following class into your project:

import sacco.*;
import sacco.gui.*;

public class MouseDemo extends SCanvas
{
        
    @Override
    public void paintCanvas(PaintBrush brush)
    {
        Font f = new Font("Arial Bold",30);
        f.setBold(true);
        brush.setFont(f);
        brush.setColor(Color.BLACK);
        brush.drawString("Click Me",10,30);
    }
    
    @Override
    public void onMousePress(MouseEvent m)
    {
        System.out.println("Hello World!!");
    }
    
    public static void main()
    {
        SFrame frame = new SFrame();       
        MouseDemo pan = new MouseDemo();
        frame.add(pan);
        frame.setSize(155,75);
        frame.setVisible(true);
    }
}

Notice that I've overridden the onMousePress method to print out "Hello World!".  Run the program and click around.  Every time you press the mouse you should get a print out. 

MouseEvent

The parameter of those two methods is not just an int this time.  It's an object called a MouseEvent, which pretty much just contains the x and y coordinate where the mouse press or release happened.  Sometimes you want to know about the location.  MouseEvent has two methods, getX() and getY(), which return the coordinates.  Often times you will start a mouse event method with the following:

int mouseX = m.getX();
int mouseY = m.getY();

Then you can use the mouseX and mouseY to guide how you want to respond.

Add the following method to the demo class:

public void onMouseRelease(MouseEvent m)
{
   int mouseX = m.getX();
   int mouseY = m.getY();
   
   System.out.println("Mouse Released at ("+mouseX+", "+mouseY+")");
}

Run it again.  Notice that when you press down on the mouse, ""Hello World" is still printed, but when you release, it tells you the coordinates of where you released.  Try clicking the upper left and see that the numbers are small.

 

Mouse Motion

The last Mouse method that you can override is called every time the mouse moves at least one pixel.

public void onMouseMove(MouseEvent m)

In the demo, change the method that says onMouseRelease to onMouseMove and run the program again.  Now the printin happens every time you move the mouse on the canvas at all.

Using instance fields:

Since you can only paint from the paintCanvas method, you'll find that many times you're going to want to use instance fields to accomplish a task. 

For example:  If you wanted to not println the mouse location, but have it written on the screen.

Copy and run the following:

 

import sacco.*;
import sacco.gui.*;

public class MotionDemo extends SCanvas
{
        
    @Override
    public void paintCanvas(PaintBrush brush)
    {        
        brush.setColor(Color.BLACK);
        brush.drawString("Mouse at (x,y)",0,10);
    }
    
    @Override
    public void onMouseMove(MouseEvent m)
    {
        
    }
    
    public static void main()
    {
        SFrame frame = new SFrame();       
        MotionDemo pan = new MotionDemo();
        frame.add(pan);
        frame.setSize(200,200);
        frame.setVisible(true);
    }
}

Run it and you'll see that it doesn't respond in any way.  That's because the onMouseMove method is empty.  Our goal is to change the x and y letters to be the actual values that the mouse is moved to.

The first thing people always try to do is to call drawString from the onMouseMove method.  This will not compile because the only way that you can paint is in the paintCanvas method.  Instead you must use instance fields.

Strategy:  Create two ints as instance fields.  Whenever the canvas draws itself, it writes whatever is stored into those fields.  Whenever the mouse moves, it changes whatever's in those fields.  This way, moving the mouse will change the instance fields, which whill change how the canvas is drawn.  Hooray.

1)  Add to the instance fields: 

private int mouseX;
private int mouseY;

2) Modify the paintCanvas method's drawString to be the following:

brush.drawString("Mouse at ("+mouseX+", "+mouseY+")",0,10);

3)  In the onMouseMove add the following, which sets the instance field to wherever the mouse is:

mouseX = m.getX();
mouseY = m.getY();

 

Run the program.  As you drag the mouse around, you should notice the values on the screen changing.  This is because we've used instance fields.  It is important that you think about instance fields as you write your programs.

1)  FaceChanger:  Create a canvas that will show one face, but then when it's clicked another face will be displayed.  Maybe it changes from happy to sad, or maybe the color changes.

Strategy:

- Add a boolean as an instance field.  When that boolean is true, it will mean the face is 'happy'.  When that boolean is false, it will mean the face is 'sad'. 

- In the paintCanvas method, use an if statement to see if that boolean is true or false and drawing the appropriate face based on that information.

- In the onMousePress method.  Use an if statement to switch that boolean between true or false.  The idea being that this is all we need to do to change which face is being drawn

Once all three of those steps are done, run the program.  Click on the screen.  Hopefully the face will change back and forth when you click.
