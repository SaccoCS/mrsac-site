Draggable Polygons

Copy the following class file to your project:

import sacco.*;
import sacco.gui.*;

public class PolyDragger extends SCanvas
{
    private Polygon myPoly;

    public PolyDragger()
    {
        myPoly = new Polygon();
    }

    public void paintCanvas(PaintBrush brush)
    {
        brush.setColor(Color.RED);
        brush.fillPolygon(myPoly);
    }

    public void onMousePress(MouseEvent m)
    {
        int mX = m.getX();
        int mY = m.getY();

        if( m.getButton() == 1)
            myPoly.addPoint(mX,mY);

        if(m.getButton() == 2)
            myPoly.reset();
    }

    public static void main()
    {
        SFrame frame = new SFrame();
        frame.add(new PolyDragger());
        frame.setSize(500,500);
        frame.setVisible(true);
    }
}

If you run the program you'll see that left clicking will help create a Polygon, like before; and pressing down on the mouse wheel will reset the Polygon.

Our goal is to be able to click and drag the Polygon around using the right mouse button.

 

1)  Detecting the mouse presses: Add a boolean as an instance field named drag.  The purpose of this boolean is to represent whether or not the Polygon is currently being dragged or not.  For now, let's assume that any time the right button is pressed the Polygon should be dragged.

Go to the onMousePress method and make sure that when the right button is clicked, the drag variable is set to true.  Then add an onMouseRelease method that will simply set it back to false.

 

2)  Determining how much the mouse moves:  Add two ints as instance field to represent the old x values and the old y values.  Similar to how you did in the Scribbler program, add an onMouseMove method.  The very last thing that the onMouseMove method should do is to set the old x and old y to m.getX() and m.getY().  This is so that the next time the onMouseMove method is called we can still tell where the mouse was the last time that it was called.  Make sure that this always remains the last thing that this method does.

These variables are important because we need to know how far the mouse has moved.  At the beginning of the onMouseMove method, create two variables named difX and difY, which will hold the difference between the current x and y, and the old x and y.

 

3)  Moving the Polygon:  The Polygon class has a method built into it that will move all of its points by however much you want.  If you said myPoly.translate(5,23); then the polygon would move to the right 5 pixels and down 23 pixels.  In the onMouseMove method, if the drag variable is true, call the Polygon's translate method and pass in the difference in x and y that you calculated earlier.  This means that for every little bit that the mouse moves, the polygon will move by the same amount.

4) Test the drag: Run the program.  Use the left click to create a polygon, and then try and drag it with the right mouse button.  At the moment it shouldn't matter where you click, the shape should move as long as you're dragging.

5)  Only drag when you click the Polygon:  It's classier for the Polygon to only be dragged if the initial mouse press is actually on the shape itself.  The math can be tricky when trying to figure out if a specific x and y are within the boundaries, but luckily there is a method in Polygon that will do it all for you.  In the onMousePress method, change the if statement that sets the drag variable to true to also check and see if myPoly.contains(m.getX(),m.getY()) is true also.

Run the program one more time and this time make sure that when you right click the shape only moves if you're clicking on it.
