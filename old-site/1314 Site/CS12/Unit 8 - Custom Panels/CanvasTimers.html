CanvasTimers

In order to add movement to your programs, you'll need to use a timer of some sort.  In our case, the CanvasTimer class should prove useful.

Copy the following class into your project:

import sacco.*;
import sacco.gui.*;

public class Flashing extends SCanvas
{
    private Color myColor;

    public Flashing()
    {
       myColor = Color.RED;
    }

    public void paintCanvas(PaintBrush brush)
    {
        brush.setColor(myColor);
        brush.fillOval(150,150,200,200);
    }

    public static void main()
    {
        SFrame frame = new SFrame();
        frame.add(new Flashing());
        frame.setSize(500,500);
        frame.setVisible(true);
    }
}

Running the program should show a simple red circle on a white background.  Our goal is to modify it so that the color of the Circle randomizes once a second.  This means that we are going to need a CanvasTimer object to help us out. 

 

Setting up the CanvasTimer:

Add the following instance field:

private CanvasTimer timer;

This labels timer as a variable to hold the CanvasTimer that will be running our program. 

In the Constructor add the following line, which will construct the CanvasTimer:

timer = new CanvasTimer(this,1000);
timer.start();

The CanvasTimer's constructor first takes in an SCanvas, and then takes in a number of milliseconds to represent the delay.  Since we are writing the SCanvas, we pass in 'this', and passing in a 1000 means that it will wait about one second in between method calls.  Once the CanvasTimer has been set up, calling the start method will start it running.

 

Coding the action:

Now that the timer has been started, every second it will call the onTimerTick method of our SCanvas.  At the moment we don't have one.  Add the following:

 

@Override
public void onTimerTick()
{

}

This is the specific method that the timer will call every time it 'ticks'.  Our job is to randomize the color of the circle so that it's flashing.  Add the following code to the onTimerTick method:

int r = (int)(Math.random()*256);
int g = (int)(Math.random()*256);
int b = (int)(Math.random()*256);
myColor = new Color(r,g,b);

Now every time the timer ticks, a random color will be put into the myColor variable.  Since this variable is what the circle is drawn with, the circle should flash colors.  Run the program and make sure that it it flashing.

Assignments:

TimedRandSquare:  This program will start with a blank background picture.  Every time the timer 'ticks', a new random square is drawn to the background.

It will start sparse, but should eventually fill the screen.

 

Clock:  This program will draw a simple clock face.  As time passes, the line should rotate around as if it were a real clock.

Strategy:  The key to this program is using PaintBrush's drawAngledLine method.  You should have an instance field to represent the angle that the 'hand' is at, which the timer decreases every time it ticks.

 

GrowShrink:  This program will draw a colored circle centered around the point 250,250.  As the timer ticks, the circle grows by a little bit.  After it hits a maximum radius of 200, it starts shrinking instead of growing.  When the radius makes it below 10, the color is randomized and it starts growing again.

Strategy:  You will need to have instance fields to represent the radius of the circle, the color of the circle, the CanvasTimer, and a boolean to represent if the circle should be growing or not.

Make sure that the paintCanvas method is using the radius variable to paint the oval.  That way all you have to do is change the radius variable in order to change the size of the painted circle. 

The onTimerTick method should look at the boolean instance field variable and either add a small amount to the radius, or subtract a small amount from the radius.  This means that changing the boolean to true or false is in charge of whether or not the circle is growing or not.  Another thing that the onTimerTick method should do is to have two if statements.  If the circle's radius gets to small, set the 'grow' boolean to true,  if the circle's radius gets too big, set it to false.
