KeyListening with SCanvas

Similar to how the SCanvas has some MouseListening methods built into it, it also has some KeyListeners built in as well.  Previously we've used KeyListeners to work with STextFields, but with an SCanvas all we have to do is override the appropriate method.

Reminder:  There are two methods that are used to listen to the keys being pressed:

@Override
public void onKeyPress(int keyCode)
{

}

@Override
public void onKeyRelease(int keyCode)
{

}

The keyCode that is passed in represents the key that was pressed

 

1)  KeySketch:  Find the canvas that you previously created for Sketching.  This time we want to be able to use the keyboard to change the color.  You may either assign a different color to each F-key (at the top of the keyboard), or have some other keys control the color (R is red, G is green, etc)

2) MovableFigure:  This program will be in charge of drawing a grid with a character at some location.  The grid should be drawn using for loops to help make the process easier.  In the instance fields, there should be ints to represent the x and y that the character is located.  When they arrow keys are pressed, the character's x and y should either increase or decrease to cause it to move one location over.  In the example, the character is 50x50 and the gaps in the grid are all 64 pixels wide.

Also make sure to only move the character if the next location is in bounds so as to prevent the character from running off of the screen.
