Creating Movement:

 

Movement in Java is created by having an object ( in this case a circle ) move by a little bit each time the Timer 'ticks'( calls the actionPerformed method ).

 

 

If I draw a ball using brush.fillOval( 25, 50, 20, 20) the ball will always be in the same location with the same size.  This is not how we want to do it.  Create and set up instance fields to represent the x and y of the ball.  Now the command brush.fillOval( x, y, 20, 20) will draw the ball at whatever the x and y instance fields currently are.

 

Making the ball move:

Whenever the Timer ticks the x coordinate and the y coordinate of the ball should change by a small amount ( usually less than 10 ).  The change in x and y are values that should be instance fields as well ( I like to call them xVect and yVect ).  These represent the speed in each direction.

 

 

 

If the yVector is 0 and the x vector is some non-zero number, this means that the ball will move horizontally.

 

 

 

The first step of our project will be creating a program that starts, with a ball on one side of the panel, then drifts to the right. 

 

Again, set up a panel with instance fields for x, y, xVect, yVect and a CanvasTimer.

 

The Constructor should set up the Timer to tick every 50 ms.

 

The paintCanvas method should paint the ball according to the instance fields.

 

The onTimerTick method should change the x coordinate by adding the xVector to it, and the same for the y.

This demo has an if statement that will look at the x coordinate and set it back to 50 if it gets too large.  Otherwise the ball would drift off into nothingness and never return
