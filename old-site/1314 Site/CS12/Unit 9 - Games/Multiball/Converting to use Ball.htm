Getting ready for objects

Before starting this project, read through the parts of an object to get the idea of how to create your own custom objects.

We are going to switch from directly handling the x,y,xVect, and yVect of the ball to having a Ball object as an instance field and calling the appropriate method of Ball in order to acheive the same effect as before.

In a new project, create a canvas that is back to the basic ball bouncing around without the paddles.  You may copy from your old project, but make sure you only keep what you need.  You should be able to location the commands that do the following:

Instance fields:

x,y,xVect,yVect,Timer

In the constructor

Set the x,y,xVect, and yVect to some appropriate values

Set up the Timer and the size of the panel

paintCanvas

paints the ball

onTimerTick

Add the xVector to x and the yVector to y. 

Run this program and make sure that it is running properly before continuing.

 

We are now going to incorporate the Ball class into our program.  A Ball object will package together the Ball's x and y as well as its vectors.  Ball will have a method named update that causes it to move one step by adding the vectors to x and y.  We will be able to tell what the x and y coordinates of the Ball is by calling getX() and getY().  Finally, the Ball object will be able to paint itself using its paintSelf method. 

 

The Ball class doesn't exist yet, but we will write it later.  This means that you will not be able to compile until the next assignment.

Now we are going to edit the code that we have so that it uses the Ball class.

1)  The instance fields:

At the top of the program, you currently have ints to represent the x,y,xVect,and yVect.  Delete them and replace with:

private Ball myBall;

The Ball object is going to hold all of the other information under one variable name, so this one line is as good as the other 4.

2)  The constructor:

If the constructor set some of the instance fields you just deleted, delete those lines.  Instead create a Ball object and pass in two numbers for the x and y for the Ball to start at.

myBall = new Ball(250,250);

The Ball object will set its x and y and will give itself random vectors when we create it this way.

3) The paintCanvas method:

Find the lines in charge of painting the ball at an x and y.  Delete these lines and replace them with:

myBall.paintSelf(brush);

The Ball object will be able to paint itself using this method.

4) The onTimerTick method, moving the ball:

Find the two lines that add the vectors to x and y.  Replace them with:

myBall.update();

The Ball class' update method will add the vectors to the x and y when called.

5) The onTimerTick method, bouncing the ball:

Find the 4 if statements that cause the ball to bounce around the screen.  Instead of calling x and y directly, replace x and y with myBall.getX() and myBall.getY().  If you have code that sets x to some number like x = 0, replace it with myBall.setX(0).  When its the ball's time to switch the vectors, replace that code with myBall.horizBounce() or myBall.vertBounce().

For example the code:

if( x<0)
{
   x=0;
   xVect *=-1;
}

 

Would be replaced with

if( myBall.getX() < 0 )
{
   myBall.setX(0);
   myBall.horizBounce();

}

 

Now that you've successfully used  the Ball class instead of just a collection of instance fields it is time to write the Ball class.
