Character Containment

You may have noticed that it's possible to have one of the block push your character off of the screen. This is because the blocks will always move, even if the character is trapped.

The solution is to use two types of movement. When a character is moving to the right it can be shown by either moving the world to the left or by moving the character to the right. Essentially, when your character is in the middle/left side of the screen, moving to the right should be done to the character itself, but once it hits a certain point, the world should move instead

Add a moveLeft and moveRight method to the JumpMan class that is similar to the one in the Block class. This will allow us to move the character in the same way that we move the Blocks.

In your Canvas class, add an int as an instance field named MaxX and assign a value of 512 to it. You may modify this later if you want.

Now go to the location in your onTimerTick method that detects if the right arrow is pressed. At the moment it has a loop that moves all the blocks and xAligns mario at the same time.

You will need to separate this action into two parts. First take care of the movement. If mario is less than the max, he moves to the right, and if he's greater than the MaxX you should loop through all the blocks and move them in the opposite direction. After that you can worry about xAligning, which will require a completely separate loop.

Test out your game. You character should be jumping around freely until it gets to the far right, were the world should then move to meet it.

After your right boundary works, add another int for the minX, and do a similar process for moving mario to the left.
