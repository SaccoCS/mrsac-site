Part 1:  The Drifting Block

 

This is similar to the Ball Bouncing project that you have previously done with some key differences.  You are going to create an Object class named Asteroid that has methods similar to the Ball class.

 

We have previously been using x and y vectors to represent the speed and direction of the Ball.  For Asteroid we are going to use variables for the SPEED (double) , DIRECTION, and ROTATION.  You will instance fields for speed, direction, rotation, width, height, x, and y, but you will not need any vectors.

 

 

The Constructor will take in NO parameters and it will set all instance fields to their starting values ( eventually these will have random )

 

You will also need the following methods that are the same as the Ball class:

 

int getX()  returns the x value

 

int getY()  returns the y value

 

void setX(int tmpX)  sets the x to the given tmpX

 

void setY(int tmpY)   sets the y to the given tmpY

void update()

 

Since the variables represent the speed and direction, we can use a little trigonometry to determine what to add to the x and y values (speed and dir are the instance fields). 

 

x+=(int)(speed*Math.cos(Math.toRadians(dir)));

This is very similar to adding vectors, just represented a little differently.
