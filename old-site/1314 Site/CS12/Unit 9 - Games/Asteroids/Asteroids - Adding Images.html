Asteroids - Adding Images

Now that we've got the basic movement of an Asteroid ready to go, all we have to do is change how each object is represented visually by drawing them using images instead of colorful squares.

The images are already in sacco.jar. Remember, when getting Picture objects out of the jar you use the Picture.loadFromJar( "space/filename.png"); command. Notice the 'space/', which is different than before. Each of the Asteroids related images are in the space folder, so you will have to add this before any of the image names in order to retrieve them.

Modifying the Canvas to have an image:

The Canvas object you've created represents the environment that all the Asteroids are on. To enhance the environment, it should draw a background picture. You may choose your own appropriate background, but sacco.jar has an image "space/bluespace.png" that will do.

1) In the canvas class, add a Picture as an instance field that will represent the background.

2) In the constructor, use Picture.loadFromJar to load the Picture INTO THE INSTANCE FIELD VARIABLE. If you have your own image saved to the folder, use Picture.loadFromFile instead.

3) In the paintCanvas method, before you loop and paint all of the Asteroids, use brush's drawPicture method to draw the background.

4) Run the program. Hopefully, you will see a picture in the background instead of a white background. Make sure that the colored squares are still in the foreground.

Modifying the Asteroids to have an image:

Now it's time to change the Asteroid class to paint an image instead of a colorful square. Similar to the canvas class, you should make changes to the instance fields, the constructor, and the paintSelf method. First every asteroid use the "space/asteroid1.png" to draw themselves.

Once you get it working where all Asteroids use the same image. Modify the constructor to use a random number to determine which image should be loaded into the instance fields. YOU DO NOT NEED TO ADD INSTANCE FIELDS. Each asteroid only has one Picture variable, the constructor simply chooses which Picture to put into that one variable.

The five image names are "space/asteroid1.png", "space/asteroid2.png", "space/asteroid3.png", "space/asteroid4.png", "space/asteroid5.png". While a five-part if statment is perfectly acceptable, the cool kids will be able to do this without one.
