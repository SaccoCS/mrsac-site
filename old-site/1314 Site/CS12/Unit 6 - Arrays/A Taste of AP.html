Trying an AP Style Problem

 When choosing their classes for the following year, students often ask what the AP class is like.  My response is usually that it mostly the same things that Computer Science 1-2 does, it just goes faster and the problems are more difficult.

The AP test at the end of the year involves writing parts of four complex problems.  Today's program is directly from the 2009 AP test. If you feel like AP might be a good class for you, try and attempt each of the problems before looking at the strategy suggestions that I give.  You are not expected to get it perfectly on the first try; a passing score on the AP test is under 50%.

Part A)  The first method has the following description:

Write the code for the getChargingCost method in your program.

Strategy:  This problem is all about looping through an array and summing the values in it.  The chargeTime tells you how many times to loop, while the startHour tells you what index to start at. 

Before you start looping you will need two variables:  one to represent the sum and one to represent the index you're looking at, which should start at startHour.

Set up your loop to loop the number of times stored in the chargeTime variable.  Note:  in this situation your looping variable is not the index that you'll be looking at, so it's ok to start it at 0.

Inside the loop your job is to take the value at index and add it to your sum.  After that, increase the index by 1 so that it represents the next index.
-Problem:  increasing the index may eventually take you out of bounds.  To fix this, add an if statement that will set index to 0 if it reaches 24.

 

Test your program with using the table in the description.  The first two columns are the parameters to use, while the returned value is the Total Cost column.

 

Part B)  Read the description for the second method below:

Strategy:  This program seems very difficult to complete if you don't realize that you have just created a method that will help you directly.  You will want to create variables to represent the "lowest cost" and the "index of the lowest cost".  Once those are created, loop through every index of the rate table and use the getChargingCost method to determine how much it costs to start at that index.

Strategy:  This is a similar strategy as finding the smallest value in an array except we're finding the lowest cost.

We'll start by assuming that the lowest cose is at index 0.  Start with the following lines:
int lowCost = getChargingCost(0,chargeTime);
int lowInd = 0;

From here your job is to loop through every index and calculate the charging cost by calling the getChargingCost method.  If you find that cost to be less than lowCost, then you should replace the value of lowCost and lowInd.

In the end you will return lowInd

Test your program using the table in the description.  The first column is the parameter, and the Start Hour of Charge should be returned.
