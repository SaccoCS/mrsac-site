Improved PigLatin

 Previously, we’ve created a program that will create rough Pig Latin translator by taking the first letter and moving it to the end of the word. This works fine for words such as “computer” and “java”, but the rules of Pig Latin actually say that the whole first consonant should be shifted to the back. When the second letter isn’t a vowel, the system breaks down.

“program” becomes rogram-pay, when it should be ogram-pray
“string” becomes tring-say, when it should be ing-stray

This breach of the rules of Pig Latin is unacceptable. Find the last version of Pig Latin that you completed and Copy-Paste it into another class, changing the name to ImprovedPigLatin.

Most of the code that is already there does not need to be changed. The main change will eventually need to be in the getPigLatin method, which was in charge of changing around a word to create its Pig Latin version. We are going to have to change the way this method works. Instead of just taking the first letter and moving it to the back, the method will have to take everything before the first vowel and move it.

 

Helper method:

In order to fix the pig latin program, we're going to need to create a helper method that will aid us in finding the first vowel

public static int indexOfFirstVowel(String str)

This is a helper method whose purpose is to return the index of the first vowel that it finds in str. One thing that’s different about this method when compared to the getPigLatin method is that instead of having a String as a return type, it has an int. This is because the method’s job is to return an index, which is represented as an int, not a String.

How are we going to find the index of the first vowel?

This part is tricky. We’ll ignore ‘y’ and just assume that the vowels are a, e, i, o, and u. We can use indexOf to find each index, but finding the lowest can be rough because of the possibility of a -1 (which is lower than every possible index).

Strategy:
Create an int named currentLowest and put a value in it larger than every index (the length of str should do nicely). After that’s completed, one at a time find the index of each vowel and check to see if it’s lower than the currentLowest. If it is, assign the index to be the new currentLowest.

For example:
If the word is “happy”, the value of currentLowest would start at 5, but since the index of ‘a’ is 2, which is lower than the 5, the currentLowest should be changed to 2.

This must be repeated once for each remaining vowel. After you’ve done that, the lowest index should be returned.

PROBLEM:
indexOf has that special property where it will return a -1 if the letter is missing. This will mess everything up because most words will be missing one of the vowels and thus, return a -1. Modify each if statement so that it will not replace currentLowest when indexOf returns a -1.
