TextAreas

Not every program can be all buttons and images.  There are other elements that can end up on an SFrame.  A basic object for text input is the STextArea.

Consider the following GUI, which adds three STextFields into a GridLayout:

import sacco.gui.*;

public class Info
{
    private SFrame frame;
    private SLabel nameLabel, dateLabel, periodLabel;
    private STextField nameText, dateText, periodText;
    
    public Info()
    {
        frame = new SFrame("Information");
        nameLabel = new SLabel("Name:");
        dateLabel = new SLabel("Date:");
        periodLabel = new SLabel("Period:");
        
        nameText = new STextField();
        dateText = new STextField();
        periodText = new STextField();
        
        frame.setLayout(new GridLayout(3,2));
        frame.add(nameLabel);
        frame.add(nameText);
        frame.add(dateLabel);
        frame.add(dateText);
        frame.add(periodLabel);
        frame.add(periodText);
        
        frame.setSize(300,200);
    }
    
    public void launch()
    {
        frame.setVisible(true);
    }
}

You can run it with the following main method:

public static void infoRunner()
{
   Info in = new Info();
   in.launch();
}

 

Stop the stretching:

At the moment the STextFields are added to a GridLayout, which causes stretching.  One way to avoid this is to add the STextField to an SPanel instead of directly to the frame.  Since the SPanels are FlowLayouts, they will let the STextField be its 'natural' size.  Replace the info object with the following code, which has this extra step:

import sacco.gui.*;

public class Info
{
    private SFrame frame;
    private SLabel nameLabel, dateLabel, periodLabel;
    private STextField nameText, dateText, periodText;
    private SPanel namePan, datePan, perPan;
    
    public Info()
    {
        frame = new SFrame("Information");
        nameLabel = new SLabel("Name:");
        dateLabel = new SLabel("Date:");
        periodLabel = new SLabel("Period:");
        
        nameText = new STextField();
        namePan = new SPanel();
        namePan.add(nameText);
        
        dateText = new STextField();
        datePan = new SPanel();
        datePan.add(dateText);
        
        periodText = new STextField();
        perPan = new SPanel();
        perPan.add(periodText);
        
        frame.setLayout(new GridLayout(3,2));
        frame.add(nameLabel);
        frame.add(namePan);
        frame.add(dateLabel);
        frame.add(datePan);
        frame.add(periodLabel);
        frame.add(perPan);
        
        frame.setSize(300,200);
    }
    
    public void launch()
    {
        frame.setVisible(true);
    }
}

Not very good.  The problem now is that we didn't start the STextField obejcts with a widith.  Since the layout is now allowing the STextField to be its 'natural' size, it is collapsed down to nothing.
