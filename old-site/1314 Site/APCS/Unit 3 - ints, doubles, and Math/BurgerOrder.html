BurgerOrder

Write a class named BurgerOrder, which is in charge of keeping track of a simple food order at a fast food restaurant.

BurgerOrder should have instance fields to represent the number (not the cost) of hamburgers, french fries, and sodas that have been ordered.

Add three constants that represent the value of each of the three items. Burgers should cost $2.00, fries are $1.50, and a soda is $1.25. Don't forget to use all caps when naming these constants.

Add a constructor that sets the non-final instance fields to 0.

Add three mutator methods: addBurgers, addFries, and addSodas, which take in an int as a parameter and add it to the appropriate instance field

Add a toString that returns a String containing how many burgers, fries, and sodas are on the order.

Add a method named getTotal that uses all the instance fields to get a total of what the meal costs. DO NOT CREATE AN INSTANCE FIELD FOR THE TOTAL. IT SEEMS LIKE A GOOD IDEA, BUT IT IS NOT GOOD PROGRAMMING.

Test out your class by using BlueJ to construct a new Burger and use the methods to add a 2 Burgers, an order of fries, and 2 sodas to the order. When you call the getTotal method, your meal should cost $8.00 even.  Don't worry if the returned value is a messy double that has more than one digit after the decimal place.



Now add constant named SALES_TAX and set it to 0.0775 (7.75%). After you've done that, add a method named totalAfterTax() which will use the SALES_TAX variable to calculate the tax owed and then adding it to the total.

If you put in the same order as before, then call the totalAfterTax method you should get 8.62 returned.
