BaseChanger

Create a class named BaseChanger, which will be responsible for representing a number in many bases.

Instance Fields:
Add two ints as instance fields: One to represent the number in decimal, and one to represent the base of the number. We will always store the number in its decimal form, and use the base to convert it before we display it.

Constructor:
Create a Constructor that takes in one int, which will be in decimal form. Assign it to the instance field, and set the base to be 10.

Methods:

public int getBase()    //basic accessor
public int getDecimalValue()  //basic accessor

public void setBase(int newBase)
This method will simply set the base instance field to be the incoming value. Do not change the value of decimal in any way.


public String convertedForm()
This method will use the two instance fields to return a String that represents the instance field number, but converted to the base in the instance field. This should look very similar to the getBinary and getOctal methods that you've seen before. MAKE SURE THAT YOU DON'T CHANGE THE INSTANCE FIELD IN ANY WAY. You will need to copy the value of the number into a local variable before you start converting.


public String toString()
This method will return a String that is made up of the convertedForm of a number along with an underscore followed by the base. For example: 11 in base 2 would be "1011_2".

Modifying to Work with a base greater than 10

The converter that you've made so far will only work for bases up to 10. This is because when the value at a certain place goes above 9, it needs to be converted to the appropriate letter before adding it to the String. You can see this if you create a BaseChanger with a value of 162 and set the base to 16. The converted form SHOULD be "A2", but currently it will put a "10" in for the "A", giving "102" instead.

{

return val+"";
else
return (char)(val+55)+"";
}

This method takes in an int and returns the single character form of it. for example, getDigit(10) returns an "A", getDigit(11) returns a "B", etc.

How does this method work? UNTESTED: YOU MAY SKIP IF YOU DON'T CARE:
There is another primitive data type called char, which is almost the exact same as int. The only difference is that it stores an ASCII value and is interpreted as a character. The useful part is that you can CAST and int into a char. What this method does is find an ASCII value of the appropriate character by adding 55 (if val is 10, 10+55 is 65, which is ASCII for "A"). After that, I cast it into a char before adding it to a String and returning it. This trick can be useful when looping through the alphabet.

Accepting values that already have a base:

Now you've got a class that takes in a decimal number and can convert it to any base we want it to. It's time to make it go the other way.

Add a NEW constructor, which only takes in a String to represent the number. This String is formatted specifically like the toString of BaseChanger. There are two parts, separated by an underscore. The part before the underscore is the number, and the part after is the base that it's in. This new constructor's job will be to use these two values to determine the decimal value (because that's what the instance field needs).

The first thing that you need to do is separate the parameter String into two parts: the part before the underscore and the part after. The part after will need to be parsed into an int in order to be any help. When you've done that, test to make sure that you've done this properly. Add a temporary println that will print out those two values, and then use BlueJ to construct a new BaseChanger with "1A5_12" as a parameter. Hopefully it prints out 1A5 and 12 separately. It's important that this functions before you move on.

Once you have the number (as a String) and the base (as an int) you can begin to convert to decimal. The way that you want to do it is one digit at a time, similar to verticalString. Since each digit is a String that can't necessarily be parsed, put the following method in your program:

STRATEGY:
In order to get the decimal version of the number, you must get the digits one at a time, multiply them times their place value, and add them to the instance field. I recommend starting an int for the place value and starting it at the largest place value needed (it's based on the length of the number and base). Then use a while loop that will take the front digit, multiply it by the place value, add it to the instance field, and then divide the place value by the base, making it smaller for the next digit. Once the place value is 0, your loop is complete. Make sure that the instance field for the base also gets set.

Add the following methods:

public BaseChanger add(BaseChanger other)
public BaseChanger subtract(BaseChanger other)
public BaseChanger multiply(BaseChanger other)

These methods will each be in charge of a different math function. They will get the decimal value, perform the operation, and return a new BaseChanger with the result. If the bases of both numbers are the same then the result has the same base, but if they're different the result's base will be the larger of the two.

For example:  If you wanted to add 1B base 16 to 25 base 5 it would look like the following:

BaseChanger b1 = new BaseChanger("1B_16");
BaseChanger b2 = new BaseChanger("25_5");

BaseChanger result = b1.add(b2);

System.out.println(result);
