Finalizing the Game:

So far we have been focusing mostly on getting the model right and adding all the methods that our program will need. Once we’ve added methods for our controller to use, the actual game part will be easy.

 

TetrisGrid:

moveBlock

Back in the TetrisGrid class, add a method named moveBlock, which takes in two locations, a start and and an end. The idea behind this method is that it takes whatever block is at the starting location and adds it at the ending location. Because of the location conundrum we cannot simply put Block into the array and be done. We need to take special care that we use a method that will set the block’s location variable, such as the addBlock method.

Worrying About null:

The addBlock method of the TetrisGrid sets the location of whatever block is passed in. This is now an issue because the moveBlock method might be given a location that has a null. Modify the addBlock method so that it will assign the value to the grid at the given location, but it only sets the location if it’s non-null.

 

dropRow

Add a method named dropRow, which takes in an int as a parameter. The goal of this method is to loop through every column in the given row and to move it “down” one row. Don’t forget that you have the moveBlock method and the Location’s getLocInDir method to help.

 

isRowFull

Add a method named isRowFull, which takes in an int as a parameter and returns a boolean. The purpose of this method is to return true if the given row is full and false if it contains at least one null value.

hasAFullRow

Add a method named hasAFullRow, which takes in no parameters and returns a boolean. This method is the next step past the isRowFull method. It should loop through every possible row and return true if any of them are full. A false value would indicate that none of the rows are full.
