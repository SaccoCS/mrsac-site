The Location Class

Tetris is a game that is all about organizing blocks in a grid. This means that we will be dealing with rows and columns to tell where each block is. The first class you create will be the Location class., which will be the most basic object that we make.

- Create a class named Location.  For instance fields, the Location class should have ints to represent the Location's row and column.

- Location's constructor should take in a row and column and apply them to the instance fields.

- Add methods getRow and getCol, but do not add a setRow or setCol method. Locations should be immutable, meaning once they’re constructed, the instance fields cannot be changed. Much like the String class.

- Add a method named plus, which takes in a second Location and returns a new Location that is the sum of 'this' Location and the parameter Location.

- Override the toString method to return something simple that contains the row and column. For example “Loc(4,2)”

 

Referring to the four directions:

During this project you are going to want to figure out an adjacent Location. We want to avoid writing a separate method for each direction, such as getLocBelow, getLocAbove, getLocLeft,or getLocRight. Our solution will be to create one method that will take in one of the four directions and return the adjacent Location in the given direction.

The problem is that there is no such thing as a 'Direction' variable; so what do we pass in?  Create four public static final ints named UP,DOWN,LEFT,RIGHT. While the specific number that you assign to them is not really important, it will be slightly more useful if you use the keycodes for the arrow keys. Up is 38, Down is 40, Left is 37, and Right is 39. Now, when a method needs to take in a direction, it will take in an int. That int should either be Location.UP, Location.DOWN, Location.LEFT, or Location.RIGHT.

Add the following method: public Location getLocInDir(int direction) This method should construct and return a new Location in the direction given. You should have different Locations for each of the four directions. If no direction is matched, return null. It is important that these work properly.
