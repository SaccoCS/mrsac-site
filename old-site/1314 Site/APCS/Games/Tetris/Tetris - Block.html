The Block Class

So far we have made the Location class so that we can encapsulate a row and column together to represent a specific place in the grid.  Now that we have standardazied that, we are now going to focus on the next most basic element of the game; the Block.  Everything in the game is represented using colorful blocks at specific locations in the grid.  Each piece in Tetris is made up of 4 Blocks, and every time a piece fully drops, the Blocks remain in the grid while another piece is created.

Create a class named Block, which as a Color and a Location as its two instance fields.

Block should have 2 constructors. One should take in a Color and assign it to the instance field. The other should take in no parameters and assign a random color to the instance field.

Add the following methods:
public Color getColor()
public Location getLocation()
public void setLocation(Location loc)
public void setColor(Color col)

Override the toString of Block to return a String that includes its Location and Color.

Testing:

Add the following tester method and run it:

public static void blockTester()
{
    Block b1 = new Block();
    b1.setLocation(new Location(2,5));
    
    Block b2 = new Block(Color.RED);
    b2.setLocation(new Location(-3,8));
    
    System.out.println(b1);
    System.out.println(b2);
}

The output should look similar to the following.  The only real difference you should expect is the first Block's color.  It was constructed using the default constructor, meaning the Color should be random.

Block at Loc(2, 5) with Color[R:44 G:41 B:80]
Block at Loc(-2, 8) with Color[R:255 G:0 B:0]
