Explosions

Create a class named Explosion. An explosion is simply in charge of displaying a series of pictures to make an animated explosion.

An explosion’s instance fields include x,y,width, height, and a Picture array.
Explosion’s constructor should take in an x,y,width, and height and apply them to the instance fields.

In order to initialize the Picture array, during the constructor call SpriteLoader.getExplosionPics(); and assign the returned value to the instance field. This array has 32 Pictures in it that should be cycled through one at a time when painted. To accomplish this, add an int as an instanced field to represent the index of the array that you’re currently on.

Add an update method that will simply increment the index.

Add a paintSelf method that will draw the Picture in the array at the index, but only after first checking to make sure the index isn’t too large. You’ll want to stick with using x and y as your centerpoint, so don’t forget to paint from a point up and to the left of the x and y.


Incorporating it into the project:

Add an Explosion variable as an instance field in the AsteroidsBoard. We don’t always want the explosion to be on the screen, so we’ll keep it null until it’s time.

In the paintWindow and onTimerTick methods, first check to make sure that the Explosion variable isn’t null, and then call the paintSelf and update methods if the object exists.

 

Testing

To test out the explosion, override the onMousePress method in the AsteroidsBoard

public void onMousePress(MouseEvent m)

in this method, extract the x and y out of the MouseEvent and construct an Explosion object into the instance field, giving it a width and height of around 50.

Run the program and click around the screen.  Hopefully wherever you click, an explosion will appear.

 

 

Exploding on Contact:

At the moment the entire game stops awkwardly when the ship hits an Asteroid.  Instead, we're going to make sure that when the ship is not active, it's invisible and has no hitBox.

In order to make the ship stop interacting with the other objects in the screen, we need to prevent some things from happening.  In the AsteroidBoard class, go to the the following places and modify the if statements to also require that the ship is active:

- Checking to see if the ship overlaps the asteroids
- Checking to see if the ship overlaps the powerup
- When it's time to shoot the bullets

Now, change the stopGame method.  Instead of telling everything to stop, make it only set the ship to inactive (and now invisible).  Immediately construct a new Explosion at the ship's x and y.  The width and height of the Explosion is your choice.
