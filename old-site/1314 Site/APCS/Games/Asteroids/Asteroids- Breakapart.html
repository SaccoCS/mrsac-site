Breaking the Asteroids

As we have the game right now, all the Asteroids just suddenly disappear when they're struck by a Bullet.  Previously we've set up the Asteroids to have one of three sizes.  The intention is that when a LARGE Asteroid is hit, it creates two MEDIUM Asteroids.  A MEDIUM Asteroid generates two SMALL Asteroids.  The SMALL Asteroids just disappear like they do now.

First we're going to need to have another way of constructing the Asteroids with information from another Asteroid.  Add another constructor to the Asteroid class that takes in 4 things: doubles for x and y, an int for the size, and a Picture.  Use them to set all the instance fields, using inherited methods when appropriate.  The size value must be put into the width and the height.

Right now there is already a getX,getY,and getWidth method.  Add one more method to the Asteroids class named getPicture, which will return the instance field.  This will be used to make sure that the two Asteroids that break apart are not different colors.

 

AsteroidsBoard:

Go to the bulletsVSAsteroids method.  At the point where you remove the bullets and Asteroids before you return, check the size of the Asteroid being returned (getWidth OR getHeight is the size).  If that size is Asteroid.LARGE, then extract the x,y, and Picture out of the removed Asteroid and construct two new Asteroids, passing those values in as parameters to the new constructor.  The only parameter that isn't exactly the same as the  original Asteroid is the size, which should be set to Asteroid.MEDIUM.  Add each of those Asteroids to the ArrayList so that they are now a part of the program.

Create a second if statement that checks to see if the size is Asteroids.MEDIUM and adds two SMALL Asteroids.  Do not create a third if statement.  The SMALL ones just disappear.
