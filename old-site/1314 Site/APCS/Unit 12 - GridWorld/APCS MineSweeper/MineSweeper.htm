MineSweeper

Using the GridWorld framework as a tool, we can create a simple MineSweeper game. 

I have created an extension of ActorWorld that takes advantage of the locationClicked and keyPressed methods that are built into GridWorld.    Basically, every time a location is clicked the Actor in that space is told to act().  The world will then check to see if any Bombs have been placed on the grid, or if every Bomb is covered by a FlaggedSquare.  To lay a flag down, hit the f key before you click and the runner will trade FlaggedSquares for Squares and vice versa.

There will be many images that you need to copy into your folder.  Download the zip file here, extract all the files out, and copy them into your project.  DO NOT JUST DRAG THE ENTIRE FOLDER, MOVE THEM TO THE PROJECT FOLDER.

The class you will be building is Square, which you read about HERE.  Essentially, a Square can be secretly a bomb.  When the Square is told to act:  if it's a bomb it will replace itself with a new Bomb actor, if it isn't a Bomb it will either the number of bombs nearby.  If there are no bombs nearby, the Square will tell all of its neighbors to act, which should reveal them as well.

The MineWorld code is posted here.  I used only GridWorld so you should be able to understand roughly what I was doing.  The locationClicked method is essentially the runner, and the rest of the methods are created to help me break the code into smaller parts.

View the hierarchy for MineWorld.  Don’t be afraid, many of the classes are merely there so the will have different icons on the grid.

PlainRock: is a class that extends Rock( we want these to do nothing when they act ), but sets the Color to null, meaning there won’t be a Color over the icon

import info.gridworld.actor.Rock;

 

public class PlainRock extends Rock

{

    public PlainRock()

    {

        setColor(null);

    }

}

 

The classes Zero, One, Two, Three, Four, Five, Six, Seven, Eight, and Bomb all extend PlainRock and nothing else.

For example:

}

Common Problems:

1)  I you are getting a NullPointerException and can’t figure out why:

When you put another Actor in the Grid at YOUR location, YOUR Grid and Location variables are set to null.  After that, any calls to getGrid() or getLocation() will return a null.  To this may be fixed by storing the current Location and Grid into a temporary variable before taking action to replace the current Square object with any of the numbers or Bomb.

 

2)  StackOverflowException (often looks like the program is partially frozen, then crashes):

When two squares are next to each other and are both Zero, they will continuously try and tell each other to act.  This will happen until Java realizes it’s stuck and crashes.  Make sure that when you tell all the Actors around you to act(), you first check to see if they’ve already acted or not.

3)   ClassCastException:

This usually occurs when you have a list of Actors and you’re trying to ask if the Actor isABomb() or not.  Often people simply cast it into a Square and call isABomb().  This will throw an Exception if the Actor isn’t a Square.  Modify your if statement to first check to see if the Actor is an instanceof Square, THEN you can cast it and ask.
