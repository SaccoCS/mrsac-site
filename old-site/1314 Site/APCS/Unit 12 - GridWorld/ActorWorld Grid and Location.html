GridWorld:  ActorWorld, Grid, and Location

For GridWorld to run, you must be familiar with constructing Grids and ActorWorlds.

ActorWorld:

The ActorWorld class is in charge of all the visual portions of GridWorld.  When an ActorWorld is told to show, it creates all of the buttons and displays any Actors,Bugs, Flowers, Rocks, or Critters that have been added to it.

Create the  following class in your GridWorld project folder.

import info.gridworld.actor.ActorWorld;
import info.gridworld.actor.Bug;
import info.gridworld.actor.Rock;
import info.gridworld.grid.*;
 
public class MyFirstGridWorlds
{
    public static void defaultActorWorld()
    {
        ActorWorld world = new ActorWorld();
        world.add(new Bug());
        world.add(new Rock());
        world.show();
    }
}

There are three important things going on here:

1)  An ActorWorld is constructed and stored as world
2)  A Bug and a Rock are added to the ActorWorld.  You'll notice when running that they show up at random locations.
3)  The ActorWorld is told to show itself.  If this method is not called, you will never see your project.

Compile and run the defaultActorWorld, A Bug and a Rock should appear in random locations.

Press the step button.  Everything in the Window will be told to act one time.  We will get into their individual behaviors later, but for now take a look at how a Bug acts and how a Rock Acts.

 

The default ActorWorld will create a standard ActorWorld, but sometimes you want a situation that's a little bit different.  The second constructor for ActorWorld takes in a Grid.  Since Grid is an interface, there is no such thing as a Grid object, but there are two classes that implement the Grid interface.

Run all three methods and look at the differences between the World that pops up in all three situations.  The Grid that is used to construct the ActorWorld is what determines the size of the World.

The Location class

Previously, we've used classes such as Point in order to encapsulate together an x and a y. Instead of an x-y system, GridWorld uses rows and columns.  If you view the Location class API you will see that there is not  a lot to the class.  The methods you will call most often are the getRow() and getCol() method.

Creating ActorWorlds using Locations

When you create an ActorWorld, using the add method will add a Bug at a random location in the Grid.  If you want to have more control over it, you may use a Location object to specify where it goes.

public static void locationWorld()
{
    ActorWorld world = new ActorWorld();
        
    Location startLoc = new Location(1,2);
    Bug myBug = new Bug();
        
    world.add(startLoc, myBug);
    world.add(new Rock());

    world.show();

}

Run the method a couple times.  Notice the difference between the Bug, which was added using a Location, and the Rock, which wasn't.

 

Locations and the Direction int

In the GridWorld universe, an int is used to determine what direction is being talked about.  Like degrees, the number is between 0 and 360.  While you should have a rough idea of what number represents each direction.  There are static constants inside the Loction class that make progamming simpler.

Add and finish the following method:

public static void directionConstants()
{
   System.out.println( "Location’s NORTH variable contains the int: " + Location.NORTH);
   //Repeat the previous code for NORTHWEST,WEST,SOUTHWEST,SOUTH,SOUTHEAST,EAST, and NORTHEAST
}

 

NORTH, SOUTH, EAST and WEST are useful for indicating direction, but there are some other variables that represent a change in direction named HALF_LEFT, HALF_RIGHT, and FULL_CIRCLE.

Add and finish the following method:

public static void turningConstants()
{
   System.out.println( "Location’s LEFT variable contains the int: " + Location.LEFT);
   //Repeat the previous code for HALF_LEFT, AHEAD, HALF_RIGHT, RIGHT, HALF_CIRCLE, and FULL_CIRCLE
}

 

Notice that the direction constants, such as NORTH, WEST, and SOUTHWEST represent the specific direction that they're facing ( NORTH is 0, meaning if the direction that you're facing has a value of 0), but the turning constants are different.  Location.NORTH + Location.LEFT would tell us what direction we would be facing if we started at NORTH ( 0 ) and made one left turn ( -90 ).  The result would be -90, meaning WEST (-90 is the same as 360 - 90, which is 270).

Location.SOUTHWEST + Location.HALF_RIGHT

If I start out facing southwest and make half of a right turn my new direction is 270, which is the same as Location.WEST.

 

Location's other methods

void getAdjacentLocation(int direction)

Add the following method to your code:

public static void nearby()
{
    Location loc = new Location(1,2);

    Location newLoc = loc.getAdjacentLocation(Location.EAST);

    System.out.println(newLoc);

}

This code tells us what the location to the east of the Location row 1, col 2.  If you read through the code you'll see that a Location is created with row 1 and column 2.  Once that Location is created it's asked what the direction is to the east of it.  The result is stored into newLoc, which is then printed out.

Common mistake:

getAdjacentLocation(loc,Location.EAST) is WRONG.  The method is not static, you must call it using an already constructed Location object.  loc.getAdjacentLocation(Location.EAST).

 

int getDirectionToward(Location tmpLoc)

Add the following method to your code:

public static void findingDirection()
{

    Location loc = new Location(1,2);
    Location loc2 = new Location(2,2);

    int dir = loc.getDirectionToward(loc2);
    System.out.println(dir);

}

This method allows a Location to tell you what direction it would have to go in order to head toward the other location.  In the sample, the Location (1,2) is asked to tell us what the direction is toward the Location one row below it (2,2).  The result should be 180, which means directly SOUTH.
