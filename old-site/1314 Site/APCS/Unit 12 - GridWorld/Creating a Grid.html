Creating a Grid

An important part of GridWorld is when the Actors interact with the other Actors around them.  In order to make this easier, the Grid interface was created.  At the core, a Grid is essentially a 2D array with methods to help add and remove Actors at specific Locations.  There are also a few methods that will provide some ArrayLists that will be useful.

While BoundedGrid and UnboundedGrid work perfectly well, we are going to create our own Grid so that we can look a little deeper into it.  Copy the following incomplete class to your project folder.

import info.gridworld.grid.*;
import info.gridworld.actor.*;
import java.util.ArrayList;

public class SaccoGrid implements Grid<Actor>
{
    private Actor[][] myArr;

    //Constructor: sets up myArr to be a 20X20 2D array
    public SaccoGrid()
    {

    }

    //Constructor: sets up myArr to have r rows and c columns
    public SaccoGrid(int r, int c)
    {

    }

    //Many more methods go here
}

Since we are implementing the Grid interface, this will not compile until we add several more methods.  Check out the Grid interface using the API to see what is supposed to be implemented.

Keep in Mind:

We are going to consider a location unoccupied if it has a null value.  You must be extra careful when dealing with nulls.

The instance field is a 2D array, but some methods return ArrayLists.  Though these are similar, the way that you use them are very different.  Don't forget that arrays use the square brackets while ArrayLists only use methods.

 

Instance Fields and Constructors:

Notice that the instance field is a 2D array of Actors.  Polymorphism allows any class that extends Actor (Bug, Rock, Flower) to be stored in this array.

Each of the Constructors should create an new 2D Array and store it in the instance field.

 

Basic Accessor Methods:

Now that there is an Array all ready to go, there are a few basic method that this object needs.

   public int getNumRows()
   public int getNumCols()

Each of these should return the number of rows or columns that this Grid has. 

 

   public boolean isValid(Location loc)

This method is only checking to see if the given Location is within the boundaries of the 2D array.  It returns false if the row or the column are too large or too small.

 

   public Actor get(Location loc)
   public Actor remove(Location loc)

These methods both return the Actor that is at the given Location.  The only difference between the two is that the remove method will set the location in the 2D array to null.

 

   public Actor put(Location loc, Actor act)

Assigns the parameter Actor to the specific Location in the 2D array.  The Actor already at that location is returned (it could be null)

 

Advanced Helper Methods:

 

   public ArrayList<Location> getOccupiedLocations()

This method will return an ArrayList of all the Locations in the entire Grid that are occupied (non-null)

 

   public ArrayList<Location> getValidAdjacentLocations(Location loc)

This method takes in a Location as a parameter and returns a list of all of the adjacent (next to) Locations that are valid.  You will need to use a nested loop along with the isValid method you've already written.  Don't forget to exclude the center Location "loc" as it is does not count as adjacent.  If you would like to tell if a Location has the same row and col, you may use Location's equals method to help you.

 

   public ArrayList<Location> getOccupiedAdjacentLocations(Location loc)
   public ArrayList<Location> getEmptyAdjacentLocations(Location loc)

These methods each return an ArrayList of Locations.  One contains the adjacent Locations that are occupied and the other contains the adjacent Locations that are empty.  Writing another nested for loop is unneccessary.  Call the getValidAdjacentLocations method first and then work from there.  Warning:  When it's time to add a Location to the list you should construct a new Location with the row and column, don't call the Actor's getLocation method (in the runner, one of the Actors will have a null location which can cause problems)

 

   public ArrayList<Actor> getNeighbors(Location loc)

This is the only method that returns an ArrayList<Actor>.  Again, a nested for loop is not needed.  Call the getOccupiedAdjacentLocations method first and use the results to produce the ArrayList you need.
