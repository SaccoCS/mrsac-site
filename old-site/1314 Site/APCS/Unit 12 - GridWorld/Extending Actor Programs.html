Extending Actor Programs

import info.gridworld.actor.*;
import info.gridworld.grid.*;
import java.util.ArrayList;
import java.awt.Color;

Complete the following extensions of Actor. To use the icons in GridWorld, download the image to your project folder ( right next to the Actor ). The image must be named the same as the Actor. If you do not like the blue tint that Actor defaults to, in your constructor call this.setColor(null)

To test your programs, add a static method to each class similar to the one below:

public static void main()
{
    ActorWorld world= new ActorWorld();
    world.add(new RotatorActor());
    world.show();
}

RotatorActor:

Create an Actor named RotatorActor ( class RotatorActor extends Actor ). Override the act() method so that a RotatorActor’s direction changes by 10 degrees each time it is called.

DiscoActor

Create an Actor named DiscoActor. Override the act method so that the DiscoActor changes Color each step.

TIP: Use an instance variable that is an array of Color objects.

private Color[] colorList = {Color.RED, Color.BLACK, Color.GREEN, Color.CYAN, Color.LIGHT_GRAY, Color.MAGENTA, Color.ORANGE, Color.PINK};

Once this is established, you may change the Color of the Actor using setColor( colorList[2] ). Usually you set the color using setColor( Color.RED );

GardenerActor

Create an Actor named GardenerActor.  Each step a GardenerActor will look at all the empty locations around it.  It will randomly choose one of those empty locations, create a flower and put it at that location (don't use Grid's put method, use Actor's putSelfInGrid method).  If there are no empty locations, the gardener will tell all the actors around it to remove themselves from the grid.

RockChaserActor

Create an Actor named RockChaserActor.  Each step a RockChaserActor will look to see if there is a Rock anywhere on the Grid.  If there is, the RockChaserActor will take one step toward the Rock.  To accomplish this you will need to find the Location of the Rock, and then use use Location's getDirectionToward method as well as its getAdjacentLocation method.

If there is no Rock in the Grid, the RockChaserActor should put a new Rock at a random location.  Make sure that the random location is not the same as the RockChaserActor.

Tip:  How to test if an Actor is a Rock

Actor tmp = ...
if( tmp instanceof Rock)
{
...

Strategy: This is a multi-step problem.  I recommend starting a local Location variable that starts at null.  If you find a Rock while looping through the occupied locations, store it's Location in that variable.  At the end of the loop you'll either have a null value, meaning you never found a Rock, or a non-null value, meaning you found a Rock and have it's Locaiton.
