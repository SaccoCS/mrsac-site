Short Circuit Evaluation

The following program should be solved using short circuit evaluation, which means that each method has an operation that might crash the program and you need to use an && or an || to protect it.  Since Java stops attempting to evaluate AND and OR statements the moment it knows the guaranteed result.  On the far left, add a boolean condition that will guarantee that the dangerous code will not run.

For example:
If I'm checking to see if the last 4 digits of a String are ".com", there is a possibility that str.substring(str.length()-4) could crash the program when the length of the String is less than 4.  So an if statement might be the following:

if( str.length()>=4 && str.substring(str.length()-4).equals(".com") )

Now it's impossible to crash during this if statement.  In the case that the length is too small, the first portion of the if statement will become false, preventing the second part from even evaluating.
