Objects as Parameters

We've begun to see Objects more and more in our programs.  An important part of Objects is the idea that you can have many different copies of an object, each with different values in the instance fields.  For example:  The Point class is simply in charge of an x and a y, but you can have several different Point objects, each with their own x an y.

When you have an object and need to access one of the instance fields, usually there is a method that will allow you to retrieve that value.  If there isn't a method, then you probably weren't meant to have access to it.

 This is true whenever you use objects, so it is always a good idea to familiarize yourself with the methods that are available to you when you're faced with one.

 

Another way of seeing objects in your code is to have them passed in as parameters in a method.  Look at the static method definition below:

public static boolean isAnAce(Card someCard)

This time the parameter is a Card object.  Our job is to figure out if the parameter is an Ace, but it is not as simple as someCard.equals("Ace") because someCard is not a String, it's an object with Strings in it.  We have to use the methods available to extract the information we need out of it first, then we can see if it equals "Ace".

The completed method below accomplishes the task by first extracting the value String using the getValue method.  Once that String is out of the object it can then be compared with "Ace".
