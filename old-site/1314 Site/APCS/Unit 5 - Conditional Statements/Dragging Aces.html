Dragging Aces:
 
For this program we are going to be using the SaccoObject methods to drag the Aces around the window using the mouse. First we'll need to set up the Ace so that it has methods that allow it to be moved, and then we'll look at some more mouse event methods and use them to tell the Ace to move when it's appropriate.

Setting up Ace to Move:

Add another method to Ace named translate, which takes in two ints from the user and adds them to the x and the y values. This will allow the Window to move the Ace around the screen.


More with Mouse Events
Create a new class named AceDragger, which should be very similar to the CardHolder class. The only difference is that it should not start out with anything in the "onMousePress" method. If you're using copy and paste, don't forget that the main method should not be creating a CardHolder.


There are three Mouse Event based methods that we have to understand before moving on. Each is called at a very specific time:

public void onMousePress(MouseEvent m)
This method we have seen before, it is called the moment that the mouse button is pressed down.

public void onMouseRelease(MouseEvent m)
This method is called the moment that the mouse button is released. It seems very similar to the onMousePress method, but will be useful in detecting if the mouse is currently held down.
 
public void onMouseMove(MouseEvent m)
This method is called every single time the mouse is moved. This will be useful for dragging cards because when the mouse moves, the Ace should move with it.
 

The Order of Events:

We don't want the Ace to move every single time that the mouse does, we want it to only do it if it moves during the time that the mouse is held down.

The order of events:
1) The mouse is pressed and is on the Ace, signaling that we want to "grab" an Ace.
2) The mouse is moved. Since the Ace is currently "grabbed", it should move
3) The mouse is release, signaling that we want to let go of the Ace.

How to Tell if the Mouse Button is Being Held

The big question to answer is, "How will the onMouseMove" method know if it should be moving the Ace or not.
Add a boolean instance field named isAceHeld and start it at false. Our job is to make sure that this variable is true the entire time that the mouse is down after clicking the Ace, and false the entire time that the mouse is up.

Add a line of code to the onMousePress method so that it sets the boolean to true if the Ace contains the point of the MouseEvent. Add a similar one to the onMouseRelease that sets it to false, no if statement required. We now have made sure that when the Ace is clicked, the boolean is set to true until the mouse button is released.

 

How Far Should the Ace Move?

Now go to the onMouseMove method. We now have the ability to tell if the mouse is down because of our isAceHeld instance field. If the mouse is held down, we want to call Ace's translate method, but that provides a new problem. The onMouseMove method's MouseEvent can tell us where the mouse is currently, but not where it was the last time it was moved.

The fix:
Add two more ints as instance fields: lastX and lastY. The idea is that AFTER the onMouseMove has done whatever it's supposed to do, its last action will be to put the x and y into those two new instance fields.  This means that the next time the onMouseMove method is called, it will have access to the current location AND the last location of the mouse.  This will be useful, since the difference between the two is exactly how far the Ace should move.

We finally have the information we need to finish the onMouseMove method. It should first check to see if the Ace is being held. If it is, it should then look at the MouseEvent and the lastX and lastY, and use them to calculate the difference between the x's and the y's. Now that you've calculated the difference in each direction, pass those two values into the Ace's translate method.  The last two lines of this method should still be setting the lastX and lastY for the next time.
