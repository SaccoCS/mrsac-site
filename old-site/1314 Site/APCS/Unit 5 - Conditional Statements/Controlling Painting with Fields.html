Controlling Painting with Instance Fields

Create a class named Ace, which is going to represent an Ace on a SaccoObject. 

Ace should have instance fields for the x,y,width, and height of the Ace. 

The constructor will take in no parameters and set the x to 200, the y to 200, the width to 71, and the height to 96.

 

Using The Picture Class (DO NOT CREATE A CLASS NAMED PICTURE)

Add a new instance field of type Picture, which is a class that Sacco made to help represent an image.

private Picture acePic;

There are a few images already inside of sacco.jar which can be accessed using the static loadFromJar method in the Picture class.  In your constructor, use the following code to put the image of an Ace of Spades in the acePic variable:

acePic = Picture.loadFromJar("/cards/AS.png");

Similar to the QuestionBlock class, the Ace class is not going to be the SaccoObject, but it is going to be able to paint itself.  Add a method named public void paintSelf(PaintBrush brush).  Inside you should use PaintBrush's drawPicture method to draw the Picture using your x,y,width, and height:

brush.drawPicture( acePic, x, y, width, height)

  

Using a SaccoObject to Display the Ace:

Since this class isn't a SaccoObject, we're going to need a SaccoObject that tells the ace to paint.  Use the following class to test your program:

import sacco.*;

public class CardHolder extends SaccoObject
{
    private Ace myAce;
    
    public CardHolder()
    {
        myAce = new Ace();
    }
    
    public void paintWindow(PaintBrush brush)
    {
        myAce.paintSelf(brush);
    }
    
    public static void main()
    {
        CardHolder ch = new CardHolder();
        ch.showWindow();
    }
    
}

Notice that this class is really just a container to hold the Ace object.  When doing SaccoObject projects this will be a common theme.

Run the main method and make sure that an ace of spades appears on the screen.

 

Using an instance field to guide painting:

If we were writing a game, the cards wouldn't always be face up.  Many times we would need the face down version of the card.  Add the following instance fields to the Ace class:

private boolean isFaceUp;
private Picture backOfCardPic;

The isFaceUp field is a boolean, meaning that it represents one of two states: true or false.  We are going to interpret a true as "face up" and a false as "face down".

Modify your constructor to set the isFaceUp field to true.  The backOfCardPic should be loaded like before, but using the String "/cards/b1fv.png".

 

The paintSelf method:

We now have an issue.  The Ace class should only really draw one Picture, but there are two Pictures to choose from.  Use an if statement to look at the isFaceUp variable and decide whether of not to draw the acePic or the backOfCardPic.

 

Test Both Ways:

Run the program the way that it is.  If done properly, the window will show the same card as before.  That is because the isFaceUp variable was set to true.  To test the other case, go back to the constructor of Ace and set the isFaceUp variable to false and run it again.  This time the back of the card should appear.

 

Controlling with a method:

Since this boolean plays a big factor in how the Ace paints itself, lets add a method that controls it:

public void flipCard()

All this method has to do is make sure that the isFaceUp variable is set to the exact opposite of what it was.  This may be done with an if statement or simply by using an exclamation point.

Test out this method by using BlueJ to construct an Ace and keep the Object inspector open while calling the flipCard method a few times.  Make sure the boolean is toggling between true and false.

 

Flipping the Card with a MouseEvent

SaccoObjects have several methods built in to help interact with the user.  Go to the CardHolder class and add the following method:

public void onMousePress(MouseEvent m)

This method is a special method that is called whenever a button on the mouse is pressed.  The parameter that's passed in carries some more information about the event, but we don't care about that right now.  We are going to treat every click exactly the same, essentially turning the whole window into one big button.

The only line of code in the onMousePress method should be calling the Ace's flipCard method.  The idea is that when anybody clicks on the window, the card's image should toggle between face up and face down.

 

 

Add the following method to the Ace class:

public boolean containsPoint(int tmpX, int tmpY)

This method will return true if the combination of tmpX an tmpY is inside the boundaries of the card.  Remember that you know the x, y, width, and height.  Determine the two values that tmpX must be between and the two values that tmpY must be between, and write a four-part AND statement to return true when tmpX and tmpY are within all the bounds.

 

This method means that now an Ace can help us tell if the point that we've clicked is within its boundaries or not.  Go back to the CardHolder class' onMousePress method.  At the moment we flip the card no matter what point is clicked.  We're going to modify it so that it only flips the card if we've clicked on it.

The MouseEvent parameter is similar to the Point class, it has a getX and a getY method that will tell you the point where the mouse was pressed.  First call those methods to get the x and y out of the object and into some ints.

Since the containsPoint method returns a boolean value, we can use it directly in an if statement.  Modify the method so that it first ask if the Ace contains the point before deciding to flip it.

if( myAce.containsPoint(x,y) )
   myAce.flipCard();

 

Test out your program again.  If done properly the card will only flip when you click within it, not any other time.
