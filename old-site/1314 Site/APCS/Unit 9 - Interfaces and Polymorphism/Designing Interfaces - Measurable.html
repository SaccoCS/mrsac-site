Designing Interfaces:  The Measurable Interface

The main purpose of interfaces is for Polymorphism, which is useful because it allows you to reuse code to work with similar objects.

Consider the DataSet class below: 

public class DataSet
{
    private double sum;
    private double max;
    private int count;

    public void add(double num)
    {
        sum += num;

        if( count == 0 || num > max)
            max = num;
            
        count++;
    }

    public int getCount()
    {
        return count;
    }

    public double getMaximum()
    {
        return max;
    }

    public String toString()
    {
        return "The average of the "+count+" items is "+sum/count+", with the largest value being "+max;
    }
    
}

This object, once constructed, will have its add method called repeatedly with a new double passed in as the parameter.  If you look at the add method you should see that it does three different things.  First adds the double to its summing variable, then it checks if the new value is the maximum value, and then a counter is incremented.

Applying it to Objects

This is a useful class, but right now it only works for doubles.  Let's modify it to work with an Object.  Consider the BankAccount class that follows:

public class BankAccount
{
    private double balance;
    private String accountName;
    private double interestRate;
    
    public BankAccount(String name, double initialDeposit)
    {
        accountName=name;
        balance = initialDeposit;
        interestRate = .01;
        
    }
    
    public void gainInterest()
    {
        balance = (1+interestRate)*balance;
    }

    public void deposit(double money)
    {
       balance += money;
    }
    
    public String getName()
    {
        return accountName;
    }

    public double getBalance()
    {
        return balance;
    }

    public String toString()
    {
        return accountName+":"+balance;
    }
}

This class manages a single BankAccount's information.  You can set an initial balance and call the gainInterest method to increase the amount of money in the account, as well as deposit money.  At its core, this is really just a double, and we may want to take a set of BankAccounts and determine the number, average, and maximum balance of each BankAccount.

 

Create a class named BankDataSet, which works identical to the DataSet class above, but it will work with BankAccount objects instead of doubles.  The "max" should be a BankAccount, the add method should take in a BankAccount as a parameter, and the getMaximum should return a BankAccount  Instead of adding a double directly to your sum, you should call the getBalance method, because that is the core double that the BankAccount is representing.

Run your code with the following method, which constructs a few BankAccounts and gains some interest on each of them before adding them to a BankDataSet object.  The average should end up being around 770 and the Maximum account should be Diana.

public static void runner()

    BankAccount[] accounts = {new BankAccount("Allen", 1000), new BankAccount("Beth", 250), new BankAccount("Charlie", 500), new BankAccount("Diana",1235), new BankAccount("Evan",722)};

}

Using Polymorphism to Make a Better DataSet

 While it's nice to have a class that will help tell us information about BankAccounts, the process should not have to necessarily be BankAccount specific.  The BankDataSet class does not care about how the BankAccount was set up or what the name was on the account.  It only cares about the balance.  There are many objects that deep down are essentially keeping track of a double, but we don't really want to design a new DataSet object for each of them.  We want to work smarter, not harder.

Create a new Interface named Measurable, which is meant to be implemented by Objects that have a core double and it would make sense for them to use it with a DataSet object.  The only method that the Measurable interface should have is a getMeasure method, which is going to be how we get the double out of the object (getBalance is too BankAccount specific of a name).

Once the Measurable interface is compiled, get to work on a MeasurableDataSet class, which will work with any object of type Measurable.  Again, it should look very similar to the BankDataSet class that you just made, except the type of object is "Measurable" and the only method you need to call is the getMeasure method.  This should compile even though no object even implements the Measurable interface yet.

Once you feel comfortable with the MeasurableDataSet class, the BankAccount class needs a little modification to work with it.  First it needs to implement the Measurable interface, and second it needs to have the getMeasure method.  The get measure method should really just call the getBalance method, since that's the double we care about.  

Now go back to the runner.  The only modification it should have is that the BankDataSet should be changed to a MeasurableDataSet.  Running the program should print out exactly the same result as before.

 

Reusing MeasurableDataSet (the whole point of Polymorphism)

The whole purpose of the Measurable interface is that it allows the MeasurableDataSet to work with multiple different objects.  Copy the Coin class below into your project:

public class Coin
{

    private double coinValue;
    private String coinName;

    public Coin(String name, double value )
    {
        coinValue = value;
        coinName = name;
    }

    public double getValue()
    {
        return coinValue;
    }

    public String getName()
    {
        return coinName;
    }
    public String toString()
    {
        return coinName+":"+coinValue;
    }
}

This class is perfect for our new MeasurableDataSet object.  First you should implement the Measurable interface for Coin, similar to how you did for BankAccount, and then you should try the following runner.  This time you do not have to make a different DataSet object.  The MeasurableDataSet object should be just fine.  Try out the following runner:

 

public static void coinRunner()
{
   Coin[] coins = {new Coin("dime",.1),new Coin("quarter",.25),new Coin("penny",.01),new Coin("quarter",.25),new Coin("nickel",.05)};
        
   MeasurableDataSet mds = new MeasurableDataSet();
        
   for( int i=0; i<coins.length; i++)
   {
      mds.add(coins[i]);
   }
   System.out.println(mds);
}

Why use the Measurable interface?

Before the Measurable interface, a custom DataSet object would have to be created for every different object that we create

Once we have the Measurable interface, there is no longer a need to create so many DataSet objects.  We only need to design one that uses 'Measurable' objects.
