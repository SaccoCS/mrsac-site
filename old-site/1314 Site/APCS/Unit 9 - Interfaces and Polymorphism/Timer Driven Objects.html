Timer Driven Objects

We’ve seen before that a SaccoObject can act as a time through its onTimerTick method. Copy the following class into your project:

import sacco.*;
public class Timer extends SaccoObject
{
    public Timer()
    {
        this.setDelay(1000);
    }
    public void onTimerTick()
    {
        System.out.println("Hello World!!!");
    }
}

It’s been a while and there’s some trickery going on here. The “extends SaccoObject” allows the Timer class to be a SaccoObject without rewriting the entire class. This means that the “this.setDelay(1000)” is calling a method that a SaccoObject can use to change the delay between clock cycles.

Don’t stress too much about how that works yet (next unit). The important thing to know is that the code in the onTimerTick method is called repeatedly with a one second delay each time. Think of it as an infinite while loop with a delay.

Use the following main method to verify that it works.

public static void main()
{ 
   Timer t = new Timer();
   t.start();
}

NOTE: The way it’s written the timer isn’t stopped properly. If you run this method multiple times you’ll see that Hello World is being printed out by multiple timers, causing it to act funny. You are going to have to manually reset java in order to stop this. Yes, it’s obnoxious.


The Cat Class

Consider the simpleCat class below, which has a speak method that, when repetitively called, will say “mew” five times and then start saying “MEOW”.

public class Cat
{
   private int count = 0;
   private String sound = "mew";

   public void adjustSound()
   {
      if(count>5)
         sound = "MEOW!";
      count++;
   }

   public void speak()
   {
      System.out.println(sound);
      this.adjustSound();
   }
}

In order to simulate a lonely cat, we’re going to create a timer class that will have a Cat as its instance field and will call the speak method every time that the timer ticks.

import sacco.*;
public class CatTimer extends SaccoObject
{
    private Cat myCat;

    public CatTimer(Cat tmpCat)
    {
        this.setDelay(1000);
        myCat = tmpCat;
    }

    @Override
    public void onTimerTick()
    {
        myCat.speak();
    }
}

Now that we have two objects that are made to work together, we’ll need a main method to get things started.

public static void main()
{
    Cat kat = new Cat();
    CatTimer timer = new CatTimer(kat);
    timer.start();
}

Run this method and make sure that the cat is meowing on the terminal.



The Dog Class

Now consider the Dog class below: 

public class Dog
{
    private String sound = "Howl";
    
    public void lengthenHowl()
    {
         sound = sound.substring(0,sound.length()-1)+"wl";
    }
    
    public void speak()
    {
       System.out.println(sound);
       this.lengthenHowl();
    }
}

Create a class named DogTimer that is exactly the same as the CatTimer class except modified to work with a Dog. Use the following method to test it out:

public static void main()
{
   Dog d = new Dog();
   DogTimer t = new DogTimer(d);
   t.start();
}

You should get increasingly long howls

The Parrot Class

Now, just to make you mad, do it again with the Parrot Class below.  Copy and Paste is your friend.

public class Parrot
{
    private String sound1 = "SQUAWK!", sound2 = "Polly want a cracker!";
    private boolean bool = true;

    public void flipSound()
    {
        bool = !bool;
    }

    public void speak()
    {
        if(bool)
            System.out.println(sound1);
        else
            System.out.println(sound2);

        this.flipSound();
    }
}
