Casting Objects

Polymorphism is useful, but there are some drawbacks.  Once an object is labeled as a different type, the methods that you can call are limited to the type that it's labeled as.

For example:  The BankAccount class had many methods in it, such as gainInterest, getName, and deposit.  Since it implements the Measurable interface it may be stored as a measurable.

Measurable myMeasure = new BankAccount(1500);

Even though we've constructed a specific type of Measurable object (a BankAccount), it's being stored with a Measurable variable.  According to the compiler, there is only one thing that a Measurable can do, which is getMeasure.  The following will not compile:

myMeasure.gainInterest();  //doesn't compile

Even though we know that myMeasure contains a BankAccount, which can gainInterest, the Measurable variable is limited to what is defined by the Measurable interface.

The solution is Casting.  If you have a Polymorphic variable that you're sure is a certain type, you may use casting to force the variable to be a certain type.

BankAccount myBank = (BankAccount)myMeasure;

This will only work if the myMeasure variable deep down inside is a BankAccount.  You cannot just cast any object and hope to get away with it.  Now that the object is labeled as a BankAccount, it can do anything that a BankAccount can do.

myBank.gainInterest();  //compiles

The first method is nearly complete, it relies heavily on the findLowest method of the Sorter class.  Compile the code and you'll see that there's a problem.  The following line has a type mismatch:

String lowest = Sorter.findLowest(names);

The compiler says that it expected a String, but found a Comparable, and since not every Comparable is guaranteed to be a String it will not compile.  The reason is that the return type of the findLowest method is a Comparable.

The solution is casting.  We as programmers know that we gave the method an array of Strings  and that the only thing that could be returned by the findLowest method would be a String.  Since we are 100% certain, let's cast.  Change the line of code like so:

String lowest = (String)Sorter.findLowest(names);

Now the Comparable being returned by the findLowest method is being cast into a String.  This allows for the use of the substring and indexOf method for use later in the program.
