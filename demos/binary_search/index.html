Searching Through a Sorted Array

When evaluating the efficiency of a search, we think about how many values from the array that we need to access for our best and worst case scenarios

Linear Search:

A linear search is the search that we've been using so far. It loops through the indices one at a time looking for the target value.
- Best Case: Value is found immediately at index 0 ( 1 value accessed )
- Worst Case: Value is found at the very end of the array ( arr.length values accessed )

In the demo below you are asked to find a single value among 41 values, but this time you have the extra benefit of knowing that the values are already sorted.

This information does not change the efficiency of a Linear Search
- Best Case: 1 value accessed
- Worst Case: 41 values accessed


The Binary Search Strategy

But we can use this information to our advantage using a Binary Search Strategy

Look at the value in the MIDDLE of the array and compare it to your target value

  • If the value is equal to the target: return the index it was found at.
  • If the value is smaller than the target, repeat the process on the lower half, checking the middle value of that set
  • If the value is larger than the target, repeat the process on the upper half, checking the middle value of that set
