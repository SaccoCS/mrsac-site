The toString Method

Object design is useful for creating customized variables with as many pieces of information as the designer would like. This can become a problem when an object get printed. Java is not smart enough to look at our design and know exactly how to arrange the instance field into something that a human would read and understand. For this reason if you print out a new object, it will not appear in a useful fashion.

Name@1a3914
Date@f35cc7
Student@ad1c80

As usual, the solution lies in adding a method. This time you don't have to think about the method header at all. It must be exactly this:

public String toString()

This is a special method in Java and must be defined this way.

The purpose of this method is to return a String that represents this object. This String could contain as many of the instance fields as the designer feels are important.

Example: Name

The Name object has two instance fields, to represent the first and last names of a person. We know that the first name goes before the last name, but the computer doesn't know anything about how we arrange our names. Consider the Name class, whose toString method returns a String in the order that we know to be important.

public class Name
{
   private String first, last;

   public Name(String f, String l)
   {
      first = f;
      last = l;
   }

   public String getFirst()
   {
      return first;
   }

   public String getLast()
   {
      return last;
   }


   public String toString()
   {
      return first +" "+ last;
   }

}

The addition of the toString method to the Name class allows the object to be concatetnated.

Name ghost = new Name("Jacob","Marley");
String str = "I'm haunted by "+ ghost;

This code now creates the String: "I'm haunted by Jacob Marley".

If the toString was not added, then the String would be "I'm haunted by Name@76ca21", which is less useful.
