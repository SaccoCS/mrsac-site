Constants

Occasionally you come across specific values that are useful for calculations in your code. A day will always have 24 hours, pi will always be 3.14159, and Euler's number will always be 2.71. Style-wise it is not very classy to have 'magic' numbers like this because it can make your code look confusing. Instead , you should use a constant to store the variable to a descriptive name so that it is clear what the value is when used.

Constants are final and static

When defining a constant it should be both static and final. The final keyword in front of a variable means that its value may never be changed once initialized.

public static final double PI = 3.14159265;

Since we have added final as a modifier, the compiler will have a problem with any attempt to assign a new value to the PI variable.

Naming Conventions

The style of naming constants is different than your regular instance fields or local variables. The variable should be ALL CAPS with underscores between any words.

public static final String MR_SACCOS_PASSWORD = "Ih82Grade!";

This means that you should intantly be able to tell if a variable is a constant by looking at the format of its name.

public or private?

Determining whether a constant should be public or private will depend on the situation it's in. If you're using a constant to be purely useful for calculations entirely within the class that it's defined, use private.

If you're using it to represent a special value that you want to access from another class, use public.

For this course we will not be graded on your choice of public or private. In general use public constants unless otherwise told.
