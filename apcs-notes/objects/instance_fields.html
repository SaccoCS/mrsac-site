Instance Fields

One way to think about an object is to consider it a variable that has multiple pieces of data bundled together. Those pieces of information are known as the instance fields or instance variables.

For Example: Consider the following class named Student and it's designed to have two instance fields, one to hold a student's name and one to hold its gpa.

public class Student
{
   private String name;
   private double gpa;

   public Student(String n, double g)
   {
      name = n;
      gpa = g;
   }
}

The variables name and gpa are the instance fields of the Student class. This means that every time a Student object is constructed, it will create its own unique version of those variables.

Student s1 = new Student("Gerald I. Joe", 3.85);
Student s2 = new Student("Tom Riddle", 2.16);

These two construction statements create two different objects, each with their own name and gpa variables. s1 has a name of "Gerald I. Joe" and a gpa of 3.85, while s2 has a name of "Tom Marvolo Riddle" and a gpa of 2.3

  • s1
  • name: "Gerald I. Joe"
  • gpa: 3.85
  • s2
  • name: "Tom Riddle"
  • gpa: 2.16

By creating objects we are encapsulating the two variables to help us group our information.

Instance Fields Should Be Private

While technically it is legal to have public instance fields, during this course we are going to keep them private and focus on more object oriented access solutions. You should never be able to change a value inside an object directly.

Student s1 = new Student("Gerald I. Joe", 3.85);
s1.gpa = 4.0; // <-- This shouldn't be allowed. Bad design.

When the instance fields are private, access becomes a challenge. The solution is to add accessor and mutator methods and take control of how the variables are accessed.

Local Variables

A local variable is more temporary than an instance field. They are created in order to assist methods in their process, but they're not necessarily as 'permanent' as instance fields. Every variable that you create that isn't an instance field is considered a local variable.

The main difference between local variables and instance fields is when they are created and when they are destroyed. A local variable is created and destroyed every time a method is called, while instance fields are created when the object is constructed and exist for the entire lifetime of the object.

Any Variable Declared Inside a Method is Local

Consider the following method, which asks the user for a quantity of seconds, and then uses int division and mod to determine how many minutes and seconds that breaks down to.

public static void minSec()
{
   String userInput = JOptionPane.showInputDialog("Enter a number of seconds");
   int totalSec = Integer.parseInt(userInput);
   
   int min = totalSec/60;
   int sec = totalSec%60;

   JOptionPane.showMessageDialog(null, totalSec + " seconds is "
                                 + min + " minutes and " + sec + " seconds");
}

In this code, every variable created is a local variable. userInput, totalSec, min, and sec have all been defined within the method. Had they been defined outside the method, they would count as instance fields.

Parameters are Local Variables Too

Parameters technically are declared in the method header, but they still count as local variables. They are created to be temporary placeholders for the incoming data.

public class Student
{
   private String name;
   private double gpa;

   public Student(String n, double g)
   {
      name = n;
      gpa = g;
   }
}

In the code above, n and g are parameters of the constructor, but they are also considered to be local variables.
