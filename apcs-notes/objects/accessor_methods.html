Accessor Methods

Methods are how we interact with our constructed objects. Since the instance fields are private, there is no way of getting any information out of the object. Any time you would like to have access to some information about the object, you need to design a method to do it for you.

These methods are going to be non-static because they are meant to be an action that the object can do. Since they are a part of the object, they have access to the instance fields of the object and can decide what to do with that information.

Each accessor method is in charge of producing a piece of data and returning it.

Basic Accessor Methods

One thing that we lose by making our instance fields private is the ability to access them directly.

Name n = new Name("Quentyn", "Martell");
System.out.println( n.last ); //<--Won't compile. Cannot access private field.

With objects like this, there isn't much point to putting the data into an object if we can never access it again. The basic accessor method's purpose is to be the method based solution to the access problem. They are a method that is simply in charge of producing a value stored in the instance fields.

Consider the Name class that we used above:

public class Name
{
   private String first, last;

   public Name(String f, String l)
   {
      first = f;
      last = l;
   }
}

These variables are trapped, we'll create a method do get them out. They should have the following method headers:

public String getFirst()
public String getLast()

When designing methods, you need to think about how they function in order to get the method header. These method's are simply meant to return the appropriate instance field.

Return Type: This method is in charge of returning one of the names, both of which are Strings.

Parameters: This is where we ask ourselves if we need any extra information to complete this method. Since the only information we need is already in the instance fields, no parameters are necessary.

Function:Our job is to return the value of the first/last variable, and since the method has access to those fields it can simply return it.

Consider the updated Name class below:

public class Name
{
   private String first, last;

   public Name(String f, String l)
   {
      first = f;
      last = l;
   }


   public String getFirst()
   {
      return first;
   }

   public String getLast()
   {
      return last;
   }

}

Now this code can run:

Name n = new Name("Quentyn", "Martell");
String f = n.getFirst();
String l = n.getLast();
System.out.println("First Name: " + f);
System.out.println("Last Name: " + l);

Adding the getFirst and getLast methods has given our Name objects the ability to tell us the information that they have stored, which makes them more useful. Otherwise the values would be trapped within the object and we wouldn't be able to retrieve them.

Non-Basic Accessor Methods

Accessor methods don't have to be as simple as simply returning a value. One of the powerful thing about objects is their ability to produce new information that might be useful to the programmer using the object.

The Name class above holds a person's first and last name. One simple thing that might make a Name object useful is the ability to produce the name's initials.

public String getInitials()

To figure out the method header, you have to think about how this method is supposed to function.

Return Type: This method is in charge of returning the initials of its name, which we would expect to be something like "I.S." or "Q.M.". The return type is a String

Parameters: This is where we ask ourselves if we need any extra information to complete this method. In order to make the initials, we need to know the first and last name. Since those are instance fields, we need no extra information. There are no parameters.

Function:Our job is to return the initials of the name. This will involve substringing out the first letter from the first and last name, both of which we have access to.

Consider the updated Name class below:

public class Name
{
   private String first, last;

   public Name(String f, String l)
   {
      first = f;
      last = l;
   }

   public String getFirst()
   {
      return first;
   }

   public String getLast()
   {
      return last;
   }

   public String getInitials()
   {
      String firstInit = first.substring(0,1);
      String lastInit = last.substring(0,1);
      return firstInit +"."+ lastInit +".";
   }

}

You can see that the getInitials method has access to all of the information that it needs inside the Name class, that is why it's a good choice for a Name method.
