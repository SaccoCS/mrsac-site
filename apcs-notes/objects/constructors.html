Constructors

Every object contains a special method called the constructor, which is responsible for initially setting up the object. The constructor is always the first method that an object runs, and its job is to make sure that the instance fields have their starting values.

Consider the Student class below

public class Student
{
   private String name;
   private double gpa;

   public Student(String n, double g)
   {
      name = n;
      gpa = g;
   }
}

The constructor is this method

public Student(String n, double g)
{
   name = n;
   gpa = g;
}

As you can see, the constructor is concerned with is assigning values to the instance fields. In this case, both are assigned to values that come from the parameters, but it could be more complicated.

When it's time to construct an object, the constructor is invoked using the word new.

Student s = new Student("Joffrey Baratheon",2.65);

Constructing is the only way to create new objects. Whenever it's time to construct an object you should be wary of what parameters the constructor requires.

The Rules

Since the constructor is a special method in an object, it has very specific rules for you to follow. The constructor's header is the only way that Java knows that it's a constructor, so if you're not careful, you might accidently make a method that never runs. In general, constructor headers follow this structure.

public ClassName( Parameters )

The constructor is named EXACTLY the same as the class

The name of a constructor is not something that you get to be creative with. It must be named after the class it's constructing. In the example above, the class was named Student so the constructor had to be named Student. Otherwise it would just be a plain-old method.

The constructor has no return type

The constructor is the only method that you don't provide any return type. This does not mean that the return type is void.

public Student(String n, double g)      <-- A valid constructor for Student
public void Student(String n, double g) <-- A normal method that is named Student

Putting in the method header causes Java to no longer recognize the method as a constructor. It is now confusing because it is considered a normal method whose name happens to be the same as a constructor.

The constructor has a specific job

The constructor is special because it is the method that runs before all other methods. Its job is to assign the initial values to the instance fields so that they are ready for the other methods to use. If you find that some of your other methods are crashing because of the value of the instance fields, often the cause is an improper constructor.

Example

Consider the Month class below.

public class Month
{
   private String monthName;
   private int numDaysInMonth;
}

The way that this object is designed, every Month object keeps track of a name and number of days. One example of a Month would have a name of "January" and number of days of 31. The problem is that there is no easy way to set up those instance fields. Right now there is no constructor, so the default constructor is used

Month m = new Month();

This will compile, but afterwards we're left with no useful information in the instance fields. m.monthName would be null, and the number of days would default to 0. What we want is to be able to do the following:

Month m1 = new Month("January",31);
Month m2 = new Month("February",28);

If the constructor allows us to provide information at the creation of the object we will be able to easily set up the initial values of the objects. Just looking at the example above we can see that one of the Months is January and has 31 days, while the other one is February and has 28. This helpful constructor is not automatically created, we have to design it.

The Header

It helps that we have an example of what we want our constructor to be. Looking at the code above, we want a constructor whose first parameter is a String representing the name, and the second parameter is an int to represent the number of days. This means that the header for this new constructor would be:

public Month(String mName, int numDays)

The order is important. In order to compile the types must match how it's used in the runner.

The Purpose

Constructors generally have the same purpose; to assign initial values to the instance fields. In the Month class the two instance fields are monthName and numDaysInMonth, so values should be assigned to them. In this case, all the appropriate values are in the parameters.

public class Month
{
   private String monthName;
   private int numDaysInMonth;
   public Month(String mName, int numDays)
   {
      monthName = mName;
      numDaysInMonth = numDays;
   }
}

Make sure you note that there are two different variables for each type of information. One is the parameter, which is temporary and will disappear after the constructor finishes, and the other is the instance field which exists for the entire life of the object. Always make sure that the instance field is receiving the value.
