State and Mutator Methods

An object's state is a general term for the values stored in the instance fields of an object. For example, consider the Student class, which holds a student's name and gpa.

public class Student
{
   private String name;
   private double gpa;

   public Student(String n, double g)
   {
      name = n;
      gpa = g;
   }
}

This class itself has no state. It is simply the blueprint for creating Student objects. The object has a state when it's constructed.

Student stu1 = new Student("Zack Morris", 1.85);
Student stu2 = new Student("Jessie Spano", 4.8);

The constructed objects stu1 and stu2 are the same type of object, but they have different states. One is named Zack and has a low gpa, while the other is named Jessie and has an excellent gpa.

Changing an Object's State with Mutator Methods

Sometimes an object is designed to have a state that isn't permanent and is meant to change as the object gets used. Consider the stu1 object above. The current state has a very poor gpa.

  • stu1
  • name: "Zack Morris"
  • gpa: 1.85

Your GPA is not permanent until you graduate. It gets updated every time a new set of grades comes in. This means that when designing the Student object we should include a way to change the state of the object by modifying the value of one of the instance fields of the object. These objects are known as mutator objects because they mutate (change) the object.

Basic Mutator Methods

A basic mutator method is a method that takes in a value and simply sets an instance field to be that value. Let's add a method to the student class that allows us to replace a GPA with a new value.

public void setGPA(double newGPA)

When designing methods, you need to think about how they function in order to get the method header. These method's are simply meant to replace the appropriate instance field with the value given as a parameter.

Return Type: This method's job is not to help the object produce anything. It's entire purpose is internal, so it will not be returning any value. We still need to put something as the return type, so we use the special keyword void to show that we don't intend on returning anything.

Parameters: This is where we ask ourselves if we need any extra information to complete this method. The purpose of this method is to set the gpa to a new value. We need the user to tell us what that new value should be. In the parameter section, we crated the newGPA variable to represent the double that the user provides.

Function:Overall, our job is to make sure that the state of the object changes. This means an instance field needs to change in some way.

Consider the updated Student class

public class Student
{
   private String name;
   private double gpa;

   public Student(String n, double g)
   {
      name = n;
      gpa = g;
   }

   public void setGPA(double newGPA)
   {
      gpa = newGPA;
   }


}

There is only one line to this method and it doesn't return anything. The purpose of this method is to change the state of the object, meaning changing the value of one of the instance fields. If you look, the one thing that the method does is assign a new value to the gpa instance field. That new value comes from the parameters of the method.

Student stu1 = new Student("Zack Morris",1.85);   //Creates the object with 1.85
stu1.setGPA(2.0);   //Changes the object's gpa to 2.0

This code first creates an object with one state, and then changes that value using the new setGPA method that we just added.

Parameters

Remember that when a method relys on extra information to complete its task we will need to provide a parameter. This means defining a temporary variable that will hold whatever is passed in when the method is run. For the setGPA method we already have access to the instance fields, we just need to know what the user wants the NEW value of the gpa to be. This is why the newGPA double is created.

public void setGPA(double newGPA)
{
   gpa = newGPA;
}

The newGPA variable is created every time that the setGPA method is called.

stu1.setGPA(2.0);

The value provided by the code calling the method is transferred into the parameter. In this example, the 2.0 provided gets transferred to the newGPA variable.
