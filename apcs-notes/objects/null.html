Object Variables and Null Values

Object Variables are Parking Spaces

Consider an object variable like a parking spot. When a parking space is created it is designed to hold a specific type (a car), but it can hold any object of type car. The car could have data: [red paint, 14.8 gallons in tank, Ford], but the only thing important to the space is that it's a Car.

Variable names are like the numbers on the parking space. If I ask you to honk the horn of the Car in space 127, it wouldn't matter what type of car was in the space. You would simply go to that location and honk the horn (space127.honk()). Depending what car is parked there it might sound different.

Null is an Empty Space

Just like with a parking space, object variables can be empty. In Java that's means the object has a null value. Consider the two lines of code below.

Car c1 = new Car("Ford","Mustang");
Car c2 = null;

This code creates two variables, but only one Car object. This is like having a two-car garage and only one car in it.

Null is Legal to Have, but Dangerous to Use

Assigning a null value to a variable is 100% legal to do, and can be useful to advanced programmers. The problem comes when trying to use a value that is null.

With the example above, if I told you to honk the horn in space c1, you would run out and I'd hear the horn of a Ford Mustang. If I then asked you to honk the horn of the Car in space c2 we'd have a problem. It's not that the request is unusual, it's that there is no car there to perform the action on.

This happens in programming too.

Car c1 = new Car("Ford","Mustang");
Car c2 = null;

c1.honk();
c2.honk();    //CRASH!!! NullPointerException

The code above will crash when it's run, but not until the very last line. The error thrown is a NullPointerException

Why Doesn't the Compiler Catch Null Problems?

You have to trust Sacco when he says that null values can be useful in more advanced programming. The compiler looks at each line an judges if it's something that can be done. It does not look too deep into the code.

The code c2.honk() is seen by the compiler as asking a car to honk, which is perfectly valid. The compiler doesn't check to see that the variable was left without an object in it.
