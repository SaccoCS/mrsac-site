Order of Operations

When doing Math with Java, there is an order of operations that will be followed. It's not exactly the same as your Dear Aunt Sally told you.

PCMMDAS

Parentheses
Casting

Multiplication
Modulus
Division

Addition
Subtraction

Note that there are no exponents in Java, and that Casting and Modulus are worked into the equation. This should be somewhat familiar from Math class.

15 + ( 2 + 3 ) * 6      Parentheses
    15 + 5 * 6          Multiplication
     15 + 30            Addition
        45

Similar Operations are Processed Left to Right

Multiplication, Mod, and Division are considered similar operations, as well as Addition and Subtraction. When there is more than one similar operation, they are processed from left to right.

8 - 2 * 6 % 9        Multiplication
  8 - 12 % 9         Mod
     8 - 3           Subtraction
       5

If you do things out of order, it can change things drastically. Observe the sameexpression, but with the Modulus evaluated first

8 - 2 * ( 6 % 9 )    Mod
  8 - 2 * 6          Multiplication
     8 - 12          Subtraction
      -4

Mixing Data Types

Order of Operations is very important when mixing different data types because when you perform an operation on two different types of data, one of the elements is converted to the other.

- A String plus an int results in a String (concatenation)
- A String plus a double results in a String (concatenation)
- An int and a double results in a double

For Example, the following expression mixes ints, doubles, and Strings

1 + 2.5 + "A" + 3 + 4       int and double = double
3.5 + "A" + 3 + 4           double and String = String
"3.5A" + 3 + 4              String and int = String
"3.5A3" + 4                 String and int = String
"3.5A34"

Before the order of operations reaches the String "A", it is only looking at ints and doubles, so the addition is mathematical and results in a double, but once the value being looked at is a String, the addition becomes concatenation and the numbers a simply added to the end of the String.

Casting

Casting is very high up on the order of operations.PCMMDAS. The only thing that supercedes it is parentheses. Look at how each different cast causes that value to be truncated and affects the value in the end.

double a = 2.2;
double b = 4.5;
double sum1 = (int)a + b;   // 2 + 4.5 --> 6.5
double sum2 = a + (int)b;   // 2.2 + 4 --> 6.2
double sum3 = (int)(a + b); // (int)(2.2 + 4.5) --> (int)(6.7) --> 6
double div1 = (int)b/2;     // 4/2 --> 2
double div2 = (int)(a + b)/2;// (int)(2.2+4.5)/2 --> (int)(6.7)/2 --> 6/2 --> 3
