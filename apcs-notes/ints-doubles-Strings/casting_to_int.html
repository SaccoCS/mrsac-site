Casting to an int

The main differences between an int and a double is that doubles are built to handle the decimal portion of a number. While any int may be stored as a double (3 becomes 3.0, 99 becomes 99.0, etc.), not every double may be stored as an int.

int num = 3.14159      // <-- Error: Possible Loss of Precision!
double value = 2;
int result = 5*value;  // <-- Error: Possible Loss of Precision!

The compiler will stop you any time you try to assign a double to an int variable. It doesn't matter if the number 'should be a round number' like 10.0. The compiler rejects is simply because of the type.

Casting: Sort of a solution

Casting is how we can temporarily change the type of a value. When casting a double to an int, casting means that the decimal portion of the number.

Casting is an awkward operation. You do it by typing (int) directly in front of the double.

double pi = 3.14159;
int almostPi = (int)pi;

Whenever you cast, you are aknowledging that in order to turn the double into an int it must be truncated, meaning the decimal value will be lost. Note that this is not the same as rounding. The integer part of the number never changes.

Order of Operations

Casting is very high up on the order of operations.PCMMDAS. The only thing that supercedes it is parentheses. Look at how each different cast causes that value to be truncated and affects the value in the end.

double a = 2.2;
double b = 4.5;
double sum1 = (int)a + b;   // 2 + 4.5 --> 6.5
double sum2 = a + (int)b;   // 2.2 + 4 --> 6.2
double sum3 = (int)(a + b); // (int)(2.2 + 4.5) --> (int)(6.7) --> 6
double div1 = (int)b/2;     // 4/2 --> 2
double div2 = (int)(a + b)/2;// (int)(2.2+4.5)/2 --> (int)(6.7)/2 --> 6/2 --> 3
