Getting ints and doubles from a String

Often times when we are getting information from the user it comes in as a String. When a program calls for an int or a double, the number that the user types is still stored as a String. This is a big deal because Strings don't do math, no matter what characters they have.

String numStr = "125";
int numInt = 125;

Both of the variables above represent 125, but in different ways. The int num is a number that can be added, subtracted, or multiplied, but the String numStr is not a number. It is a String of characters with no mathematical value that happen to represent an int.

Because Java is very concerned with data type, the compiler will stop you if you casually try to do math without considering the type. Consider the code below, which is an attempt to print out five less than the value of each variable.

System.out.println( numStr - 5 ); // <-- WILL NOT COMPILE
System.out.println( numInt - 5 );

The first println will not compile simply because the numStr variable is a String. Java sees numStr - 5 as a String minus an int, meaning it could be "Hello World" - 5, which makes no sense. The fact that the value in the numStr String has numeric characters is irrelevant to the compiler. Type is important.

Converting a String to an int

Luckily, if you know that you have a String that only has numeric characters there is a method that will help you convert it to the type you need. It is a static method in the Integer class named parseInt. As a parameter it takes in a String and returns an int. Consider the code below.

String numStr = "125";
int num = Integer.parseInt(numStr);

The parseInt method has the String "125" passed in as a parameter. It looks at each character an figures out what the actual numeric value is and returns it as an int. From there, the num variable can have math operations performed on it.

Watch for crashes with a NumberFormatException

One important thing to know is that not all Strings have an int stored in them. This can cause a problem. Consider the code below.

String str = "Hello Mr. Sacco";
int num = Integer.parseInt(str);

In this situation the str variable is in no way a number. What int would you expect to be returned? It's an impossible question to answer. This will cause the code to crash with a NumberFormatException. As the programmer, you are responsible for making sure that you only provide Strings that only contain ints.

Converting a String to a double

The parseInt method from the previous section will crash if it is not given a String that contains an int. This means that the String "Sacco" will crash the method, but also the String "3.14", which is also not an int (anything with a decimal point is a double).

Luckily, there is a method that performs the same operation, but converts to doubles.

String str = "3.14";
double dubNum = Double.parseDouble(str);

Whenever you have a String that you want to turn into an int, make sure that you know whether or not you need a double or an int. Choosing the wrong one may cause a crash.

Getting a Number from the User

Getting a number from the user is a multi-step process. First you get the user's input as a String, and then you call the appropriate parsing method to get the value that you actually want.

String userInStr = JOptionPane.showInputDialog("Enter an amount of money");
double cash = Double.parseDouble(userInStr);

These two lines work together to get specifically a double from the user that we can then do math with. For an int, simply choose the Integer class method.

String userInStr = JOptionPane.showInputDialog("How many hamburgers?");
int numBurgers = Integer.parseInt(userInStr);

You should always make sure that the information you take from the user turns into the appropriate type.
