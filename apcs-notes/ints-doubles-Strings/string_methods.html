String Methods

Strings are one of the very first objects that we're going to get acquainted with. They not only are able to store all the characters that we give them, they also have a lot of methods that are useful for making changes and generating new Strings.

Object Methods are Called Using the Variable

One important difference between static methods and object methods is that when calling the object methods, you ask the individual object to do the work, not the class. For example, Strings have a length method that returns the length of a String. Look at the following code, which calls the length method in two different ways.

String firstName = "Ian";
String lastName = "Sacco";
System.out.println( String.length()); // <-- Will not compile
 System.out.println( lastName.length() );

Try to think of why the first println doesn't make sense. It's asking for the length of a String, but which one? Since the method takes in no parameters, there is no way that we can tell which String the programmer wants to know the length of. The second line, however, is asking the String stored in the lastName variable to do the work. Since lastName knows that it holds a String of "Sacco", it can tell me that its length is 5.

toUpperCase and toLowerCase

public String toUpperCase()
public String toLowerCase()

The purpose of the toUpperCase/toLowerCase method is to generate a String that is the all upper or lower case version of the original String.

String str = "Hello World!!!";
String upperStr = str.toUpperCase();
String lowerStr = str.toLowerCase();
System.out.println("Original: "+str);
System.out.println("UPPER: "+upperStr);
System.out.println("lower: "+lowerStr);

Output:
Original: Hello World!!!
UPPER: HELLO WORLD!!!
lower: hello world!!!


Notice that the original String is not affected by calling either method. Strings are immutable, meaning they can only generate new Strings, never modify the existing one.

length

public int length()

The length method is useful for telling how long a String is.

String str = "Mr. Sacco";
int len = str.length();
System.out.println(str " has a length of " + len);

Output:
Mr. Sacco has a length of 9


Notice that the length of a String includes the punctuatinos and spaces. They count as characters in the String even though they aren't letters.

length vs index

Inside of a String, every location has an index assosciated with it.

Sacco
01234

It is important that you remember that the labeling for indices starts at 0. It is an efficient way of labeling, but causes some problems for some people who are casual. Many students who are told that the length of a String is 5 would could assume that the first character is at index 1 and the last is at index 5. In reality, the first character is at 0 and the last index is at 4.

Assuming that the length of a String represents an index is a common error and can cause crashes.

Remember: The last index is always one less than the length.

substring

public String substring(int start, int end)

The substring method is frequently used in String manipulation when you want to 'chop' off a piece of a String. The parameters that you need to provide are the starting and stopping indices of the String. For example, if the String str contains the String "Sacco" and the substring method was called with a starting index of 1 and an ending index of 4.

String str = "Sacco";
String sub = str.substring(1,4);

Sacco
01234

The value of sub would be "acc" after the code above runs.

Warning: Notice that "Sacco" has an "o" at index 4, but it is not a part of the substring, even though the method call put 4 as the ending index. When using the substring method, the ending index will not be included. It seems strange, but the parameters are inclusive on the starting index and exclusive for the ending index. For example, if you have a String str, and you want to get the first letter of a String, which is at index 0, you would put the following.

String firstLetter = str.substring(0,1);

Since the first letter of a String always falls at index 0, we put index 1 as the stopping index. This is counter intuitive to a lot of people who think that index 1 will be included, but it's more of a "go up to, but don't include" index.

Alternate substring

public String substring(int start)

There is a second version of the substring method that only takes in a single parameter. When this version of the method is used, the substring returned will start at the given index and go all the way to the end of the String.

String str = "Sacco";
String sub = str.substring(1);

Sacco
01234

The value of sub would be "acco" after the code above runs. It started at index 1 and went all the way to the end. This is identical to saying str.substring(1,5), it just takes less typing.

indexOf

public int indexOf(String toFind)

The indexOf method is a searching method that is in some ways the opposite of substring. For this method you must know the String that you're looking for and the method provides the index of that String.

String phrase = "Mister Sacco is a very strange man";
int saccoIndex = phrase.indexOf("Sacco");
System.out.println("Sacco is at index "+ saccoIndex);

Output:
Sacco is at index 7

Note that the indexOf method can only return a single int. Even though "Sacco" occupies indices 7,8,9,10, and 11, only the first index is returned.

What happens when indexOf can't find the String?

The indexOf method always returns an int, even if it can't find the given String.

String phrase = "I like candy";
int saccoIndex = phrase.indexOf("Sacco");
System.out.println("Sacco is at index "+ saccoIndex);

Output:
Sacco is at index -1

Since "Sacco" is not found in the String "I like candy", indexOf is in a bind. It cannot return numbers like 2 or 10, because that would imply it was found at those indices, which it wasn't. Instead, indexOf is defined to always return a -1 when it cannot find the given String. This is known as a sentinal value because the -1 is not an index; it's just a special value that indices something to the programmer.

You need to always remember that indexOf has a possibility of a -1, otherwise you will eventually strange crashes in your program.

Combining Method Calls Example

Generally when you're manipulating Strings you need to perform more than one method call to get the job done.

For Example: Assuming you have a String from the user in a variable str and you want to access the last letter of it. The problem is that the last letter doesn't always fall at the same location. In the String "Sacco" the last letter is at index 4, but in "computer" the last letter is at index 7. This location is dependent on the length of the String, so we should look for a pattern.

Work some simple examples:
"Sacco" has a length of 5 and the substring would be (4,5)
"computer" has a length of 8 and the substring would be (7,8)
"Java" has a length of 4 and the substring would be (3,4)
In general the substring starts at an index that is one less than the length. So our code should make sure to ask for the length of a String before it substring's out the last letter.

String str = //Some value from the user
int length = str.length();
String last = str.substring(length-1,length);

It's worth noting that this situation does not require the two-parameter version of the substring method. If you ever feel like a substring that "goes all the way to the end", user the version of substring that only takes in a single parameter. This means that the last line is better written as:

String last = str.substring(length-1);

The second parameter was unneccessary
