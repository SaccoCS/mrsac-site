Casting

Casting is an operation that allows us to temporarily consider a certain piece of data to be temporarily considered a different type. Casting counts as an operation, but doesn't have a symbol like +.-,*,%,/. Instead you put the desired type in parentheses ahead of the value that you want to change.

You cannot simply cast between any type that you want. For example if you want to turn a String to an int casting would not work. That's why we have the Integer.parseInt method. Two main uses of casting are to cast between ints and doubles, but they are useful in different ways.

Casting to a double to Avoid int Division

The following code is flawed.

int base = 3;
int height = 5;
double area = base*height/2;

The issue is that we accidentally assumed that all division was equal. Since there are only ints in the expression base*height/2, the result will be calculated using int division and the area will be 7.0 instead of 7.5 . Assuming that we still want the base and height variables to remain ints, we can use casting to fix our problem.

int base = 3;
int height = 5;
double area = (double)base*height/2;

By putting double in parentheses in front, we're telling java that the value immediately to its right should be considered a double for this calculation. This means that the base variable is considered a double when the division occurs, so the result will not be truncated.

Casting to an int to Force Truncation

The main differences between an int and a double is that doubles are built to handle the decimal portion of a number. While any int may be stored as a double (3 becomes 3.0, 99 becomes 99.0, etc.), not every double may be stored as an int.

int num = 3.14159      // <-- Error: Possible Loss of Precision!
double value = 2;
int result = 5*value;  // <-- Error: Possible Loss of Precision!

The compiler will stop you any time you try to assign a double to an int variable and tell you there's a 'Possible Loss of Precision'. It doesn't matter if the number 'should be a round number' like 10.0. The compiler rejects is simply because of the type.

Casting is how we can temporarily change the type of a value. When casting a double to an int, casting means that the decimal portion of the number.

Casting is an awkward operation. You do it by typing (int) directly in front of the double.

double pi = 3.14159;
int almostPi = (int)pi;

Be careful: When casting to an int, you are agreeing to lose some information. Make sure that an int is what you want.

Casting Comes Before Any Calculations

In the Order of Operations, casting comes very early ( PCMMDAS ). It will immediately 'convert' the type to its right unless you use parentheses to denote that you want to cast the result of some other operation. Look at how each different cast below causes that value to be truncated and affects the value in the end.

double a = 2.2;
double b = 4.5;
double sum1 = (int)a + b;   // 2 + 4.5 --> 6.5
double sum2 = a + (int)b;   // 2.2 + 4 --> 6.2
double sum3 = (int)(a + b); // (int)(2.2 + 4.5) --> (int)(6.7) --> 6
double div1 = (int)b/2;     // 4/2 --> 2
double div2 = (int)(a + b)/2;// (int)(2.2+4.5)/2 --> (int)(6.7)/2 --> 6/2 --> 3
