Modulus

You should already be familiar with most of the basic math operations +,-,*, and /. They are all performing operations that you've done before and the only weird one is int division.

The new operation that is commonly seen in programming is the modulus operator or 'mod' operator, which is represented with a percent sign %. It represents the remainder of a division. For example:

14 % 3 --> 2

The expression above is asking for the value of fourteen mod three. This is asking what the remainder is after taking away as many threes as possible. The fact that three goes into fourteen four times is unnecessary knowledge because all we need to know is that there is two remaining.

15 % 6 --> 3
27 % 2 --> 1
40 % 10 --> 0
6 % 4 --> 2
100 % 99 --> 1
55 % 22 --> 11

Just as with division, the two terms in a mod statement are named dividend and divisor.

Watch Out for Small Dividends

The basic concept of mod makes sense, but many people are confused when the dividend is smaller than the divisor.

3 % 6

The way to picture it is that the dividend, three, is the value that you start with. You then take away as many sixes as you can, which is none. The remaining amount is the three that you started with.

3 % 6 --> 3
12 % 14 --> 12
2 % 7 --> 2
10 % 15 --> 10

Every time the dividend is less than the divisor, the result is simply the dividend.

Patterns with Mod

Telling when a number divides evenly into another

When a number goes evenly into another number, the result of their mod operation is zero because there is no remainder

6 % 3 --> 0
12 % 4 --> 0
15 % 5 --> 0
100 % 10 --> 0

 

Odd or Even

Similarly, you can tell if a number is odd or even based on the result when you mod the number by two. Since even numbers are divisible by two, they will go in evenly and have a remainder of zero

4 % 2 --> 0
12 % 2 --> 0
2 % 2 --> 0
10 % 2 --> 0
88 % 2 --> 0

Odd numbers, not divisible by two, will always give a remainder of 1

13 % 2 --> 1
1 % 2 --> 1
11 % 2 --> 1
77 % 2 --> 1

Example Program

Mod is often useful when paired up with int Division. Division provides the quotient and mod provides the remainder. The program below is meant to ask the user for a large number of days, and then print out how many weeks and days that is.

public class NumWeeks
{
   public static void main()
   {
      String numDayStr = JOptionPane.showInputDialog("How many days?");
      int numDays = Integer.parseInt(numDayStr);
      int numWeeks = numDays/7;
      int leftoverDays = numDays%7;

      System.out.println(numDays + " days is " + numWeeks + " weeks and "
         + leftoverDays + " days");
   }
}

If the user were to input 25 days, they output would be:
25 days is 3 weeks and 4 days

The key to this program is knowing that there are 7 days in a week. Dividing by 7 will tell us how many weeks there are, but the mod will tell us how many days are levtover after all the weeks are removed.
