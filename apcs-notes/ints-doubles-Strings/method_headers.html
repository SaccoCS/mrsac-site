Method Headers

As we define and use our own custom methods it is important that we understand the structure of a method header. The header is only a few words long, but each word tells you something about how to use the method.

The general format of a static method header is:

public static returnType methodName(Parameter list)

The two important parts of a method header to pay attention to are the parameters and return type.

Parameters

When calling a method, the parameters are the extra information that methods require in order to run properly. For example, consider the two lines of code below, which are a common way of getting an int from the user.

String input = JOptionPane.showInputDialog("Enter a number");
int num = Integer.parseInt(input);

There are two methods being called here. First the showInputDialog method is called and retrieves a String from the user, and then the parseInt is used to turn that String into an int. Each of those method calls required the use of a String to operate.

The showInputDialog method's parameter is "Enter a number", which is so it knows what message to display. The parseInt method was called to turn a String into an int. The parameter, is the input variable that we want turned into the int.

Return Type

The return type of a method is the type of data that the method will generate.

int num = Integer.parseInt("642");

In the code above, the value of num is an int because the parseInt method takes in a String and returns that as an int. Since its return type is an int, it wouldn't make sense to assign it to a String or a double variable.

Return types are situational and depend on what the method's purpose is. The showInputDialog method, for example, returns whatever the user typed into a text box, so its return type is a String.

Method Headers vs. Method Usage

It's important to know both how methods are used and how they are defined. You should be able to look at how a method is used and determince what its header should look like. Let's use the parseInt method as an example.

int num = Integer.parseInt("642"); //<-- takes in a String and returns an int

You can see that this method takes int a String as a parameter and returns a value that's an int. The general formula is returnType methodName(Parameter list), so the method header for the parseInt method must be:

public static int parseInt(String str)

Note that the parameter's variable name is not very important. While it's class to name it something useful, we only really care about the type.

Reading a Method's Header Tells You How to Use It

When you're only given a method's header, you can read it to tell which types of data to use with it. For example, the showInputDialog method's header is:

public static String showInputDialog(String str)

We can see that both the return type and the parameter are a String, which is why we supply a String and also expect one to be returned when using the method.

String name = JOptionPane.showInputDialog("Enter your name");

When dealing with static methods, you start the method call with the name of the class that the method's located. In this case it is JOptionPane.

void

Not all methods are supposed to return a type. One common example is the println method that we frequently use.

System.out.println("Hello World!!!");

This style of method is to perform a useful action, but it does not generate and return any data, which is why you don't see the following in your code.

String str = System.out.println("Hello World!!!");

This will not compile because the println method doesn't return a String. In this situation, the word void is used in place of a specific return type. The println method takes in a String and returns nothing, meaning it has a String as a parameter and void as a return type.

public static void println(String str)
