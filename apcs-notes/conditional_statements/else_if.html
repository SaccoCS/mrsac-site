Expanding Our Choices with else-if

The basic if-else structure is useful for breaking your logic into two paths, but there are often situations that require more than two choices. In order to achieve this, we need to use an else-if to expand our choices.

Consider the situation where a car is pulled over for speeding in a 65 zone. In a strict situation, the logic is simple.

if( speed <= 65 )
{
   System.out.println("NO TICKET");
}
else
{
   System.out.println("TICKET");
}

But sometimes things aren't so black and white. Perhaps there's a police officer that feels that wants to give a warning for tickets that are still 70 or under.

A Nesting Solution

One way to solve this is to use a nesting solution. Instead of automatically assuming that anything entering the 'else' zone is a ticket, we can treat it as an opportunity to make a second decision. Since we already know that any speed that makes it here is over 65, we only need to check for values that are 70 and lower.

if( speed <= 65 )
{
   System.out.println("NO TICKET");
}
else
{
   if( speed <= 70 )
   {
      System.out.println("WARNING");
   }
   else
   {
      System.out.println("TICKET");
   }
}

There are now three different possible results. Anything under 65 will still print "NO TICKET", but now there is a difference between numbers like 68 and 72. Now numbers like 68 will print "WARNING", while 72 will print "TICKET".

One key thing to notice above is that even though one of the if statements checks if( speed <= 70 ), it does not mean that numbers like 45 will be a "WARNING". It is important to recognize how you get there. The way this is written, we must first check if the speed is <= 65. By the time we check if the speed is less than 70, we already know that it's greater than 65.

An else-if Solution

If you follow the above strategy, you can keep nesting more and more to add more 'choices' for your if statement. You simply need to keep nesting inside of the previous else. The problem is that this becomes a mess of curly braces very quickly. A much more common way to represent this would be to use an else-if.

if( speed <= 65 )
{
   System.out.println("NO TICKET");
}
else if( speed <= 70 )
{
   System.out.println("WARNING");
}
else
{
   System.out.println("TICKET");
}

An else-if is similar to an if because it has another condition to check, but it's also similar to an else because it will only run if the previous condition is false. This structure will proceed in exactly the same way as before: First it will check speed <= 65, after that it will check speed <= 70, and only if both of those are false will it print "TICKET"

We can use else ifs to help expand our choices to as many as we need. As long as we stick to the structure of if, else-if, else, we can guarantee that only one portion of the if statement will run.

Using the Structure to Your Advantage

It is important to understand what assumptions can be made when you use an else if. By the time you get to any else statement, you've already determined that at least one thing is false. Sometimes you can use this to your advantage.

Assume that you want to write a program that will look at a person's age along with the day of the week. If it's a weekday, the program should print out either "Go to work" or "Go to school" based on their age. If it's a weekend, everybody will get the message to "RELAX".

The solution below performs the proper task, but in an inelegant way.

if( age <= 18 && !(day.equals("Saturday") || day.equals("Sunday")))
{
   System.out.println("Go to School");
}
if (age > 18 && !(day.equals("Saturday") || day.equals("Sunday")) )
{
   System.out.println("Go to Work");
}
if (age <= 18 && (day.equals("Saturday") || day.equals("Sunday")) )
{
   System.out.println("RELAX");
}
if (age > 18 && (day.equals("Saturday") || day.equals("Sunday")) )
{
   System.out.println("RELAX");
}

This code breaks up the four possible situations very literally, spelling out the exact logic for each age and day. Another way to think about this problem would be to see that all weekend results are the same and to start the if statement with that. This will allow each subsequent else to be written with the assumption that it is a weekday without explicitly stating it.

if( day.equals("Saturday") || day.equals("Sunday"))
{
   System.out.println("RELAX");
}
else if (age > 18 )
{
   System.out.println("Go to Work");
}
else
{
   System.out.println("Go to School");
}

Note that the 'else' of the example represented a young person on a weekday, but we do it purely by first checking if it's a weekday and then if they're old. This leaves nobody left for the 'else' but the young peouple on a weekday.
