The boolean Primitive Type

A boolean variable is one that can only hold true or false. It is one of the simplest pieces of data that you can have because it is only really representing one bit of data, meaning a single 1 or 0. In some languages you even use a 1 for true and a 0 for false.

boolean b1 = true;
boolean b2 = false;

Be careful that you recognize the difference between the boolean value true, which has no quotes and "true", which is a String.

The Relational Operators: >, >=, ==, !=, <=, <

A very common way to generate a boolean value is using one of the Relational Operators, which look at the numbers on either side to generate a boolean value. The most recogniable being the greater than and less than symbols.

boolean b1 = 25.8 > 7;   // <--- true
boolean b2 = 30 > 99;    // <--- false
boolean b3 = 7 < 8;      // <--- true
boolean b4 = 30 > 30;    // <--- false

...or equal to

Though it may look funny, the two-symbol operators >= and <= are greater than/less than or equal to. We don't use the symbols ≥ and ≤ because they're not symbols that you can type on your keyboard.

boolean b4 = 30 > 30;    // <--- false
boolean b5 = 30 >= 30;   // <--- true

The line above can look a little strange with an equals sign and an assignment operator. Remember that the right side of the assignment operator is evaluated first, 30 >= 30 is true because they are equal values. That result is then assigned to the boolean variable to the left.

Equals and Not Equals

The two-character symbol == is the equals symbol, which is distinctly different than the assignment operator =. It will evaluate to true only if the two ints are the same value. The symbol != is the opposite, only evaluating to true when they are NOT equal (≠).

int a = 30;
int b = 25;
boolean b6 = 55 == a + b;    // <--- true
boolean b7 = 55 != a + b;    // <--- false

Again this code is a bit of a mess. First java will simplify the expression a + b, which gives a 55. Java will then evaluate 55 == 55 and get a true value, which is finally assigned to the b6 variable. The line after that has the same numbers, but when it reaches 55 != 55, it evaluates to false because they are not different values.
