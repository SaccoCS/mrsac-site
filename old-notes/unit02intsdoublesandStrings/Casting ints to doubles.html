Casting ints to doubles

Casting is an operation that will temporarily turn a variable of one type into another (providing they're compatible). 

When dealing with doubles and ints, there are problems that can arise when you have a number of one type and would prefer it to be a different type.  For example, consider the following code:

int numerator = 4;
int denominator = 5;
double result = numerator/denominator;
System.out.println(numerator + "/" + denominator + " = " + result);
Desired Output:
4/5 = 0.8		 Actual Output:
4/5 = 0.0

The problem occurs because numerator and denominator are defined as ints, so dividing them will result in int division.  There are some tricks that we could employ, such as changing it to say 1.0*numerator/denominator, but the classiest way to solve this problem is via casting.

 

How to cast:

The casting operation is a little different than we're used to.  In order to cast from an int to a double, a (double) must be put in front of the int that you want to temporarily change.

double result = (double)numerator/denominator;

In this case the numerator variable, which is an int, is being cast into a double.  This is only temporary.  It does not truly change numerator, it only makes it so that numerator is treated like a double during this line of code.

Now that numerator has been cast to a double, the division is no longer purely ints and will use double-style division.  The result variable will end up with the value 0.8.

Order of Operations:

Casting is very high up on the Order of Operations.  The only thing higher than it is parentheses.  The perfect line of code below:

double result = (double)numerator/denominator;

Can be ruined if you add a couple parentheses:

double result = (double)(numerator/denominator);

This statement now is telling Java to do the int division first, and cast it into a double afterwards, which was going to happen naturally anyway.
