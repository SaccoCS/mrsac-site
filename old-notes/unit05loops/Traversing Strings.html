Traversing Strings:

Loops are a powerful tool in programming.  One way that they are useful is when analyzing data and coming up with a new piece of data.

Strings are a Group of Data:

If you think about a String, you can start thinking of it as a data type that holds many smaller bits of data.  The String "Sacco" is actually five individual characters (their data type is char) organized by index.  This is the first group of data that we can traverse across.  To traverse a set of data is to loop trhough it and analyze it one piece at a time with the hopes of determining something new.

For example, consider the countTheAs method below:

public static int countTheAs(String str)

The purpose of this method will be to return an int that represents the number of lowercase A's appear in the parameter String.

 

One Letter at a Time:

In order to determine how many a's there are, were going to have to substring out every single letter and check to see if its an "a".  Without loops we might attempts the following:

String first = str.susbtring(0,1);
String second = str.substring(1,2);
String third = str.substring(2,3);
String fourth = str.substring(3,4);
etc...

This would only work if we knew exactly how many letters were in the String, but we don't.  Look at the pattern in the substrings: the two values are exactly one apart, and each time the starting index is different.  This is what loops are good at doing.  Consider the following substring, which uses an int i instead of a set value.

String letter = str.substring(i,i+1);

When i is 0, the result is the first letter, when it's 1 the result is the second letter, 2 is the third letter, etc.

 

Loop the entire String:

We have a substring that can grab the letter at a value i.  Now we just need a loop that will make i be every index that a String has.

Remember:  A String's first index is 0, and the last one is one less than the length of the String.

for(int i = 0; i < str.length(); i++)

Notice the use of str.length().  We don't know how long the string is, but it doesn't matter.  This loop is set up so that the last value of i is one less than the length of the String.  This prevents the out-of-bounds issue that occurs. 

for( int i =0; i<str.length(); i++)
{
    String letter = str.substring(i,i+1);
    System.out.println(letter);
}

In the code above, assume that str is "Sacco". 

- The first iteration of the loop has i being 0.  This makes the substring grab a letter at index 0, which is "S".  It's printed 
- The second iteration of the loop has i being 1.  This makes the substring grab a letter at index 1, which is "a".  It's printed 
- The third iteration of the loop has i being 2.  This makes the substring grab a letter at index 2, which is "c".  It's printed 
- The fourth iteration of the loop has i being 3.  This makes the substring grab a letter at index 3, which is "c".  It's printed
- The fifth iteration of the loop has i being 4.  This makes the substring grab a letter at index 4, which is "o".  It's printed

The loop stops when i hits 5, because that's the length of the String.  Had we kept looping, the program would have crashed.

The result here is that each of the letters is printed out one at a time.

S
a
c
c
o

 

Counting the As:

Now that we can look at every letter of the String individually, it should be easy to count the a's.  To do this we will need a counter variable to accumulate the number of "a" characters that we come across.  We will declare it to be 0 at the beginning, and then increment it every time that we find an "a".  

 
public static int countTheAs(String str)
{
   int count = 0;
   for( int i =0; i<str.length(); i++)
   {
       String letter = str.substring(i,i+1);

       if(letter.equals("a"))
          count++;
   }

   return count;
}

 

If the method above was run with "alphabet" as the parameter, the returned value should be 2.

 

When traversing a set of data:

Let your looping variable represent the index of the data.
Access one piece of data at a time to analyze it.
