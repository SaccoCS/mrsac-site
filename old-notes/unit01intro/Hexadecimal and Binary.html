Hexadecimal and Binary
The Rule of Four

Hexadecimal is used specifically because it works so nicely with Binary. All you need to know is the Rule of Four

The Rule of Four (not an official rule, just a Sacco-ism) states that each digit in a Hexadecimal number is directly related to exactly four corresponding bits in the Binary number.

To understand it, look at the following Hexadecimal number and its Binary representation:

7A3F16
1111010001111112

Altogether it looks like it would take a lot of math in order to convert between them (the number is 31,295 in decimal) it gets easier if you break it up into groups of four:

 7      A      3      F
0111   1010   0111   1111

Notice the red zero, which was added to 'fill out' the Binary number so that it had enough. You always add the extra zeros to the front, because adding them there won't change the value. Adding zeros to the end of the number would cause it to change the value of the number.

Looking at the numbers grouped in this way, you may be able to see the correllation between the Hexadecimal digit and the four corresponding bits.

7 in binary is 0111
A in binary is 1010
3 in binary is 0011
F in binary is 1111


This is always true for every Hex/Binary number and it makes converting between the two much simpler. When broken up using the Rule of Four, the largest number that you would every have to convert to binary is 15.

Converting from Hexadecimal to Binary

2B10D16

The rule of four says that all we have to do is convert all five Hex digits into four bit binary numbers. For the digits larger than 9, convert them to decimal first

2 is 0010
B is 11, which is 1011
1 is 0001
0 is 0000
D is 13, which is 1101

New you just have to put them together in order. It is proper to put spaces between every four bits in a binary number

0010 1011 0001 0000 11012

Converting from Binary to Hexadecimal

11100111100102

In order to use the Rule of Four, we must first break the number into groups of four bits. Since there are 13 bits in the number we'll add zeros to the front.

0001 1100 1111 00102

Now we just have four smaller conversions to make.

0001 is 1
1100 is 12, which is C
1111 is 15, which is F
0010 is 2

The resulting Hexadecimal number is 1CF216
