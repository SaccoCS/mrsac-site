Object Methods

One key characteristic of objects is that they have methods that belong to the object.  This means that once the client/runner has an object, it can call its methods to work with the instance fields.

 

There are two major categories of methods:

- Accessor Methods: These methods don't change the instance fields, but return some information about the object

- Mutator Methods: The purpose of these methods is to change the instance fields in some way.  Usually nothing is returned from these methods.

 

Method Headers:

When defining method headers, you must first know three things:

- The name of the method
- The return type of the method.  You may only have ONE
- Any Parameters that the method might require.  There can be as many as you want, just separate them with commas.

The method header goes in the following order

public returnType methodName( Parameter List )

While it is legal for methods to be private, our methods will always be public.

 

Return Type:

The return type is is type (String, int, double, or other object) that is expected when the method is called. 

For example:  We use String's indexOf method to locate a String within another String.

String str = "Hello World!!!";
int index = str.indexOf("!");

After this code has run, the value 11 will be stored into the index variable.

Within String, indexOf's method header looks like the following:

public int indexOf(String s)

The return type is an int, which makes sense because its job is to return an index, which is always an int.

 

How does this affect the client

As the person using the indexOf method, the return type tells me what type of variable to assign the result of the method to.

int index = str.indexOf("!");

This would not work if index was anything but an int.

 

 

Parameters:

The parameters of the method are the values that hold the extra information that the method might need in order to complete its task.

For example:  We use String's substring method to get a portion of a String.

String str = "Hello World!!!";
String small = str.substring(0,5);

After this code has run, the value "Hello" will be stored in the small variable.

Within String, substring's method header looks like the following:

public String substring(int start, int end)

Unlike return types, methods can accept as many parameters as it wants.  The substring method has two parameters, a starting index and an ending index.  Those are necessary pieces of information in order for the method to function.  Please note that the substring method is a part of the String class, so it already has access to the characters stored in its instance fields.  The indices provided are extra.

 

 

How does this affect the client

As the person using the indexOf method, the return type tells me what type of variable to assign the result of the method to.

String small = str.substring(5,11);

This would not work if you passed anything but an int into the parameter area.

String small = str.substring("Hello", "World");

 

 

Example:

Consider the Student class below:

public class Student
{
   private String name;
   private double gpa;
   
   public Student(String n, double g)
   {
      name = n;
      gpa = g;
   }

   public String getName()
   {
      return name;
   }
   
   public double getGPA()
   {
      return gpa;
   }

   public void setName(String n)
   {
      name = n;
   }

   public void setGPA(double newGPA)
   {
      gpa = newGPA;
   }
}

 

The getName method is a basic accessor method.  Notice that the return type is a String and that there are no parameters.  This means that the following code could be in the runner:

Student s = new Student("Gerald I. Joe",4.2);
String n = s.getName();

Notice that there are no parameters and that the returned value is caught with a String variable.

 

The setName method, on the other hand, has no return type and a String as a parameter.  This means that the following code could be in the runner.

Student s = new Student("Gerald I. Joe",4.2);
s.setName("Steve");

Notice that we are required to provide a String ("Steve"), and that there is no variable to 'catch' a value because the return type is void so there is nothing to 'catch'.
