Instance Fields

One way to think about an object is to consider it a variable that has multiple pieces of data bundled together.  Those pieces of information are known as the instance fields or instance variables.  For Example:  Consider the following class named Student and it's designed to have two instance fields, one to hold a student's name and one to hold its gpa.

public class Student
{
   private String name;
   private double gpa;
   
   public Student(String n, double g)
   {
      name = n;
      gpa = g;
   }
}

What this means in the Runner:

The variables name and gpa are the instance fields, meaning when a client class constructs a Student object, it in turn is creating two variables, a String and a gpa:

Student stu = new Student("Gerald I. Joe", 3.85);

The Student variable stu has a name of "Gerald I. Joe" and a gpa of 3.85

 

 

Instance fields should be private:

Yes, it is technically legal to have public instance fields.  In this case you would be able to access the instance fields directly

Student stu = new Student("Gerald I. Joe", 3.85);
stu.gpa = 4.0;

Even though this compiles, we are going to control the access of our instance fields by making them private.  This will mean that later we should add methods to the Student class in order to help access the variables.
