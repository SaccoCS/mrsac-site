State

The state of the object is a general term used to describe the values that are currently in its instance fields.  Usually it is spoken about when discussing changes that are made to the object by some mutator method.

For example:  A Car object starts with 0 gallons in the tank.  One might describe the state of the Car as "out of gas", but you rarely need to be specific about the values.  By calling the addGas method the gas instance field of the Car object will change from 0 to some new value, thus the addGas method has changed the state of the object.

 

Mutator Methods Change the State:

State is usually discussed when writing mutator methods, because the goal of a mutator is the change the state of the object in a certain way.  If the instance fields don't change, the state of the object stays the same and has not been properly mutated.
