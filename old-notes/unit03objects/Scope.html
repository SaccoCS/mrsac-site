Scope

In object design, different parts of the object have access to different sets of variables.  If a method has access to a variable, we can say that it is within the scope of the variable.  When writing a method, you must be careful to think about scope so you know which variables you have access to.

For Example, consider the Point3D class below and specifically look at the getX method.

public class Point3D
{
   private int x,y,z;
   
   public Point3D(int tmpX, int tmpY, int tmpX)
   {
      x = tmpX;
      y = tmpY;
      z = tmpZ;
   }
   
   public int getX()
   {
      return tmpX;
   }
   
   //Other methods not shown

}

This is a common mistake that people make when attempting basic accessors.  The programmer is confused between the instance field x and the parameter tmpX.  This will not compile because getX is not within the scope of the tmpX variable.  What the programmer meant to do was to return the x variable, which the getX method has access to.

 

 

Block Scope

Java uses block scope, meaning that the easiest way to determine whether or not you have access to the variable is to follow the following rule:

The scope of a variable is limited to the block (curly braces) that it is declared within.

If you look at the Point3D object again.  Notice that the block that instance fields are declared in is the entire class, this is why the instance fields are accessible from any place in the object.

The tmpX, tmpY, and tmpZ variables are declared in the constructor, meaning that you can only use them during that constructor (hence the error when trying to return tmpX from another part of the program.
