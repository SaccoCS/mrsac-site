String Accumulators

We've been getting used to the idea of looping through our Strings one index at a time using variations of this loop:

for( let i =0; i < input.length; i++){
     let letter = input.charAt(i);

}

The i variable represents every index in the string, and each iteration it's used to access the individual character at each index.

 

So far we've mostly been using these loops with a counting variable. For example, this loop would count how many letters in the String are upper case.

let count = 0;
for( let i = 0; i < input.length; i++){
     let letter = input.charAt(i);

     if( letter >= "A" && letter <= "Z")  {      //detect if letter is a capital letter.
          count++;
     }           
}
return count;

 

A counting variable is a type of variable known as an accumulator. 
- Before the loop it is initialized to a neutral value
- During the loop it is occasionally increased
- After the loop has completed, the variable has its final value

 

 

String Accumulator Demo: justTheCaps()

Open the justTheCaps() workspace.

For this assignment we are going to focus on String Accumulators, which are treated similarly to counting variables , but their purpose is usually to gather text information into a new String.

Take a look at the justTheCaps() function provided to you, which is meant to return a String made up of all the capital letters that the parameter String has.

justTheCaps( "Mister Sacco") --> "MS"
justTheCaps( "Scripps Ranch High School" ) --> "SRHS"
justTheCaps("HeLlO wOrLd!!!!")  --> "HLOOL"

function justTheCaps(input){
  let caps = "";
  for( let i =0; i < input.length; i++){
       let letter = input.charAt(i);

       //detect if letter is a capital letter.
       if( letter >= "A" && letter <= "Z")  {      
            caps += letter;
       }           
  }
  return caps;
}

Look at the caps variable here, which is being used similarly to a counting variable:
 - It starts at a neutral value: ""
 - During the loop, we occasionally add to it.
 - After the loop, we consider it complete and return it.

The difference this time is that we aren't keeping numerical track of the capitals. When we say caps += letter; we are adding the character itself to the String. By only adding when letter is a capital, we end up with a string full of capitals.

The nextClue() function:

This function is what is called when a letter is pressed on the keyboard. Its purpose is to determine what the next clue should be when a letter is guessed.

  • The current clue: a mix of underscores and letters
  • The letter being guessed
  • The answer to the game, to determine if the letter matches a specific location

 

nextClue( "S___", "C", "SACCO")  ->  "S_CC_"
nextClue("CO___T_R", "M", "COMPUTER")  ->  "COM__T_R_"
nextClue("P__G__M", "R", "PROGRAM")  -> "PR_GR_M"

Notice that the returned string is almost a copy of the original clue. The only characters that are not directly from the clue are the ones from the answer that match our guessLetter.

Phase 1) Revealing the guessed letter among underscores.

To simplify it a little bit, the first version of this function will assume that the current clue is entirely underscores. Which means that it will return a String that is made up of either the guessLetter or and underscore.

For example: Assuming the answer is "SACCO"

guess("A") would return "_A___"

This function is another String accumulator problem. While looping through the answer string, your job is to add a single character each time: either an underscore or the parameter letter. The way you will decide which to add should be based at the character in answer at the given location. If it matches the parameter, add it to your accumulator. Otherwise, add an underscore.

Note: You should not worry about adding spaces. The addGaps() function will add them later. You only need to worry about each character.

Phase 2) Incorporating the Current Clue

As you type, you should see the problem. We want to reveal the letter from answer, but assuming that all the other characters are underscores means that any previously revealed letters are forgotten. The clue parameter is where that information is being held. 

Modify your loop to, instead of adding an underscore, add a corresponding character from the clue variable. In the beginning they will all be underscores, but as characters are 'revealed' in clue, they will never go back to underscores.

At this point your should be using charAt() to get characters from both the clue and the answer. Most of the time, you are simply adding whichever character is in the clue at a given location. The only reason that you wouldn't pull the letter from clue would be if the answer has a character at that location that matches the guessLetter parameter. In that case, you can add the guessLetter instead, effectively 'revealing' it in the clue.
