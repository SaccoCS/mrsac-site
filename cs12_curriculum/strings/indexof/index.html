indexOf()

We've been using the substring method to determine which characters were at a specific location. You give it indices and get back a String in return. The indexOf() method works in the opposite way. It is a searching method whose purpose is to find the index of a specific String.

It accepts a String as a parameter and will return the index that it can be found.

let name = "Mr. Sacco";
let saccoIndex = name.indexOf("Sacco");   //returns the index where 'Sacco' can be found (4);

Again, we are using functions that belong to the String as an object, so we call it using dot-notation. We are asking the String to tell us information about itself.

 

Special Value: -1

The indexOf() function is great at locating a String within another String, but we need to consider the situation where a specific substring cannot be found. Consider the ridiculous example below.

let schoolName = "Scripps Ranch High";
let saccoLoc = schoolName.indexOf( "Sacco" );              //CANNOT BE FOUND:    -1

The String "Sacco" is nowhere to be found in "Scripps Ranch High". To indicate that it does not exist in the string, a special value of -1 is returned.

Pig Latin 2.0

Our first version of Pig Latin was 'Lazy' because it assumes that the first letter of the word is the same as the first syllable and thus hardcodes index 1 as the index that we need to use to break the String apart. This works a lot of the time, but fails for many words:

"school" -> "choolsay"
"thunder" -> "hundertay'
"storm"  -> "tormsay"

 

The pigLatin2() function is meant to be more careful about where the String is broken up. Instead of always assuming that index 1 will be the dividing point, it will instead use the location of the first vowel as the basis for your substrings.

- Find the index of all five vowels
- Determine the lowest one that is not -1

 

King of the Mountain Strategy

We are attempting to find the lowest index in a group of five possible indices.The best way to accomplish this is to use a king of the mountain strategy. We will first call indexOf() with all five vowels to get five different possible indices: aIndex, eIndex, iIndex, oIndex, uIndex. To find the lowest, we will create a separate variable that represents our 'lowest so far'. Then, one-by-one we will look at each vowel index and see if it is lower. If so, it becomes the new 'lowest so far' to be compared with the next.

let lowIndex = inStr.length;          //lowIndex represents the 'lowest so far'. 
                                      //We are starting it at a large value, and 
                                      //replacing when we come across a smaller one

if(  aIndex != -1 && aIndex < lowIndex )         //if a is found and its index is less than the 'lowest so far', 
     lowIndex = aIndex;                          //It becomes the 'lowest so far'

//repeat the process for the eIndex, iIndex, oIndex, and uIndex
//After all possibilities have been considered, the lowIndex variable will be the lowest of the bunch

Notice how the if statement is checking two things:
- Is there an "a" in the word:  aIndex != -1
- It's index the lowest that we've found so far   (aIndex < lowIndex )

After the if statement runs, the lowIndex variable may contain the location of the "a", but it is only the lowest so far

You can then finish creating a pig latin word using the lowIndex variable as the break-point.

OPTIONAL CHALLENGE: Cash Format

The purpose of the cashFormat() function is to accept a String that is meant to represent a dollar amount. Your job is to make sure that it is in 'Cash Format'. This means that there are two key features:

1) A dollar sign at the very front
2) A decimal point followed by two digits

"45.23"  -> "$45.23"
"$18"  --> "$18.00"
"25"   --> "$25.00"
"1.5"  --> "$1.50"

Your function needs to detect if the input is missing either the dollar sign or the decimal with two digits and return the corrected version.

 

Detecting Missing Elements with indexOf()

A key to this function will be the ability to detect if a String is missing something. It's easy to tell if the dollar sign is missing because it should always be at index 0, but what about the decimal point? Depending on the number, it could be at almost any index.

Don't forget that indexOf() will signal to you that something cannot be found by returning a -1.

if( inStr.indexOf( "." )  == -1 )    //if a decimal doesn't exist inside of inStr

 

 

Lots to Check

There are lots of ways that a number cannot be in 'cash format'. Check to make sure that your code will make corrections for the following problems:

Missing Dollar Sign:  "12.34"   -> "$12.34"
Missing Decimal Point:   "$12"  ->  "$12.00"
Too few digits after the decimal point:  "$12.5"  ->  "$12.50"
Too many digit after the decimal point:  $3.14159

All four of these situations can be detected by finding the index of the dollar sign and the decimal point. A -1 will indicate that either is missing, and you can compare the decimal's index to the String's length to determine if you need to add a "0" to make the string longer, or a substring() to make it shorter.

Don't forget that there may be multiple problems, like no dollar sign and no decimal place  ( 25 ). I recommend using if statements to see if there is a problem, and then update the variable to be corrected. For example, we would use an if statement like this to 'update' the cashStr variable with a better version.

if( /* a dollar sign is needed in cashStr */){
   cashStr = "$" + cashStr;
}

This doesn't do anything about the decimal point, but you can add more if statements after to further fix the cashStr String before returning it in the end.
