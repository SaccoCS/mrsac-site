Wordle Clues

The game of Wordle has you guessing five letter words, trying to discover the secret word. After each guess, the squares are colored to give you a hint.

A green square means that you have the correct letter at the correct location
A yellow square means the letter is at some other location in the secret
A gray square means that the letter is not in the secret at all

For example: if the secret string were "SACCO" and the guess was "SPACE", we would see this:

Based on the colors, I can see that:
- The S and the C are at the correct locations.
- An A exists in the word, just not at that index.
- P and E are not in the secret.
 

Open the Wordle workspace in CodeHS

There you are met with a basic outline to get you started. Looking at the top of the sketch you should see some variables:
- The secret variable will hold this puzzle's solution. Of course it's "SACCO" right now, but we can change it later.
- Color variables: GREEN, YELLOW, and DARK_GRAY, which contain some html color codes.

 

 

Part 1: The getColor() Function

There is not a draw function yet, but there is a new function named getColor(), and a setup() function with a little bit of code to test it.

The getColor() function accepts two parameters, a five-letter guess and the focus index of that guess. Its purpose is to return the color that the box at that index should be. This will involve determining the letter at the focus index and comparing it with the global secret variable to see what you should return. When returning, you should return one of the global color variables: GREEN, YELLOW, DARK_GRAY

 

 

The setup() function is testing one of each of these situations, keep in mind that the secret is currently "SACCO"
  - getColor( "SPACE", 0 ) is GREEN because the S at index 0 is a match
  - getColor( "SPACE", 2 ) is YELLOW because the A at index 2 is found elsewhere
  - getColor( "SPACE", 4 ) is DARK_GRAY because the E at index 4 is not found at all

 

Your Task:

Write the getColor() function, which should rely on string functions to decide on returning GREEN, YELLOW, or DARK_GRAY:
 - charAt(  index ):  Return the character at a given index
 - indexOf( str ) : Returns the first index that str is found, or a -1 if not.

 

The three pieces of information that you'll need are:
1) The letter from the guess at the given index
2) The letter from the secret at the given index
3) The index of the 'guess letter' in the secret ( which may be -1 if it doesn't exist)

Part 2: Drawing the Guess

Further down the sketch, find the drawGuess() function, which takes in a guess as a parameter and is responsible for drawing it to the window.  Right now, the window is 360 pixels wide. This is enough for five 60x60 squares, spaced 5 pixels apart, with a buffer of 20 on each side.

For each index in the guess parameter, you will need to:
  - Get the character at that index
  - Call getColor() and draw a square with a fill() of the resulting color.
  - Draw the character in white on top of that square

 

Remember that an accumulator variable would be helpful for tracking the x values of each square:
  - Create a local x variable and assign it to 20 before the loop ( 50 if using rectMode(CENTER) )
  - During the loop, draw each square at x
  - After drawing, add 65 to the x to increase it for the next square

 

Test

Run your program. The default guess is SPACE, which will look like the above example, but you should be able to type others and get an accurate drawing.
