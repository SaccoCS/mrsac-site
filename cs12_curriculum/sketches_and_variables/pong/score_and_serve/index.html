The Left Serve

The serve function's job will be to put values into the x, y, xSpeed, and ySpeed variables. This will be four assignment statements, similar to what was just removed from the setup() function but less random.

Instead we want to simulate a 'serve' from left to right. This means two things need to happen:

  1. The location of the ball should be on the left side of the canvas. Inside the serve() function, set the x and y variable to values representing a location that is just to the right of the left paddle's x and halfway down the canvas.
  2. The direction of the ball should be heading to the right. The serve function should set the ySpeed to be a random value from -5 to +5 like before, but the xSpeed should be randomized from +2 to +5. Making the xSpeed positive guarantees that it will move in the right direction.

Serve Delay

After an intense round of Pong a player's mind might be overloaded. Let's give them a little bit of a delay before serving the next time.

 

Variable Based Delay

We've seen before the value of a 'countdown' variable, which the draw function is constantly trying to decrease towards zero. Variables like this are useful for introducing a delay to the game.

Add a global variable named delay to the program. In the serve() function initialize it to around 30. We will then need that delay variable to always be counting toward 0. In the draw function add an if statement that will decrease the delay variable by one, but only if it's above 0.

With this setup, the delay variable is in one of two states

  • Counting down toward zero
  • Sitting at zero

We want the delay to affect the motion of the ball. Currently the moveBall() function automatically increases x and y by xSpeed and ySpeed, causing the ball to always be in motion. Modify the function with an if statement so that it will only do this when the delay variable is 0. This change will make it so that the ball will not be moving as long as the delay is in the 'counting down' state.

You can test this by running the program. Since the delay variable is starting at 30, you should see a small pause before the ball initially moves. Since the delay variable is reset to 30 every time a serve() occurs, you should get a delay after each score.

Paddle Bouncing

Bounce Angle

Our Pong game is a little bit predictable because the bounce angle is a little predictable.  To add a little more skill, the angle that the ball bounces should be related to where on the paddle the ball hit. In the demo above, notice that when the ball hits the paddle, it bounces off at an angle relate to where on the paddle it hits

If it hits the bottom half of the paddle, it will bounce down. Hitting the top will bounce up.

To accomplish this we will be changing what happens during the bouncePaddles() function. When a paddle is hit we will still be flipping the xSpeed to its opposite, but now a new value for ySpeed will be chosen based on the ball's distance from the middle of the paddle.  This will give the player a little more control over where the ball is going.

The rule is that the further from the center we go, the sharper the angle of travel should be. 

 

To obtain this effect, we'll keep the xSpeed going at whichever pace it desires, and we'll set the ySpeed to a value that gets larger when the ball is further from the center of the paddle.

The first thing you should do is calculate the difference between ball Y and the paddle Y.  Your new ySpeed should be about one sixth ofthe difference of the two centers.

ySpeed = (y - paddleY) / 6

This strategy is not perfect.  The speed of the ball will be faster if the angle is larger.  In reality, maintaining the speed of the ball requires the distance formula and some trigonometry.(Don't tell your math teacher that it has a practical application.  Just keep thinking it's a useless skill to have)

 

 

 

 

Increasing Difficulty with xSpeed

The way that our game is set up, the xSpeed is in charge of how long it takes for the ball to make its way across the window.  Right now its magnitude is always the same, which makes for an easy game. As the game goes on, we want to make the ball move faster and faster to help make sure that one of the players eventually misses.

To do this we will have the ball's xSpeed increase in magnitude a little bit every time that it bounces off of a paddle.  This can be done by changing the calculation slightly from:
