Timing

We've seen before that the draw() function is repeatedly called while the program is running. The programs in this assignment take advantage of this timing.

The Final Countdown

Observe the workspace above. Clicking the workspace causes the number in the middle to be set to 20, and it immediately starts counting down to 0. Once the countdown reaches 0, the background starts flashing random colors.

Open the Final Countdown workspace. There you have been given some code to help you get the basic structure, but the provided code is static and doesn’t do anything. You can see that the call to text() is always drawing a 9 on the screen.

Counting Down

We need to incorporate a variable to help us perform the “counting down” action:

  • Add a global variable named count to the sketch, and initialize it to be 20 in the setup() function.
  • In the draw() function, modify the call to text() to draw count instead of the "9" that it is currently drawing.

At this point you should be able to run the sketch and see a 20 being drawn on the square. There will not be any counting yet.

Counting Down Each Frame

The way that we are going to count down is by having the count variable change a little bit each time that the draw() function is called. But if we simply add count--; to the draw() function, it would never stop counting down.

Our goal is for it to stop at 0, so we can wrap it in an if statement that only decreases the count if it’s larger than 0: 

if (count > 0){
    count--; 
  }

Add this if statement to the draw() function and run your program. If done properly, the number should countdown and end at 0.

The Two States of the Countdown

With a countdown variable set up like this, we can think of there being two different states:

  • The countdown is occurring (count > 0).
  • After the countdown has ended (count == 0).

This is useful information because the background of the sketch is drawn differently in each of those states. Modify the draw() function with an if statement to draw a white background while the countdown is occurring, and to randomize the background if the countdown has ended.

Resetting the Countdown

Currently your program should countdown from 20 to 0 and stop there. The last step is to “reset” the countdown when the user clicks the canvas.

Add a mousePressed() function to the program. To have it start the countdown over again, all you have to do is assign a value to the countdown variable: 

count = 20;

Resetting the variable in this way will cause the countdown to start again, with each call to draw() decreasing it back towards 0.

Goals:

  • Give each button its own countdown variable.
  • Make the buttons fill based on the state of their countdown variable.
  • Use mouse presses within the squares to reset the specific countdown.

A Countdown for Each Button

Each button is to be controlled individually. Add three global variables to the program: redCount, greenCount, and blueCount. In the setup() function initialize these variables to all be 50.

During the draw() function is when the countdown occurs. Have a separate if statement for each variable that, like before, will decrease the variable if it is above 0.

Countdown-Based Fill

We have now established the individual countdowns, but they so far have no effect on how the canvas looks. Remember that there are two states that the countdown can be in: actively counting down, or at zero. The next task is to have each of the squares choose their fill based on that count.

Modify the draw() function so that, when a square’s countdown is actively counting (count > 0) it chooses a fill of its color, and otherwise will choose noFill().

At the current moment, all countdowns are initialized to be 50, so running the program should cause all three buttons to be filled for a small amount of time before clearing out.

Triggering with the Mouse

Now you have three buttons that have their own individual countdowns. The next step is to get them responding to mouse presses. The goal is to have it so that clicking the individual buttons resets their counter to 50. This requires the mousePressed() function detecting if the mouse is within a box.

Your task will be to modify the mousePressed() function so that if you click within any of the three boxes, their individual count will be set to 50. One way to handle this is to go and grab the mouseInBox() function from the Boolean Functions assignments and copy it into this project. Once it's there you can call it during the mousePressed() function to help you tell if you clicked any of the three boxes.

Test

Add a new global countdown variable, which will represent the amount of time that the dice will be “rolling” before stopping. Like before, in the beginning initialize it to be a value. Spinner

Click the demo below a few times to see what the next sketch is intended to look like.

There are a few elements that we need to notice:

  • The rotation of the spinner is temporary, but it doesn’t go full speed and then suddenly halt. The rotation slows down and eventually comes to a stop.
  • The message drawn disappears while the spinner is spinning.

Open the Spinner workspace in CodeHS. There you have been provided with some structure to help you complete this program:

  • There is a function named drawSpinner(), which uses translation and rotation to draw the arcs of the spinner.
  • There is a global variable named angle. This variable is used by the drawSpinner() method to determine which angle to rotate the spinner to. 0–72 is when the red section is up, 73–144 is orange, 145–216 is green, 217–288 is blue, and 289–359 is purple.

In order to get the motion that we want, we are going to have a variable that is similar to our previous countdown variables, but is a little different. Add a global variable named rotationSpeed. Its purpose is to determine how fast the spinner is rotating. In the setup() function assign a value of 15.

Like before, modify the draw() function to decrease the rotationSpeed as long as it is positive, but this time decreasing by 1 will be too much. Instead, subtract a smaller number, like 0.1.

So far, this rotationSpeed variable has no effect on the way the sketch is drawn. It is meant to represent how fast the angle is changing, so add the following line to the if statement: 

angle += rotationSpeed;

This will cause the spinner to rotate as fast as the rotationSpeed tells it to. As the rotationSpeed gets smaller, the spinner will rotate slower and slower.

Triggering Another Spin

Like before, clicking the canvas should re-trigger our spinning. Modify the mousePressed() function to assign a random number to the rotationSpeed variable, between 15 and 30.

Run your program to verify that this works.

The Result Message

The last bit that we need to add to our spinner is the text that appears on the bottom of the window after the spinner has stopped, showing the result.

This means that our draw() function either wants to be rotating the spinner or displaying the text, so add an else section to your original if statement to represent when the spinner has stopped.

When stopped, you need to use the value of angle to determine which color has been chosen:

  • 0–72 is red
  • 73–144 is orange
  • 145–216 is green
  • 217–288 is blue
  • 289–359 is purple

Dealing with Large Angles: One issue that arises is that we've been adding so much to the angle that it has gone far past the 360 range that the results message expects. The solution is to mod the angle by 360 before you compare it. 

angle = angle % 360;

This will 'subtract as many 360s as possible' from the large angle. The result will be a value that is less than 360 and represent the same amount of rotation as the original.

Test

After adding the result message you should have a program that functions like the demo. Run it and try spinning many times to make sure that you are getting a proper spin and result.
