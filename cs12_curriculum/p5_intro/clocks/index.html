Checking the Time

p5 comes with three methods to help you look at your computer's clock.

hour()
minute()
second()

Each of these functions returns a number representing that particular section of time. This allows us to write programs that act as a clock.

Open the Displaying the Time workspace. There you will find a simple sketch that calls each of the time functions, then display's the current time.

  let h = hour();
  let m = minute();
  let s = second();  
  
  text(h+":", 10 , 35); 
  text(m+":", 70, 35);  
  text(s, 130 , 35 ); 

Since it uses the hour(), minute(), and second() functions, the time displayed should match that of your computer's clock.

 

Your Task: Guaranteeing Two Digits

Right now the Clock display looks a little weird if there is only one digit. It's better if there is always the same number of digits, so adding a leading 0 would be good for the formatting.  Modify the clock to have an if statement that first looks at the value of s to see if it's less than 10.  If it is, draw the seconds with a leading zero in front.  If not, just draw like normal.

For example:  9 seconds should be represented with '09', while 15 seconds should still be '15'. All numbers should be represented with exactly two digits.

Though you're only going to really be able to check the seconds, you should perform the same action with the minutes and the hours.

 

Slider Clock

The Slider Clock represents the time as a circle that's drawn along a line, like slider.

Mapping Hint: Notice in the example that the time changes the x location of each circle. Since each line is drawn from x:50 to x:350, we can use that as our desired range.

This means that we need to map seconds, for example, from 0-60 to 50-350. It may be weird that the destination range doesn't start at 0, but the map function handles it just fine.

let secX = map( s, 0, 60, 50, 350);

 

Warning: Hour is 0 - 24

The hour function returns the hour in the non-ambiguous military format, so it is in the range 0-24.  Since we want it to be in our more familiar '1-12' range.  Have an if statement that will look at the hour and subtract 12 from it if it's too large.  Note: This strategy technically doesn't work for midnight, but you don't have to fix it.  Mr. Sacco won't be grading your assignments then.

 

Create a Slider Clock that is similar to the example.  FOR FULL CREDIT: You may not use the range 50-350, like the demo.  Use different values.

 

 

Representing H:M:S as RGB

This time your job will be to create a clock where the current time is represented as a color:

- Red is derived from hour()
- Green is derived from minute()
- Blue is derived from second()

Since there are different ranges involved here, the map() function should be on the hour(), minute(), and second() values to get a number in the RGB range of 0 - 256.

 

Step 1) Setting the background to the time

Every new sketch has a default background(220); statement as its first line, giving a gray background. But that can be changed to represent a more colorful background if you give it three RGB parameters: background(255, 0, 0); would be a red background.

Your first task is to make the background color match the time. First use the map() function three time to get the proper red, green, and blue values and assigning them to variables. Only after that should you call background(r,g,b); using those variables. If done properly, your background should match the demo above.

 

Step 2) Set the text to the Negative color

After your program's background matches the example, you can move on to writing the text.  The fill of the text should be the negative of the background.  The negative of an R, G, and B color is the color you get by subtracting each of them from 255.  A negative would be (255-R,255-G,255-B).  Use that to write the time on top of your background.  Again, since everything is based on the current time the colors should match the examples exactly if it's done right.

 

 

Traditional Clock Face

A traditional clock has hands that rotate with the time.

There are ways of drawing the hands, but the first thing that you must do is to think of the angle that each hand should have. For this, you will need to map() each value from it's natural range to the 360 range.

 

There are two different ways that the hands can be drawn. Choose your favorite:

 

Drawing the Hands Using Rotation

- To draw a line rotated to a specific angle, first translate to the center of the clock, rotate to the angle you want, and then draw a vertical line from (0, 0) to (0, -50).  Even though you're always drawing the line 'up', the rotation before it will cause 'up' to be the appropriate direction.

Be careful: Don't forget that the rotations you make will persist.  You may need to resetMatrix() between drawing hands.

 

Drawing the Hands Using Trig

Each hand is just a line, where one end is at the centerpoint, and the other is somewhere on a circle. Since you know the angle that you want to draw the hands, and the center of the circle, you can break out our circle equations to calculate where the other endpoint would be.

let x = centerX + radius * cos( angle );
let y = centerY + radius * sin( angle );

Note: With trigonometry, angle 0 is directly to the right of center, which will cause the hands to be rotated 90 from where you want them to be. You can counteract this by using angle-90 in the equations instead.
