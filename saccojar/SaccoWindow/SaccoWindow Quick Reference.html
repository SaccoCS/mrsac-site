SaccoWindow Quick Reference

 

Building a Basic SaccoWindow:

SaccoWindows can do a few things, but they all build off of first creating a custom SaccoWindow object.  In order to define your own custom SaccoWindow, you must follow a few rules.

1)  Link your BlueJ to sacco.jar:  This only has to be done once

2)  import sacco.*;  - This is the location of the SaccoWindow that we are building off of

3)  Extend SaccoWindow - In order to inherit all of SaccoWindow's abilities, you must make sure that your class extends SaccoWindow

4)  Override the paintWindow Method:  This is where you use the Canvas object to define what the window looks like (unless you're happy with a blank, white window

 

Example:  The DemoWindow below will appear with a big green rectangle in the center of it.

import sacco.*;
public class DemoWindow extends SaccoWindow
{
    @Override
    public void paintWindow(Canvas c)
    {
       c.setColor("GREEN");
       c.fillRectangle(50,50,400,400);
    }
}

Event Handling:

A SaccoWindow is intended to not interact with the user using a terminal.  The way that the SaccoWindow interacts is by responding to events (mouse, keyboard, or timer) as they occur.  This is called Event Handling.

For each type of event there is a corresponding method that you must override in order to use it.  In that method you put the code that you want to run when that event occurs.
